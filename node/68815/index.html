Path Direction Question

March 26, 2010 • TypeSETit

I have been converting fonts from PostScript / TrueType that I originally designed in Fontographer to OpenType fonts using Font Lab 5.0.4 MAC version.

In Fontographer, a CORRECT path direction is defined as having the outer contour or path of a glyph running clockwise, and the inside counters of a glyph running counter-clockwise. An INCORRECT path direction will result in the character looking heavy.

[img:sites/default/files/old-images/pathdirection_6532.jpg]

It appears to me that FONT LAB has the paths running in the opposite direction from Fontographer. Is this true, or am I reading it incorrectly?

[img:sites/default/files/old-images/pathdirections1_4611.jpg]

Does it matter which direction the paths are created so long as the counters run opposite to the outer contours? If so, which is correct… the way FONTLAB defaults, or the way FONTOGRAPHER says paths are correct?

This is a matter of curiosity AND I need to know before I go into over 700 glyphs to make sure they have correct paths.

Thanks in advance.

RL

I don’t think it matters - just depends on whether you’re making PS or TT flavoured OT. PS flavour has outside contours running anti-clockwise. I usually just select all glyphs and change contours to postscript.

2010-03-26 18:42:13 • TypeSETit
FontLab Studio renders PostScript paths in the direction recommended by the Adobe Type 1 font format specification: the outer path runs counterclockwise. See http://www.adobe.com/devnet/font/pdfs/T1_SPEC.PDF pages 27-28.
