2010-08-24 18:20:38 • dberlow
We should avoid the term ‘conversion’ when talking about creating WOFF files. WOFF is not a font format, it is a font delivery format. The SFNT font data content of a WOFF file is 100% identical to the un-WOFF'd font: there is no conversion of that data taking place, only compression and wrapping. I don't think it will be possible to get all foundries to formally agree on a single policy with regard to whether to permit WOFF creation by customers, any more than it has been possible to get them all to agree on a common EULA. One can certainly point out to foundries that permitting customers to create their own WOFF files from licensed desktop fonts sort of means not taking advantage of one of the features of WOFF, which is the ability to include web specific and customer specific metadata in the WOFF'd font. If you allow end users to create their own WOFF files, those files are indistinguishable on the web from illegitimately created WOFF files from existing desktop fonts. Since one of the main ideas behind WOFF was to draw a line, at the delivery format level, between new web licensed fonts and existing desktop licensed fonts, surrendering control of the WOFF metadata to customers undermines this. Smart foundries will be serialising their WOFF files, matching fonts to customers and to licensed domains. Very smart foundries will be inserting serialisation information and whatever recommendations or other metadata into the font prior to WOFFing. A smart WOFF creation tool for foundries will automatically perform these steps, e.g. using data from a customer database, as part of creating a WOFF file for delivery to or serving on behalf of a customer: insert license specific data into name table and add any custom tables to font; digitally sign font; create WOFF file.

2010-10-06 04:43:14 • abattis
Dave: Of course it is - the font is distributed to each visitor to the website. While I agree that using a font on a web page constitutes a kind of distribution, it isn't what I would consider the primary meaning of ‘redistribution’ in terms of a EULA, which would be intentional selling or giving away copies of a font for use by other parties. A font on a website is displayed to each visitor to the site, and only as a by-product is it sent to that local machine in some format and possibly cached. It is convenient that existing font licenses that prohibit redistribution also prohibit web serving on the grounds that this technically involves distributing the font, but moving forward I don't think it is in anyone's interest to continue to lump web font serving with redistribution, any more than it has been to lump PDF embedding with redistribution. These are specialised uses of fonts, which should be clearly and independently covered by appropriate license terms. As new technologies make new things possible, updated license agreements should explicitly clarify whether these things are permitted or not.
