2007-09-17 11:27:10 • mimi lambourghini

I am not sure what would cause the ‘Missing OpenType data’ for an OTF, but I know that TrueType fonts which lack a ‘fpgm’, ‘cvt ‘, or ‘prep’ table will generate that in Font Book under OS X 10.4. This has been confirmed as being a bogus error; as this condition is common in unhinted TrueType fonts. You can avoid the error by generating TTF with hints; or just ignore the error and install the font anyway.

It would not surprise me if there’s something similar going on for OTFs (i.e. a table that Font Book is assuming should be there but isn’t generated by default).

2007-09-20 15:23:40 • sherman zipf

It sounds like you’re talking about the ‘gasp’ table settings. The ‘gasp’ table controls sizes at which the rasterizer will apply gridfitting, if the font contains hints, and greyscaling (antialiasing) under Windows (no effect under Mac OS as far as I know). If the font does not contain hints, then the gridfitting setting will have no effect (there are no hints to apply). But this setting does not control whether the font contains hints to begin with.

Back to the issue of the Font Book validation error, you can safely ignore that if the font is otherwise okay (check out Microsoft’s Font Validator if you haven’t already); it’s a bogus error. But if you want to eliminate the error from appearing, you’ll need to get hints into the font somehow (specifically, the ‘fpgm’, ‘prep’, and ‘cvt ' tables). I don’t know whether FontCreator is capable of generating new hints (autohinting); you may need some other tool if not.
