The designer (who’s also using OS X) has uploaded them to her ftp site, where each displays as a series of stuffed files and a few empty files (e.g., “0 KB”)

When I download them and run them through Stuffit Expander, I end up with LWFN files which the computer refuses to recognize. Sounds like you’re using old OS9 fonts. Ask the designer to Stuffit the fonts before he/she uploads them. (This way the information contained in the resource fork of the font files won’t be lost when they are uploaded. Stuffing the files will temporarily transform their resource fork into a data fork ‘format’, which other OSes can read. In other words, the files won’t show up having a zero KB size.)

Here’s the short explanation of what’s going on: Resource Fork: The PC and Mac differ in the way they store files. On a Mac [Classic] each file can have two parts called forks. The data fork holds data (text, images, etc.). The resource fork holds resources (icons, fonts, menus, sounds, etc.). There are actually two files linked to one name in the file system. PCs only have one file linked to each name. Fork: A fork is a stream of data associated with a file name in a file system. Macintosh HFS files have two forks. PC FAT and FAT32 files have one fork. NTFS files can have many forks. http://www.asy.com/glossary.htm

And here’s the long one: http://en.wikipedia.org/wiki/Resource_fork
