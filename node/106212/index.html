2013-10-06 05:41:43 • noisiak

Thanks noisiac

It’s a Big project, and not so trendy, but we love it :)

Working in the browser is challenging 1. for computing resources, since we allow real time editing of over 100 glyphs simultaneously, so newer computers recommended for better performance. 2. cursor sensitivity is different in the browser than on desktop and harder to be controlled. in some cases you better edit paths in 100% zoom (there are 100% and Fit, zoom buttons on the bottom right of the work area for quick zoom toggling).

Editing the Outline of the glyph is done in the Outline layer (the layers buttons are the big ones on the top of the work area), there you have full control over the Outline regardless to the skeleton (if that’s what you meant by “actual stroke”). You can see it here: http://www.youtube.com/watch?v=EYMyuffi1O0#t=64

The thing is, in this stage of the development, outline changes are not synced, you have to do it manually to each glyph on it’s own, same as in any other software, and you better edit the outline after you’ve finished with the skeleton because currently any modification of the skeleton is regenerating the outline automatically. It’ll take some time until we’ll finish the outline synchronization development and advanced outline modification tools but we’re working hard on it.
