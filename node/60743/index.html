Problems testing fonts with Font Explorer X

August 7, 2009 • jonathanhughes

I'm having issues when I'm trying to test out fonts I've generated from Fontlab. Here's my usual process: • I generate the font • I create a new set in Font Explorer X and import my new font into that set • I activate the set and try it out in Illustrator or Indesign At this point, everything is fine. But then if I make changes to the font in Fontlab and generate a new font, I run into problems: • I generate the font • I deactivate the old font in Font Explorer X and delete it (Font Explorer isn't moving my fonts to its own library, so the font I'm generating from Fontlab is replacing the version the Font Explorer is accessing). • I import the new version of the font At this point, two things happen. Either the applications still use the old version of the font (even after restarting them), or when I go to use the font in InDesign (or whatever), no characters show up. Am I doing something wrong, or is there a better way to test out in-progress fonts? thanks! Jonathan

A few things: • If you can avoid it, don’t use FontExplorer for testing fonts unless it really is the final version. Just drop the fonts into the Adobe fonts directory. That way if something goes wrong you don’t have to consider FEX as a variable. • Clear the application and system caches and reboot • Make sure that your regular weight is named correctly. If the regular weight of a font has misconfigured name information, the Adobe apps can freak out and not load fonts from the family properly until a correctly named regular weight has been installed.
