2010-08-26 19:58:47 • Freems

The anchors you set in FontLab have no relevance to finished font. They are just used internally in FL to produce composite glyphs, and they don’t automatically produce a »mark« feature when you save a feature file from FontLab. You have to write the mark feature code yourself. (You wouldn’t even need to set FL anchors in order to do so, though it’s easier to check the positioning of components when you have anchors and can preview the effect in the FL preview panel.)

If you have for example an anchor in the glyph »A« at position (365, 725) and an anchor on the glyph »tildecomb« at position (0, 500), the corresponding mark feature code could look like this:

feature mark { markClass tildecomb @ANCHORS_top; position base A mark @ANCHORS_top; } mark;

When this code is in your OpenType panel, FontLab won’t know what to do with it (pressing the compile button in FL will fail), but saving the complete feature code to a file and generating the font with Fontographer (or use the command-line tool makeotf from the Adobe FDK 2.5 directly) should work. (I don’t have Fontographer to check it).

Hope this helps! Jens
