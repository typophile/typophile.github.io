Best font for contracts

September 19, 2011 • Andrew Feldstein

Is there a better compromise font than Times New Roman for creating contract forms?

I’ve read a lot of “best font” discussion. I basically get the difference between fonts designed for reading on screen and those for reading on paper. Our contracts are typically 3-6 pages (occasionally longer) with 1,750 or so words per page (more or less depending). Generally we use a 9pt, 10pt, or 11pt font (never less than 8pt). Personally, I read and edit almost 100% onscreen. However, my company’s customers and my fellow employees read both onscreen (in Word and PDF formats) and on laser-printed pages. And in all cases the final contracts are laser printed before being signed (and scanned—so OCR accuracy is a plus).

So, is there a better alternative?

2011-09-20 22:12:15 • Andrew Feldstein

The problem is, essentially, balancing screen readability, for which only a few fonts excel, with printed readability (and attractiveness), for which many excel–and they seem to be mutually exclusive.

Right, but you’re overlooking one key issue. The Microsoft system fonts (Georgia, Calibri, Cambria, et al) are designed & engineered to look great on screen in Windows applications that use Windows text rendering, like Word. But PDF is a Postscript-based format and PDF readers use their own text rendering. (For instance, Adobe Reader uses a technology called CoolType. Why? So ensure consistent text rendering between platforms.)

Therefore, in PDF, the MS system fonts have no screen-legibility advantage over print-optimized fonts. And since any PDF could also be printed, your best bet is always going to be a print-optimized font (like Miller) not an MS font (like Georgia).

I don’t frame the typography question in terms of “how will readers use this document?”, because you can’t control that. Rather, I frame the question as “how does this document invite readers to use it?” because that’s within your control.

For example, when a software installer puts 2500 words of teeny text in a small scrolling box, the typography sends a message. That message is: “I’d prefer you didn’t pay attention to what’s in this box.” When half the words are in caps, that message is even louder.

For another example, I was writing a font license last week. My view is that someone buying a font should read the license.

As research, I read about 25 font EULAs from other foundries. But a lot of those licenses — no disrespect to the people behind them — are crazily long and dense. I have a law degree, and I still had to make little pencil diagrams to follow what was going on in some of them. I’ve no idea what the average MyFonts customer is supposed to do with them. But I know what the average MyFonts customer actually does: ignores them. Because they beg to be ignored.

So how do I get someone to read a font license? I decided that it should be one short, inviting 8.5" x 11" page. The customer can print it if they want, but it also had to be legible on screen. The text of the license had to fit on the page without any typographic trickery (tiny font, condensed font, etc.) That, in turn, put some constraints on how the license was written. But I finished it, and the first customer who got it said “Wow, that’s a very clear license.”

That’s a long way of saying that the answer to your question depends on whether you assume that contracts can learn nothing from the “casual narratives” or “brief snippets” that we usually see on the web.

But then came copiers that could enlarge the small print–a godsend. (Long ago my company used preprinted forms in 6pt that were often faxed around!)

Nowadays with editable word processing files it’s easy enough to use the zoom function on the computer, this being especially useful if a document is formatted in columns–with “draft” view you see only one column at a time and there is no horizontal scrolling. Sometimes though, a PDF, fax, or hard copy is the only version provided for review (bad form, I say, so I always ask for an “editable” copy if possible). In this case I’ve resorted to OCR’ing the document so that I can reformat it to read it (and to search and use text comparison software).
