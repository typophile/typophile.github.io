There’s an option in Windows 7 for setting the “compatibility mode” of an (older) application so as to help it run better under the new OS. Select the app icon (or a shortcut, even), right-click, get “properties,” go to the “compatibility” tab and look at your options. You could for example try setting compatibility to “Windows XP”….

No promises that this will help, but it’s the one other thing to try (besides a virtual machine) before giving up.

@ijgolf

Thomas is on to something, I hope. (I’m a Microsoft Certified Network Engineer and former 3rd level desktop support guy.)

  1. Try installing under the Administrator account. Even if your user account has Administrator privileges, I’ve seen problems. Makes no sense but such is Windows.

  2. Uninstall CS3. And then, find the msi or, more probably the exe intallation file, right click it, and select Properties -> Compatibility Tab -> Run this program in compatibility mode for… and select Vista or XP or whatever.

This way, the whole installation gets done in backwards compatibility mode.

Once that’s done, try it. If still no luck, add Thomas’s step to the process and run the program executable in compatibility mode, too.
