how do i combine them into one .otf?

You … don’t. An OTF file is not a font family, it’s a single font.

There is a small workaround for ‘family-aware’ fonts, with the ‘size’ feature, but that has been poorly constructed and has led to errors in the past. Besides, no application supports it (perhaps InDesign CS4 does). Besides #2, you cannot add it with FontLab, you would need ADFKO for that.

ps. what unicodes do i use for alternate characters?

Using Unicodes in the private area would be okay, I guess.

2009-01-12 19:47:03 • Theunis de Jong

how do you export a font with several families? at the moment i’ve got seperate fontlab files for regular/bold, but how do i combine them into one .otf?

As Theunis points out, you can’t make a single OTF file with several fonts in it, but, depending on the way you set up the name fields in your fonts, you can create associations between them so they show up in programs as one family. (See [[Font Naming]] in the TypoWiki.)

follow-up question: in file menu, what’s the difference between – generate font – generate all – generate suitcase ?

Generate Font: Generates a single TTF or OTF font file. (Some other formats, too, but mainly it’s used for those.)

Generate All: Same as Generate Font, except for multiple fonts at one time. Can include all fonts that are open in FontLab or just some of them.

Generate Suitcase: For making old style Mac PostScript and TrueType fonts. These only work on Macs.

ps. what unicodes do i use for alternate characters?

These should either be left unencoded (no Unicode number) or given Private Use Area (E000-F8FF) codes. There are no standards for this area by definition, so you can use them any way you want. They will be unlikely to match glyphs in other fonts that use the PUA, but that’s the nature of PUA. Adobe and other recommend not encoding alternate characters, but there are some reasons for doing so.

… if it’s un-coded …

In addition to Jens, OpenType features are not Unicode dependent; they work with ‘raw glyphs’ instead. But assigning Unicodes to the alternates has the advantage they are visible and can be entered into non-OpenType aware applications. Also, you can use Unicode characters in the InDesign Search & Replace fields, but non-encoded glyphs have to be searched and replaced with the Search For Glyphs (CS3+) function (which does that and nothing more).

Adobe and other recommend not encoding alternate characters, but there are some reasons for doing so.
