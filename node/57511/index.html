John,

The comment line # from AAT ’morx’; type 1, sel: 0, cvrg: 20000002 reveals that the OpenType Layout feature definition was not taken directly from the OpenType Layout tables but instead, was converted from the AAT “morx” table. For this conversion, FontLab Studio is mapping some morx functions into OTL features with the approximately same functionality, which in this case happens to be “rlig”. This conversion never is perfect or accurate, since AAT and OpenType Layout use entirely different models.

We do have the AAT -> OTL conversion practically just for the handful of developers who would like to develop AAT tables and then semi-automatically create matching OTL tables. However, since Mac OS X 10.4 added OpenType Layout support, the necessity of creating AAT tables is of less and less importance. It only makes sense now for complex script fonts, as long as Apple does not implement all the major complex scripts in OpenType Layout. In Mac OS X 10.5, basic support for Arabic has been added and I hope that in 10.6, more will be done (but I don’t know anything specific).

Charles,

Some fonts made by SIL contain both OTL and AAT code. To just get the OTL code, disable “Interpret GX/AAT mort & morx tables” in FLS Preferences, and re-open the font again.

Cheers, Adam
