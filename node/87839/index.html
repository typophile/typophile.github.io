2011-12-04 10:50:11 • Té Rowan
I think I have the solution for titles: the string sent to flash in the flashvars is currently built with escape which does not encode utf-8 characters the way flash is expecting them. This happens at line 561 of the file http://typophile.com/files/js/js_906ebf552a4d48f32d6b19cd75bded6f.js in the definition of the function toFlashvarsString. In order to change as little as possible the code, I would simply replace escape with escape_utf8 at line 561 and define escape_utf8 as follows: 
// encodings for lower characters  returned by escape
//    that differ from those returned by encodeURI
function nkch(c)
{
  var nkc = {};
  nkc["~"]="%7E"; nkc["!"]="%21"; nkc["'"]="%27";
  nkc["("]="%28"; nkc[")"]="%29"; nkc["#"]="%23";
  nkc["$"]="%24"; nkc["&"]="%26"; nkc[","]="%2C";
  nkc[":"]="%3A"; nkc[";"]="%3B"; nkc["="]="%3D";
  nkc["?"]="%3F";                  return nkc[c];
}

// Same as escape for lower characters (from 1 to 127)
// Same as encodeURI above 127
function escape_utf8(s)
{
  rexp=/~|!|'|\(|\)|#|\$|&|,|:|;|=|\?/g ;
  return encodeURI(s).replace(rexp, nkch);
}
The behaviour should stay unchanged on characters from 1 to 127. Michel
