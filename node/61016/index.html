Yikes! So you’re trying to do ketiv/qere distinction display in glyph substitutions? That seems to me a really bad idea. This is a text distinction, and if you want a text that includes both forms and can switch between different display options, then you probably should use XML tagging of the character strings and base your display options on that. OpenType glyph substitution isn’t designed to be able to handle this kind of thing.

Never mind hijacking layout features: you’re hijacking the whole layout model.

2009-08-17 03:30:30 • John Hudson
I wrote to my WinSoft contacts about the ‘Stretched Alternates’ function in InDesign ME. This is a legacy function that makes use to two private features, not part of the OpenType specification, to identify left and right parts of stretched connecting forms in a particular Arabic font. This was implemented to support this Arabic font, Decotype Naskh (not to be confused with the Naskh type from Decotype used in the new Tasmeem add in to InDesign), which was originally in the Middle East version of PageMaker and which used this mechanism because OpenType contextual substitutions were not available at that time. This function is only included in InDesign ME for backwards compatibility.
