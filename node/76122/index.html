> I am using Indesign. I am unsure what the > hyphenation settings is Spacing problems in justified text can often be improved by telling InDesign to use more hyphens, but the hyphenation controls are a little hard to find. 1. Select the paragraph(s) you wish to adjust 2. Open the Paragraph palette (Window > Type and Tables > Paragraph) 3. In the upper right corner of the Paragraph palette is a tiny little arrowhead. Click and hold the arrow and select Hyphenation in the popup menu. (An alternate way to reach the hyphenation controls is to assign a style sheet to the text. The style sheet options include hyphenation controls.) Anyway, once you've reach the hyphenation controls, you'll have various options which are mostly self-explanatory. Try different settings and see how they affect the appearance of your selected text. Be sure that "Preview" is selected so you can see immediately how the text is affected. Here's what the controls looks like (these are default settings). And remember that your changes will only affect the selected paragraphs. [img:sites/default/files/old-images/hyphenation_6256.jpg]

2010-11-10 01:03:12 • skald89
The problem is that you've got a narrow column and 0 hyphens. Select the paragraph and then change the first three numbers in the settings to 5 2 2 (instead of 3 10 1) and see if that helps create more hyphens. Also keep in mind that it's difficult to get attractive justification when there are only a few words per line. Try to get more words per line by widening the column, or by making the type smaller, or by switching to a condensed font. And fix "place,titles" (space missing); InDesign thinks its one long word. Making these changes should help, but remember that InDesign justifies text by adding extra space between words. So you're unlikely to get perfectly consistent spacing between words.

2010-11-10 07:06:21 • skald89
> What do you do then to fix it? I would avoid using justified type in that situation. But if you had no choice, adjust hyphenation settings to get the best results, hyphenating manually if necessary. You might find it helpful to temporarily change the justification to flush left rag right so you can clearly compare the actual lengths of each line and adjust the hyphenation until each line is as close to being the same length as possible. After doing that, change it back to justified. You could also adjust the tracking of each line to reduce the gaps between words, and make adjustments to the Justification control panel, but if you've got huge gaps between words to begin with then I don't think those things will give you really attractive lines. If this problem is occurring only in a few spots in your document, another approach is to rewrite those sentences so they break better.

>I tried that also nothing would change no matter what settings I changed. I am surprised there's no book teaching how to kern, The solution to your problem is proper Hyphenation & Justification, as others have tried to point out — not kerning. (The general public is getting into a disturbing habit of using the term “kerning” to address a multitude of things which it is not. This is becoming a pet peeve of mine.) With a narrow column, you will have to employ hyphenation to have any hope of getting good justification. The numbers in your screenshot are way off base. There are plenty of books discussing the principles behind good hyphenation & justification. Some have been mentioned. Find one and educate yourself about the purpose of each of the settings in both the Hyphenation and Justification dialogs. There are also plenty of previous discussion on Typophile, with examples. I don’t recommend using the actual numbers in Bojev’s example above. You’ll need much tighter minimum word spacing for that particular setting.
