2010-07-24 22:21:25 • JanekZ

There are a few reasons why text might be invisible.

One is that the outlines are not closed – i.e. that you have two nodes, one on top of the other, with no line connecting them –; this typically only affects one or two buggy glyphs, not a whole font, but it’s possible that something in your Illustrator-to-Fontographer workflow has left the outlines open. As I recall, Illustrator doesn’t care about open outlines. It should be possible to select all the glyphs in Fontographer and perform a remove overlap function, which will close any open glyphs.

Another possibility is that this is a font cache issue, which might explain why the font looks okay in FontBook but not in an application. I get this issue on Windows sometimes, if I am installing new versions of a font, and I need to restart the system in order to get the font to display properly.
