creating a "sometimes" styalistic alternative?

December 6, 2005 • charles ellertson

We have a job in to typeset, the first in what will be a long series, the letters of Henry James. It turns out that the publisher wants to use a special ampersand in some places where James used “his” ampersand. While the publisher has not yet found money for an XML version of the book(s), It does seem likely, especially since the eventual publication of all the letters will run many years.

So while it is not hard to create and set the “special” ampersand, if I put it as such in the composition file, there will be a problem downstream.

I believe InDesign CS II does not support salt, which might be one way to typeset different ampersands, while retaining the general Unicode number for an ampersand.

The other way I can think of is to embed a “private use” character of zero width in the file before all the ampersands to be rendered with the special character, then use “contextual alternatives” to set the special ampersand in the book. From limited testing on another matter, the private use character will drop out in plain text, leaving just the call for the ampersand, so an XML file using any font will at least display an ampersand – though not, of course, James’ rendition.

Sill it bugs me to approach the matter this way. Anybody think of a better solution?

TIA

2005-12-06 14:41:17 • charles ellertson

You could also use the titl - Titling - feature, though that’s probably not as Kosher as a Stylistic Set, but it would be supported by Illustrator and Photoshop. (Stylistic Sets are presently only supported by InDesign CS2.)

As long as your alternate ampersand is named “right”, you shouldn’t have problems. The correct form would be ampersand.alt (or ampersand.James, or something like that, but I would stick with .alt.)

Another alternative might be to have a separate version of the face with the “basic” ampersand changed to the Jamesian ampersand, and use that font for all James quotations.
