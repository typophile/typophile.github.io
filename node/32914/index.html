What you could try as a test would be to use InDesign's single-line composing mode, pick the same values you use in Quark (70-100-150?), using the the same old Type 1 font, and see what happens. Most single-line composing programs work about the same, so if the two are making the same line ending decisions, you have a starting point. I came to ID from TeX, which uses a paragraph optimization routine which is to my mind superior to ID's paragraph composition mode. Could just be sour grapes on my part, I suppose. Anyway, I found that I had to go into the fonts & make the "ideal" wordspace smaller, usually right around 200/1000, then use 80% 100% 120% as justification settings. You could get the same effect by varying the ideal & min percentages, I suppose, but we're trying to set up fonts so our comps can just use 80-100-120 for text settings. To give you some idea of the difference with TeX, we use to use the equivalent of 65% 100% 140% to get roughly the same results -- of course, TeX would not exceed certain limits (you get an overfull box to resolve by hand); Neither Quark nor ID do that. I did look at your PDF. As you say, it is a mess & I can't imagine what values gave you so tight a setting. As to loose, hum. Anyway, try the single line composition mode.

2007-04-10 04:12:07 • pattyfab
Did you define basic parameters for Justification? To do this, close all documents in InDesign. Then go the Paragraph-pallette and choose Justification (via the triangle-button) or use the keyboard-combination Command-Option-Shift-J. Here you define values for Word & Letter Spacing and Glyph Scaling. Eg my standard values are: Word Spacing - Minimum 75% - Desired 87% - Maximum 100% Letter Spacing - Minimum 0% - Desired 0% - Maximum 0% Glyph Scaling - Minimum 99% - Desired 100% - Maximum 101% Auto Leading: 100% These values will be used for all documents you CREATE afterwards. The values above give the Composer some latitude when Justifying text. For unjustified text reasonably tight wordspacing is achieved. For some typefaces it is necessary to change these values, eg Gill Sans, which is way too loose. Hope this helps.
