> How do you render half a pixel?

1) In grayscale rendering you have fractional pixels.
2) In Windows the native OS resolution is 96 ppi, so some half-pixel sizes will have their own bitmaps. Specifically, 10 point rounds to 13 PPEM, 11 point to 15 PPEM, and 10.5 point to 14 PPEM. Since 96/72 is 4/3, one out of every three half-pixel sizes will have its own bitmap.

2003-10-08 00:06:08 • kakaze
Hrant - in what I assume was a slip, you wrote that PPEM is "points per em", which makes no sense because a 14 point font will always be 14 points per em, by definition. Of course ppem = pixels per em. TrueType font technicians use this pixel measurement all the time, and don't think in real-world units such as points or millimetres. When an application requests a particular point size (and implicitly also a zoom & resolution specification), the font returned is mapped to the closest ppem size.

If you want to examine the actual pixel-tuned ppem sizes in fonts, a nice trick in MS Word is to set the zoom to 75%. Since 72 is 75% of 96, the point size setting will now exactly equal the ppem setting. Set some lines in consecutive point sizes, and you'll see how the font behaves at all low resolutions. Use Ctrl-[ and Ctrl-] to whizz up & down sizes in selected text.
