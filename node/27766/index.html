Please explain what you mean by “support for superscript and subscript”? Very few fonts have sups or subs beyond the handful of common Unicode encoded ones you find in western codepages.

In most cases applications will scale and position the normal characters in the font to form the sups and subs. Some apps use values in the font as a guide, others use an algorithm.

Of course OpenType supports the sub and sup features so an OpenType font could include specifically drawn glyphs for every normal glyph, but I don’t think any cover more than a handful.

So finding a font manager or viewer to sort through this – showing true, scaled and OpenType sups and subs seems a bit, er specialized. You might be better off using an app like InDesign and some scripts to generate samples of all the fonts you have with the sups and subs you care about.
