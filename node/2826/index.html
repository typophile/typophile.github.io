Alan is correct, the fonts to which you refer do not contain Cyrillic smallcap glyphs. However, because they do contain an <smcp> feature, InDesign will not apply fake smallcaps. InDesign will only apply fake smallcaps if the font contains no OpenType feature for smallcaps, regardless of whether particular characters are convered in that feature.

There are relatively few Cyrillic fonts with proper smallcaps. I know of several in development, but only a handful from Paratype (non-OT, last I checked their catalogue) that are currently shipping. My take on this is that if a font contains any smallcaps it should contain appropriate smallcaps for all the characters supported, but obviously Adobe have decided to spend their resources in other ways.

2004-01-25 22:43:04 • vanisaac
Cyrillic miniscules are already functionally small caps anyway.

No, they are not functionally smallcaps: many of them are <i>visually</i> similar to small versions of the capital letters in upright style (not in cursive italics), but there is a typographic function for actual smallcaps in Cyrillic text just as there is in Latin text, e.g. acronyms, running heads, etc. The typical difference between Cyrillic smallcaps and Latin ones is that the former tend to be slightly taller, in order to distinguish them more clearly from the similar lowercase letters, whose proportions also differ. This is something to be kept in mind when designing fonts that combine support for multiple scripts.

A few years ago I was commissioned to design a set of Cyrillic smallcaps for Minion Cyrillic for a publisher who needed them for a book that included some words that always appeared in all-caps. See attached PDF.
