Please bring up your issues on the fontforge-devel mailing list. That way George will get to fixing your issues for the next package release which happens almost monthly (probably also issuing a patch if you want to build from source), or tell you what you’re doing wrong, or tell you that he can’t reproduce your problem and needs more information.

Are you running the most recent build of FontForge? Did you install libSpiro before FontForge? If not, you may have to re-run the FontForge installation again. It’s OK to do this with the installer the same version already installed.

There are also some [[http://fontforge.sourceforge.net/source-build.html#Dependencies|dependencies in FontForge]] provided by Fink (notably a library which makes Mac menu shortcuts work again, others to enable different image imports to your background layer, and more) which you’ll have to install if you haven’t already, as well as the Apple Developer Tools, so that Fink can properly update. This will occupy a big chunk of your hard disk (1-3 Gb), but on a new machine it’s not so huge percentage wise.

This is all something I had to do with the latest build too. Previous builds used a masking method to make the command key function as the control key for the purpose of menu shortcuts. This worked but I figure it wasn’t a clean programming style so it was changed. The new version, however, requires an extra library that’s not part of Apple’s X11 by default, but is provided by Fink. You also have to toggle the “Enable keyboard shortcuts under X11” item in the X11 preferences to Off.

I do notice that, even after all this, the command key does not activate keyboard shortcuts in dialog windows that don’t have menu bars, but the control key combos do work.
