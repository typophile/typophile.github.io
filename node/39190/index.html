Stylistic Alternate Characters

November 17, 2007 • Michael Jarboe

I’m curious of the perspective on building Stylistic Alternates into a font. I have a typeface that I would like to have an alternate ‘O’ that wouldn’t have a corresponding spot in my font glyph window in the way that ‘Oslash’, as an alternate does.

I’m new to this, but in this situation is the way to go about it to just create a new glyph and name it something unique that relates to my alternate character?, since, in this situation this is a completely original character that I want to be accessed as an open type feature just to change up the style of the font if desired.

I have other fonts that I would like to do this with in the future containing multiple sylistic characters?

I’m curious how this is done since what I’m doing isn’t an alternate in terms of being small caps or swash characters just creatively a unique variation on one particular character and in the future multiple characters with unique variations.

Thanks for your input.

Mike

Just to to clarify, ‘Oslash’ is a unique character, just like “Oacute” etc.

The unique name should be “O.something” where “something” is whatever you want.

I’d recommend using the “stylistic sets” features such as ‘ss01’, because you’d probably want the same effect to work on accented O’s as well, and it’s nice to be able to apply something like this on a whole block of text. So you’d also have “Odieresis.something” and so on.

You don’t necessarily need to encode the alternate glyphs at all, or if you do encode them, use PUA (Private Use Area) codepoints, as discussed in another current thread.
