2012-04-16 16:27:41 • Theunis de Jong
TdJ - Fontlab doesn't know whether you're going to generate a TT or T1 font; it will set the contour directions as you instruct it, and will render the glyphs properly regardless. So if you draw a circle, it will appear solid regardless of its direction. If you draw two circles and drag so they overlap (like a Venn diagram) the overlap will be black if the two circles are in the same direction and white if they're opposite. Doesn't matter if they're defined as PS or TT direction (Fontlab's terminology).

2012-04-16 17:54:26 • hrant
Fontlab doesn't know whether you're going to generate a TT or T1 font; it will set the contour directions as you instruct it, and will render the glyphs properly regardless. Note that you need to be careful with FontLab's auto PS/TT path direction tools if you have made a glyph in which a bounded white space is intersected by a line and you have not yet merged the paths. So, for example, if you have made a Cyrillic Ө and the bar is not yet merged with the O shape the auto path direction tools will tend to get confused and you end up with this sort of thing: [img:sites/default/files/old-images/BarredOops_6393.jpg] ______ As to why the PS and TT formats use opposite path direction rules, I reckon when Apple developed TT they simply set out to do as many things differently from Adobe as possible. Hence quadratic rather than cubic beziers, and hence the different path directions. I don't know if actual lawyers were involved, or if Sampo and his colleagues simply decided to play it safe.
