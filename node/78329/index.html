The OP is working in CS3 …

I’ve tried using several Arabic fonts in CS3 and am having trouble getting the characters to connect properly (they don’t connect at all).

Correct. By default, when you type a single character, InDesign picks the “Isolated” form, whereas software designed to work with Arabic automatically use the “Initial”, “Medial”, and “Final” character forms. If you want to do this by hand: it’s not very complicated, but it’s a lot of work: select each character, one by one, and look in the Glyphs panel for its alternates. Then pick the appropriate one, by hand. Take care in selecting the correct ones: the right hand side is – in Arabic Right-to-Left writing – the “Initial”; all inbetweenies are “Medials”, and the left hand side is the “Final”.

Note that not all characters come in all 4 shapes; for example, Times New Roman only has Isolated and Final forms for the Alef. I think (but I could be dearly mistaken) that the Final form should only be used as such, so in this case you should use the only alternative everywhere else.

I hope you have a good example to work from … It might be faster and more accurate to do this in Word, print to PDF, and insert as image.

I’d not recommend this for any serious work, proper Arabic shaping has more than piking initial/medial/final glyphs (and Arabic characters have infinite number or forms not only four, the only limit is what subset the type designer choose to put in his font), not to mention ligatures (for fonts that have them) or mark positioning.
