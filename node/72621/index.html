Question on the different Font Formats

July 26, 2010 • Queneau

Hi all,

I have a question. Over a time I’ve built up a reasonable library of typefaces, from different foundries and sellers etc. I’ve basically have fonts in all MAC osX compatible formats (PS type 1, Mac TrueType, PC Truetype, OpenType (both OTF and TTF). Some fonts came in several formats. Can anyone tell me about the benifits and problems of the different formats, when it is best to work with which formats, etc.? I guess Opentype is the preferred choice nowadays (because of compatibility, character sets and extra goodies), but truetype and postscript are still widely used and a lot of type is still sold as PS or TTF.

If for instance I would like to use a typeface for a print project, and I have the choice between TTF and Postscript, which would you prefer and why?

If your font files may be rather old, beware.

“Old” Macintosh TTF files are not usable under Windows (modern ones should be, and all Windows TTFs are usable on Mac OS X).

Mac Type 1 fonts contain the same data, but have a different encoding than Windows Type 1 fonts.

Additionally, older font files make heavy use of pre-OSX’s “Resource Forks”, which makes them all but invisible to Windows systems.

The OTF font file format has been designed to overcome all of these problems, and most (if not all) ‘modern’ TTFs are, in fact, fully qualified OTFs as well.

(That said: I think you need some rather advanced software to tell Good from Bad fonts on your hard disk. And don’t use Ugly fonts at all.)
