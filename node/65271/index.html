2010-01-01 00:56:00 • John Hudson

There isn’t a lot of pixelfont action these days, but when they were “in” the accepted way to make them render out predictably was to shoot for 8 point (at 72 dpi) no matter how many pixels high the font actually was. For example my three sizes of Mana* all need to be set at 8 to come out right (you just need to determine the leading accordingly). Yes, it’s a hack, but it makes -or at least made- things as controllable as possible.

  • http://www.themicrofoundry.com/manademo/

2010-01-01 14:13:18 • nina

Yes.

Of course if your font is very large (which rarely makes sense though) you get in trouble, so you might use 20. Nothing in between works well* because Windows uses a nominal dpi of 96, so you want 75% (72/96) of the number to result in an integer. The problem with anything besides 8 is that the user needs to be aware of what size to use for it to click.

  • Remember, it has to divide 1000 evenly.
