Font Management Software (by the users)

March 2, 2007 • AndrewSipe

If you were building the ultimate font management software what features would you add? My suggestions will be based on Extensis Suitcase for Windows, the "Missing Features" and design changes: •I love the simplistic approach Suitcase takes, but it's missing features that seem apparent to me? Such as, Drag and Drop file management, in the key of iTunes. It creates a data base/file management system without me having to keep up with it. •To be able to maintain rich data about the fonts I use, including Foundry, Style and Family and any other information that I'd find useful for finding and using my fonts. I could sort through on demand by columns such as name, styles, weights, foundry, designer, category or whatever. Such as sorting all the sans serif fonts by simply clicking a tab/column that would make life so much easier. •A quick "Mix" feature using keywords or drag and drop to create "Sets". Just like iTunes. •A character counter for quick viewing and a detailed pop-out character map. Since the advent of OpenType, Character maps can get up into the 1000s for a single font, it would be nice to see all of those characters before activating a font. •Font Management should appear easy, I feel as if I do a lot of "maintenance" outside of the software and that should be the softwares job. I'd like to hear what some of your thoughts are for features you'd like to see.
A lot of those things are addressed in the fantastic, and free, Font Explorer from Linotype. And they're working on the Windows version... ---

2007-03-03 06:39:34 • Sharon Van Lieu
I'm really suprised that no one else wants control of OpenType layout. I would like to be able to control which OTF features where being used when viewing a specimen. Sophisticated font version control - Session based activation of specific font versions on a per application or even per document basis - for when you absolutely need Comic Sans MS version 2.01 and not 2.03. sii, as far as I know Suitcase is the only font manager that really addresses this issue. The new Suitcase for Windows implements the same Font Sense technology from Suitcase Fusion. When a font is added to the database, a unique identifier is created based on several values within the font. It's similar to a checksum. During auto-activation, this can be used to distinguish between different fonts of the same name. The other advantage of Font Sense is that it could be used to match up metadata. I wish Suitcase had a good way to import/export metadata. I think I want everything filip blazek lists. Lots of metadata-- the ability to sort and group by metadata--and the ability to share it. A related feature that I consider critical is the ability to back up data. There's no point creating a bunch of sets and adding metadata if I don't have the ability to back up this effort or migrate it to a new system. I probably replace my laptop every 18 months. I back it up once a month. If my current hd dies, I know that I can restore my setup. On Suitcase for Windows, this can now be accomplished by restoring the Vault to the same relative location. However, I consider it a necessary feature for any font manager. In the professional publishing world, some of these features are accomodated by client/server font management solutions like Font Reserve Server.
