Fontlab Key Glyph

March 7, 2009 • Michael Jarboe

In Fontlab, if a font is complete regarding all kerning classes and kerning pairs and the information is compiled into the OpenType kern feature… and you happen to copy and paste a non key glyph over a glyph that is marked with the red ‘key glyph’ mark… does anything actually change?

I imagine nothing would change being that all the kerning and class information is already compiled but I’m still curious about this. I’d hate to ‘update kern feature’ and re-compile and have things change… It doesn’t look like there’s a quick way to re-tag a key glyph after it changes either… any thoughts?

Thanks,

Mike

Mike,

in FontLab Studio 5, to make sure nothing changes, you should have a “kern” feature generated in the OpenType panel, and also all the classes imported into the lower-right part of the OpenType panel (using Import FontLab Classes from the OpenType panel flyout menu). Only then you can dump or freely rearrange your classes in the Classes panel.

Of course once you’ve imported the classes into the OpenType panel (i.e. converted them from the FontLab syntax into AFDKO syntax), in Preferences / General Options / Unicode and OpenType, disable Add all glyph classes to OpenType feature definition code, and in Preferences / Generating OpenType & TrueType / Kerning, disable Generate OpenType kern feature if it is undefined or outdated.
