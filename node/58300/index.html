It’s possible to have a font with a “baseline” as part of the design, but there would be no way to make the baseline a different color than the letters. A character in a font can only have one color applied to it, and it must be for the whole character, not just part of it.

None of this matters because you can get the same effect on a web page using CSS. Make a class something like this:

.baseline { border-bottom: solid 2px #e9f2f9; }

Then you apply this class to the text you want the baseline to appear under.

It has nothing to do with @font-face. This is an old CSS technique that will work with any text (no matter what font) that can be displayed in a reasonably modern browser (meaning any browser released since the late nineties). The border property works on all kinds of html elements. The rule in my example could just as easily be applied to anything, not just text.

2009-05-24 04:32:02 • Mark Simonson

There are actually two problems with using border-bottom in this case. First is that the descenders might get cut off on some of the less compliant browsers and second is a lack of support for multiple lines of text. If you’re looking to use multiple lines of text, you could carefully construct an image to use as a repeating background on a body of text. That way, it will appear where you want it, be a different color than the text and repeat properly. It should also be possible to apply the same methodology to a sIFR implementation (though probably a bit harder).

Of course, you’re also trying to use a unique font on the web, so if you are only looking at a small number of these elements, then maybe just use photoshop to construct the text the way you want it to look with the baseline you want and place it on the web as an image.

@Cameron Basically, the issue is that by default PC & Mac (I’m ignoring linux for now) ship with a certain set of fonts already installed. While for the most part these fonts different from each other, there are some that overlap. Since webpage rendering is done on user’s personal computers and any fonts requested by the website are pulled from a user’s personal database, it is important to know which fonts will be accessible no matter who visits the site. So, these overlapping fonts are considered “web safe” and may be used without concern of someone not having that font.

Here is a list of web safe fonts

Oh, and the reason I left out linux is because they ship with a totally different set of fonts and need to download, but I assume if someone is using linux, they know enough to take care of that themselves.
