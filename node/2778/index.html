You can use FontLab to change the info in the font header - Family name, Style name, etc. In lieu of that, I believe you can open the AFM file in a text editor and change the info there.

I'm assuming that the Family name is something like 'Fontname-Regular'. If you want them to not be their own unique entries, then the Family name should be just 'Fontname' for each font, the Style name should be 'Regular', & the Font name should be 'Fontname Regular'. Same for the other members of the family (e.g., italic, bold, bold italic).

If you want a huge family (i.e., more than 4 members) to be contained in one family, you'll need to add OpenType names in FontLab. The family name for every font will be the same, and the Style names will all be different.

Yeah, it seems implausible to pay $549 to rename 3 fonts. I'm not a Mac user, but I'd try this. Open your AFM files in Simpletext. You'll see something like this at the top:

StartFontMetrics 2.0
FontName Paralucent-Italic
FullName Paralucent Italic
FamilyName Paralucent-Italic
Weight Regular

Change that info to this:

StartFontMetrics 2.0
FontName Paralucent-Italic
FullName Paralucent Italic
FamilyName Paralucent
Weight Regular

Notice the change in FamilyName. Maybe that'll sort you out.

Be sure you're not violating your EULA by modifying the information though.

Mac files have two 'forks'. A resource and a data fork IIRC. All of the font info resides in the resource fork, which a PC doesn't read (PC files have only one 'fork').

I have to jump back & forth between platforms sometimes, and on my PC I use a piece of software called TransMac. It'll take a Mac file & write a file that a PC understands. Pretty cheap to license, too.
