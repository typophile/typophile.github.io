Are you using the actual “Book” feature of InDesign?

It’s been a little while since I worked with a project set up with this feature. But, I’m pretty sure that if you have the .indb file set up properly, with all the individual chapters correctly entered, then adding pages to one chapter should flow through to affect the start of others automatically, whether odd or even. You can even see evidence of the changes in the Book palette via the pagination.

The individual chapters need to be set up as Facing Pages, obviously.

If you have the Book.indb file already set up and it’s not working, then look in the Pages palette commands to see that Allow Pages to Shuffle is checked on. You may need to check each individual chapter.

Eric, my impression is that António wants each chapter to start on the whatever page (odd or even) makes sequential sense after the previous chapter. But that he wants it to change if a previous chapter shifts.

So, if a chapter currently ends even (on a left page) the next chapter will start on the right (an odd page). But then if that chapter gets changed and flows onto another page and then ends odd (a right page), the following chapter will automatically shift to begin on the next even page (left). And so on throughout the document, shifting back and forth as needed.

This seems to be the perfect situation for a Book file.

I would say that for the situation I described, then you want to set Book > Book Page Numbering Options… to be “Page Order: Continue from previous document” and also turn on “Automatic Pagination”.

I’m not sure what’s not working for António. How did you set up your Book file? Did you generate all the chapter documents and then assemble them into a Book at the end? Or did you assemble them chapter by chapter as you created the documents?

You’re using Automatic Pagination, right? If you’re manually setting the start page for each chapter, then that could be messing up the Book synchronization. So, if your Chapter 1 ends on page 25, and you manually start the next chapter (document) on page 26, and then if the first chapter reflows and adds another page (pg. 26), your book will have two page 26s – one at the end of Chapter 1 and another at the beginning of Chapter 2.

Is this what’s happening?

António –

I think the problem is indeed that you have each chapter starting with unique page numbering 2.1, 2.2, etc.

As you know, a book typically has odd pages on the left and even on the right, and I think the Book feature assumes this. So if you force each chapter to restart numbering with page 1, then the Book will force it to start right each time.

Usually this kind of scheme (1.1, 1.2 … 2.1, 2.2, 2.3 …) will only open on the right, using blank pages if necessary to fill out each chapter to end left. I think I’ve only really seen this scheme used in manuals.

Given this style of page numbering, then I don’t think you can get InDesign to do what you want in terms of automatically shifting the chapter starts right and left.

– Kent.
