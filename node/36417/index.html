Bruno’s blog article makes a fair number of good points; he’s right that Ou’s original piece ignored/missed that OS X has sub-pixel anti-aliasing available, even if it’s off by default. I happen to strongly disagree with the general thrust of his argument that OS X’s full-fuzz anti-aliasing is a good thing, but without any actual studies on either side, that’s mostly a subjective question right now. I was sad to see pointless ad hominem attacks which detracted from the strengths of the post, however.

But the post also appears to have some highly dubious assertions. One of them: “The fact of the matter is that Windows implements font rasterization pretty much the way it always has.” WTF?

Si, isn’t ClearType on by default in Vista, even for GDI apps? Even in XP, ClearType can be turned on - and it certainly isn’t “pretty much the way it always has.” Plus, Bruno must not remember the Windows 3.1 days when there was no anti-aliasing, as best as I can recall.

Anybody know when and with which version Windows added anti-aliasing? Font Bureau claims it was in 1999 in this article: http://type101.fontbureau.com/archives/19. In any case, I’m pretty sure they didn’t have system-level anti-aliasing at all back in Windows 3.1. My point is just that Windows and Mac OS have made significant rendering changes several times over the years.

Another strange statement from Bruno: “Your browser is likely using a different font rendering engine than your operating system as well.”

The overwhelming majority of Mac users are using Safari, which Bruno admits is using Mac OS’s Quartz. The blog says it’s aimed at Mac users. Something like 75-80% of Windows users are using IE, which is using Windows GDI rendering. Who exactly is Bruno talking to?
