Long s code

July 7, 2010 • Nick Shinn

Although the “hist” feature is not presently supported by any layout applications, I am putting it in all my new fonts, along with longs. sub s by longs; sub longs' f by s; sub longs longs' by s; sub f longs' by s; ignore sub longs' @letter; sub longs' by s;

This satisfies all the circumstances listed by Andrew West, except the Genesis abbreviation, of course.

I haven’t developed codes for other languages.

It would be cool to be able to apply this feature to e-books, especially those written in the longs days.

nb. “@letter” is a class comprising all letters other than “s”. The “ignore” command is a clever way to handle substitutions at the end of words.

Although the “hist” feature is not presently supported by any layout applications

…except all those that use Mac OS X’s native text engines with access to the system Fonts palette, such as Apple Pages or Pagehand, as well as TeXWorks/XeTeX and some others.

Nick, InDesign supports ‘hist’ feature, although it is hidden. Choose Edit > Keyboard Shortcuts, then select the list of keys for the panels. There is a list of shortcuts named Characters:OpenType. And there is a possibility to add a keyboard shortcut for historical variants of glyphs. You can’t access this feature through menu, only via keyboard. An easier way to access the feature is to use free plugin Keyboard Shortcuts by DTP Tools (http://www.dtptools.com/product.asp?id=ksin) - you can type the word “hist” to the search window to quickly find the correct function.

2010-07-09 13:05:00 • Jens Kutilek
Juan Pascoe, a great printer and researcher settled in Mexico, has just told me via e-mail that, in his opinion, in Spanish some complicated uses had to do with the lack of some characters: for example, he says that the ſb ligature was not common, so if you had to write about Liſboa (Lisbon) the space between these two characters was too wide, so you used a short s instead. The same goes for something like “ilustriſſimo”: the ſſi ligature was pretty rare, so you can only use the best available combination: “ilustriſsimo”.

2010-07-09 20:11:29 • Nick Shinn

Nick,

Typographic aesthetics over-ruling orthography.

this has been happening quite often. My favourite example is the Polish ą and ę. First, there was the Latin “e caudata”, which was used as an alternative graphic form of the Latin vowel “æ”:

http://www.flickr.com/photos/adamt/3630379481/

As it was present in typefaces that were imported to Poland, typesetters adopted the glyph to be used as the graphic representation of the Polish “nasal e”. The other nasalized vowel (“nasal a”) was represented as an “a” with a thin, simple stroke — which did not at all look like the “ogonek” in “e caudata”. Look at the “ą” and “ę” in: http://www.flickr.com/photos/adamt/3630378529/ http://www.flickr.com/photos/adamt/3630381453/

Only some time later, the typographic æsthetics “ruled” that those two forms should be “made alike”, and the more sophisticated, mature one (i.e. “e caudata”) has been chosen as the source.

Adam
