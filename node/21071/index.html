2006-07-02 00:22:11 • Miss Tiffany

I don’t use any Font Managers. This is because:

  1. My job does not require me to have many fonts installed;
  2. I work with Mac OS X, and — unlike OS 9 — it allows me to keep the fonts inside of folders, which I can manage at my discretion;
  3. This way I know and have full control of which fonts are installed and where — made available to all the applications or just some;
  4. I’m a font developer and there are times when I need to generate and test tenths of fonts a day;
  5. It’s easier to troubleshoot font problems because there’s less font-cache issues to worry about.

Having said that, I find that a big part of the problems that people run into regarding fonts, are related with cache issues. Therefore, it would be interesting if Font Managers could supply documentation on this matter, and provide (to their users and people in general) easy and detailed methods on how to solve this problem. (I’m aware that Linotype FontExplorer X already does this to some extent.)

A big problem with testing fonts under development is that Mac OS X caches actual font data. When you activate an updated version of a font, either with a font manager or by placing it in a system font folder, you can’t be certain that Mac OS has updated it’s cache correctly. The only sure way is to clear the system caches, which requires a restart. I actually signed up for the font testing workshop at Typecon. I’m really excited to hear what professional font testers have to say.

I think the font manager is a really useful piece of software for typesetters who use a lot of fonts. People who work in ad production for print or web may need to use a wide variety of typefaces because customers demand that their ad have a unique look and feel. Just to find a typeface in a certain class may involve looking at dozens of typefaces side by side. I suppose you could use the web based search tools, but many designers would prefer to restrict their previews to fonts they currently own and have readily available. Secondly if you have to go back and open old documents, its a real timesaver if the fonts simply turn on automatically.

Some designers are fortunate enough to be able to use the same few fonts. Others aren’t so lucky, usually because they are working in a market where customers demand novelty.

2006-07-03 17:11:39 • Paul Cutler

Two points:

The simple way to deal with Apple’s “flexibility” regarding font folders is to pick one (either the one in your home folder or the one at the root of the start up drive) and stick with it. Don’t install fonts anywhere else. Ever.

The simple way for font developers to avoid font cache problems is to always rename your font (Myfont A, Myfont B, Myfont C, etc.) when you generate a new version. It takes a little extra effort, but it has other benefits, including being able to have multiple versions of your font open at the same time for comparison, and it’s much, much less hassle than cleaning the cache.
