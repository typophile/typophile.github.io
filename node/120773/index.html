2014-10-23 23:54:07 • Martin Silvertant
It seems unnecessarily cautious to use backslashes routinely. Especially given that there are no standard glyph names that are also OpenType feature keywords. It is handy in the unlikely case that you need to name a glyph something like "sub" or "lookup", but if you do happen to use a keyword as a glyph name, the compiler will complain, letting you know to use a backslash (or a different name). But, no need to do it all the time preemptively, unless you want to.

It's a classic catch-22. The app makers (e.g., Adobe) need to support this stuff better, but they don't because they say users aren't asking for it. Given this situation, workarounds are understandable, but we would all be better off if the basic problem were resolved. There's a petition effort going on right now to (hopefully) address this.
