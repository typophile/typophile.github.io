>Fontlab tells me the number is 217 for "eth" >so i tried ALT + 217 but all i get is the numbers 217 ALT codes only work under Windows (and perhaps some *nix systems). Fontographer maps these characters to certain control keys: Eth=Ctrl-a; eth=Ctrl-b; Thorn=Ctrl-k; thorn=Ctrl=l. The problem is that these keystrokes map to other keyboard controls like backspace and page scroll. You can use the Insert Glyphs palette in InDesign to plop these characters into your document without resorting to importing .eps files. What if you used Fontlab to remap those characters to some other glyph position occupied by other, seldom used characters?

The other way to get "hidden" characters in OS X is to activate the Character Palette. Go to System Prefs, International. Click on the Input Menu tab and select Character Palette. You'll get a small menu after "Help" that lets you select and insert any character you want. The default is Lucida Grande, but if you click the triangle at the bottom you see the glyph in other fonts.
