2007-06-06 07:38:55 • clauses

The kerning was imported directly from another font that didn’t have that error.

So the source font seems to have contained such a pair (you could search the .afm for it). Thing is, if you import kerning for one master, FLS will automatically create a kerning pair for the other masters too,* and if no values have been defined for the other masters yet, these will be set to zero. (At the same time this means, if you generate instances of this font later, you should remove all zero values since they will unnecessarily increase the size of the font. Or at least check the kern feature if FLS has removed them automatically – I am not sure if it does.)

  • More exact, technically: There is only one kerning pair for a given glyph pair, and in case of a MM-font, the kerning value for this pair is a list of kerning values, one value per each master.
