Whyawhelk: eh? :) (Is it possible you are confusing fonts with zipped files?)

The font file description even says it’s a Windows True Type Font.

Where does it say so? Some file descriptions are generic, i.e., based on the file name but not on the file contents. Perhaps this is not a Windows Truetype font but an older version of Truetype, which can only be used on a Mac. The file size of 0 Kb seems to point to that – the actual data resides in the Mac specific “resource fork”, a portion of the file that Windows has no access to. Compression and mail software is rigged to honour that, even when opening on a system that does not use resource forks, which boils down to “the entire world minus Mac OS”.

If that’s the case, you cannot use the original font file on your Windows machine and you have to convert it from a Macintosh Truetype format to Windows Truetype format. If the EULA of this font allows it, Fontforge might be able to convert it.

[Edit]

On further reflection: you could try and see what happens when you copy the Resource Fork part into the ‘regular’ part of the file. For that, you need a binary editor such as 0xED, and some nerve. Still no guarantees, though.
