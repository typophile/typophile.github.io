Opentype Kern only with or without expanded kerning, what is your opinion

September 15, 2011 • dimitre

I like to mantain only opentype kern feature and not use expanded kerning. I know this creates a limitation but I tend to think things are evolving, I don’t want to build fonts compatible with Win98 or something.

In other hand I know some famous software still lacks Opentype kern support, more notably Myfonts site and OS X Font Book (already filled a feedback form here http://www.apple.com/feedback/macosx.html)

What are your thoughts about this subject?

I build my kern tables in MetricsMachine, export a feature file, and compile it into my fonts. Non-Opentype kerning does not exist within my workflow. I guess if big publisher ever wanted a 1,000-seat license with type 1 fonts I would be screwed, but otherwise it saves me from a lot of headaches.
