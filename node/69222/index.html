FontLab vs. Fontographer VIDEO

April 8, 2010 • TypeSETit

I've made a couple of quick videos to show the difference in editing between Fontographer and FontLab. FontLab has so much more power than Fontographer, so I'm not complaining overall. But I AM frustrated by some of the actual drawing/editing behaviors. [[http://www.typesetit.com/video/fog-demo.mov|In this first video]], I show how I use Fontographer to connect two letters to make a ligature. In this case, I used the "r" to the letter "k." I began by removing the overlap of two paths, then using the merge tool to get rid of unwanted nodes. With Fontographer, the interpretation of that merge is so much more intuitive with regard to what I want it to do. The amount of editing is limited. [[http://www.typesetit.com/video/fl-demo.mov|In this second video,]], I perform the identical task of making the same ligature. But what happens in FontLab with regard to the merge (in FontLab, the delete button does the same task as Merge in Fontographer) is a bit frustrating. Its interpretation of what I want it to do is different and requires more editing on my part. With this simple task demonstrated in the two videos, I'm able to perform more than three times faster in Fontographer than in FontLab. QUESTION: Is it that I am just very accustomed to how Fontographer works, or is there something I could do differently in FONTLAB that would make this process easier (and, more like how Fontographer performs)? Or is my interpretation that Fontographer is more intuitive a correct one? P.S. I hope the videos explain what I mean... I didn't want to have to try explaining by typing all of the steps out. :)
In fact, over the years I often found myself using Fontographer to do work that most people would do in Adobe Illustrator. I simply liked Fontographer's drawing methods better than Illustrator. It seems to be so much more intuitive. Perhaps you find Fontographer's tools more inutitive because you've been using them for so long. I find FOG's scaling and rotation tools utterly incomprehensible but, then again, I've been using CorelDraw since 1990, so maybe it's just me...

2010-04-11 18:15:40 • Tim Ahrens
Removing nodes in FLS is a lot better than it used to be in previous versions of FL, in which removing even a single node would often result in major outline distortions. In FLS there are still plenty of annoyances, though: Removing nodes individually usually works differently (and better) than removing a bunch of nodes all at once. This means one needs to carefully select and delete nodes in sequence, trying to guess from experience what order will result in the least distortion and extra editing. Removing a start point results in a straight line, even if the start point had a curve on both sides. This means one needs to redefine the start point to somewhere else on the outline before removing nodes.
2010-04-11 19:41:10 • John Hudson
Removing a start point results in a straight line, even if the start point had a curve on both sides. As someone pointed out above, this problem can be avoided by using the Erase Tool instead of the delete key. You can also left-right-click the node, which retains the curve even if it is a start node.
