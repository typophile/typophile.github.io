2009-05-26 14:54:50 • gontijo

Are you sure you are using the term “antialiased”, and the reverse, “make it looks not anti-aliased”, correctly? “Aliased” is jaggy, with hard edges. “Antialiased” is with constructed soft edges.

Using a pixel font, the text will (nearly) always be aliased. The “nearly” qualifier is because Acrobat also is able to antialias monochrome bitmaps of sufficiently high resolution (one can guess that to be > 1.5 x physical screen dpi).

If you have a real pixel (bitmap) font that you can use in InDesign, that will be passed on to the PDF unchanged; i.e., as pixels.

However, I’m pretty sure you cannot use real bitmap fonts in InDesign–it’s way too smart to allow that. Perhaps you have a simulated pixel font instead ([[http://www.zone38.net/font/|Press Start]] comes first to mind). Those fonts are vector fonts, just like anything else, and just like other vector fonts it will be antialiased for screen in Acrobat.

If you really really want a pixel font, create the text as image in Photoshop and place that into InDesign.

Alternatively, you can do it the other way around: read your InDesign created PDF into Photoshop and rasterize; then save that as PDF again.
