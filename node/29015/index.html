Hello. Contextually, you cannot substitute two glyphs at once. So, instead of

sub f' f' @prob by f.salt f.salt2;

you need to do it in two steps:

sub f' f @prob by f.salt; sub f.salt f' @prob by f.salt2;

Note that in the context defined in the second line, the first f already has been substituted by f.salt!

You can read Adam’s note as: You can use contextual substitutions in liga, but keep in mind that then some applications may ignore either the lookup containing contextual substitution, or at worst, the entire liga feature. (I think the second is true for Apple’s applications.)

If I look at your examples, it seems more appropriate to put them into calt which is on by default in some applications, just as liga.

OpenType spec, actually, allows this kind of substitution. It is current OpenType layout compilers/editors that don’t support this.

You should use lower level tools like ttoasm or TTX to do that, but at the price of turning whole project maintenance into nightmare. It is practically like programming all lookups in assembler.
