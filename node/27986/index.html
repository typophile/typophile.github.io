They can unencoded, so long as there is a way to access them through OTL features. Since you rely on applications to expose features to users, you should try to find appropriate features among the exist OTL features as documented here:

http://www.microsoft.com/typography/otspec/featuretags.htm

Old style figures, of course, should be mapped in the ‘onum’ feature.

Fancy caps perhaps in the ‘titl’ feature?

I’m not sure what you mean by ‘monostroke lowercase’, so can’t make a guess as to what feature would be appropriate.

It is possible to define custom features in VOLT, but that won’t do you much good unless you’re also creating your own applications or plugins that will recognise these features.

If you think you have an idea for a feature that other font developers might find useful, I recommend writing it up, following the format of the published features, and submitting it to the OpenType discussion list for review and possible inclusion in future versions of the standard.

I’ve OT-mapped the text figures to ‘onum’. they are unmapped in Unicode.
