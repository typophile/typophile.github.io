Fontlab crashes: "multiple glyphs mapped to code"

September 5, 2009 • pierre84

Hi, Maybe someone can help!

My Fontlab Studio 5.0.4. keeps crashing. I did quite a lot of copying and pasting glyphs with a handdrawn font. Now I’m ready to generate the font, but it freezes on 98% and says: ‘multiple glyphs oslash _191 mapped to code’.

I’n thinking of applying the command Glyph > Glyph names > Generate names or ‘Generating unicode’ but I’m not sure if these commands mix everything up.

Anybody knows the best solution?

Pierre

Unless you have custom named glyphs it’s probably okay to wipe out the Unicode table and start anew. Save a backup though. As long as the glyphs are named correctly, you’ll be okay. If you have glyphs with custom names you may not be able to do this.

Generate Unicode standard table DON’T try to keep unicode indexes

Sort glyphs by Unicode when you’re done. Switch to Names mode and make sure the names correspond with the proper glyphs.

. . . or this could be your problem:

I’ve come across a rare case where I get an error message that multiple glyphs are mapped to the same code but it’s not indicated by FontLab in names mode or codepage mode. If you check the supposedly offending glyph, the Unicode is set correctly, the name is set correctly, everything seems fine. If you look around for a duplicate glyph, it can’t be found. Mark the glyph with a color then switch to index mode. Now you can find the duplicate glyph. You’ll find two identical glyphs wityh the same name and Unicode but two unique index positions. It’s only happed to me around five or six times.
