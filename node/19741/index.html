Without OpenType support you only get the basic latin character set, which includes everything you would normally expect to find in a font–caps, lowercase, numbers, punctuation, etc.–, but not the extra characters–alternates, swashes, Central European, etc.
2006-05-24 19:57:54 • Mark Simonson

I’d like to point out a terminologic unclarity: “OpenType support” means different things. I usually say “OpenType Layout support” when I refer to the ability to handle the advanced typographic features. Freehand “supports” OpenType as a font format to the extent that it does support other font formats: the Roman codepage (256 character) and no advanced typography support. On the other hand, Adobe CS applications offer Unicode and OpenType Layout support.

A.

It is also worth noting that Kern Pairs may not necessarily be supported. We certainly had a problem with a TTF flavoured font that had both Pairs and Kern Feature tables. Neither Freehand 10 nor MX saw the pairs. Killing the Kern Feature resolved the problem on the TTF at least. I seem to remember that kern feature table fonts give no access to older apps. I am not sure now, if you can actually build OTF (CFF) based fonts using Kern Pair tables or if they only come as layout feature.
