2007-02-22 03:44:46 • Miguel Sousa

It would be easier to make a separate Tall Caps font. That way all you would have to do is substitute the gylphs in the normal cap spots, and you’d get features like c2sc, smcp and case without a lot of messing around with classes and coding.

And you would be able to recognize and specify the different versions at the top level of the app, without having to dive down into the OT menu.

That is the benefit of the feature being named in plain English, as part of the font name. Stylistic sets are problematic because they are an uninformative code number: a descriptive name field which shows in the application menu would be helpful, but it hasn’t been implemented yet. So the typographer has to play “spot the difference” or, heaven forbid, consult a reference document, to figure out what the feature does.

2007-02-22 07:46:26 • frau_jenson

I’d name the capital glyphs “A.titl”, “B.titl” etc., and would associate them with three features: “titl”, “ss01” and “salt”:

@UC = [A B C …]; @UC_TITL = [A.titl B.titl C.titl …];

feature titl { lookup titl1 { sub @UC by @UC_TITL; } titl1; } titl;

feature ss01 { lookup titl1; } ss01;

feature salt { lookup titl1; } salt;

Regards, Adam

Nick, it all depends on what you are doing. As a font user rather than a font maker, I really prefer all glyphs in one font. Suppose, for example, that I need to go back to a font and change kerning between two lower-case pairs. Or add a character. Or change a character. Or. . . With one font, you make the change(s) and are done with it. With split fonts, you have to remember to make it in both. That is the primary reason I made up database fonts when we were using the Type 1 format. More work starting out, but a lot less work in font maintenance.

I can live without case. As for cpsp, most book designers specify the amount of letterspacing for caps & small caps, and it is hardly every the amount you build in with cpsp. c2sc is nice, but there is 10+ years of authors & editors putting (eventual) small caps in the lower case due to the Type-1 font structures. Yes, it makes the downstream document harder to prepare, but you know history.

I’ll take a look at titl, though.

The cpsp code reminds me of a funny story. Sumner Stone, trying to save the ignorant from themselves, built in a rather generous right sidebearing with the small caps in a font (Type 1). The small caps set by themselves did look pretty good. But when you had Cap-small caps, things went to hell in a hurry – the letter following a cap was tighter to it than to the following letter. Punctuation also was a problem – which would include abbreviations that used the period (e.g., B.C.).

Moral: you can’t save the ignorant from themselves, and for the moment anyway, there is still a bit of use for compositors.
