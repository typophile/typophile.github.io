2010-11-01 12:10:03 • Theunis de Jong

If you need to create a Windows PostScript font, you can just use a different codepage for the font. This will give you the slots that you need for Polish (and a number of other Eastern European languages). You can switch between different codepages or different encoding files in FontLab.

What is the typeface that your newspaper uses? You might be able to license a Central European version of the font in Windows PostScript. (At Linotype, for instance, we still have thousands of fonts in this format, but only for fonts made before about 2004).

2010-11-02 00:40:18 • JoergGustafs

Chris,

  1. In FontLab Studio’s Font Window, switch to Names mode, and from the encoding dropdown list, choose “Type 1 World Microsoft” / “MS Windows 1250 Central European”.
  2. In Font Info / Encoding and Unicode / Microsoft Character Set, you may need to choose “Central (Eastern) European”. Generate the font as Type 1 – that would be the standards-compliant way (the font will have a custom encoding, marked as “CE).
  3. If your system does not recognize non-Western Type 1 fonts (which is possible), choose “Bitstream font set” in Font Info / Encoding and Unicode / Microsoft Character Set instead (this way, the font will have a custom encoding but marked as “Western”).

(Damn, I already started thinking about removing the non-Unicode font generation options, but it seems they’re still being used by some folks. :) ).
