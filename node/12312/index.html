When creating a type family I start with the 2 extreme weights - Thin and Extra Bold.

I then carefully space and kern these 2 weights keeping them consistent. Once I am happy with the 2 weights I blend percentages between them - for example 20%, 40%, 60% and 80% to create a six weight family. Doing it this way ensures correct spacing and kerning throughout the family. I don’t use the same kerning for both weights.

This worked fine in Fontographer.

I tried this in Fontlab - it was OK blending some of the characters, but others could not be blended even though the outlines (or contours) are totally compatible with similar numbers and types of points (or nodes), path directions and starting points. In the resulting font extra nodes were inserted even when the ‘Do not interpolate compatible outlines’ box is checked. This apparently shouldn’t be the case.

Also - and this is very bad - there is no kerning in the blended font. Where is the kerning? it also should be a percentage blend between the kerning of the Thin and Extra Bold originals.

Any advice?

Thanks

A better way to do it is to set up your two extremes as a one-axis multiple master font, and then generate instances (between 0 and 1000) to get the final fonts. Kerning is correctly blended when you do it this way.

I wouldn’t trust interpolated kerning pairs, just me though. Better, you might want to give the Transform>Metrics>Adjust Metrics a try. I usually dump kerning from another weight into a new weight and adjust in relation to the change in spacing metrics. Of course, you’ll want to proof the results of this as well.

As for just a straight inerpolation, why not use FOG for interps if you don’t like the results in FL? I’ve been preparing a font with round tips recently and I’ve found the FL interps with this data to be garbage. So back to FOG for some stuff.
