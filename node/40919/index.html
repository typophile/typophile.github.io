If you generate PS OpenType fonts then "Use PostScript FontName as FullName on Windows" is a must to make sure that your fonts will not conform to the specs. (If you generate TT OpenType fonts, then this is not required.) [a] I've followed the naming conventions suggested by Adam Twardoch except I have abandoned the short name usage (Instead replacing Reg with Regular, Lig with Light etc). [b] The reason I ask this question is that ... I am not sure if I understand. [a] What do you assign as 'Family Name' and 'Style Name', and as 'OT Family Name' and 'OT Style Name'? You have read this in Adam Twardoch's document but to emphasize it: For a family to work in Windows it is important that a family as defined by the 'Family Name' does not have more, or other, styles (i.e. 'Style Name') than Regular, Italic, Bold, Bold Italic. [b] Windows applications usually show the 'Family Name' only, and styles are accessed by Bold/Italic buttons. I cannot remember of any Windows application showing 'PS Font Name' or 'Full Name' in the font menu, so in this respect, it does not matter how 'PS Font Name' and 'Full Name' look like. (It is different with Adobe applications. These show 'OT Family Name' and 'OT Style Name' in the font menu, and only if not present, show 'Family Name' and 'Style Name'.) What is important for cross-platform compatibility of Word documents is that in the regular style, 'Font Name' is identical with 'Mac Name', or if you don't provide the latter, 'Font Name' is identical with 'Full Name' (see here).
