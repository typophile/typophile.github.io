Are you referring to users having the ability to move fonts around with their user accounts? Because that's something people have been doing on networked computer systems for decades. Right now I could stick my OS X user account on a network share and be free to use my fonts at every single machine capable out mounting said share. Plenty of Windows users do it in large enterprises. Any font designer who tries to fight networked home directories will be on the wrong side in a battle he's guaranteed to lose, because it's going to get his designs locked out of most Windows and Unix networks.

Hi Stuart! To be honest, I don't know exactly what this is referring to. But I could imagine that a user's 'setup' and files could be mirrored securely on-line in some way and a 'session' to that virtual machine could be established in much the same way as you might TS (terminal server) into your work computer from home. Traditionally when you TS into a machine I don't think there's any font specific licensing issues if the host machine is locked while the client machine accesses it. Where things become more tricky, with respect to licensing, is when you have multiple concurrent sessions connecting to the same machine - but that applies to apps, and other things as well as fonts.
