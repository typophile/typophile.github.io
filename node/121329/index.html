On OS X 10.9, only Kaiti SC Black has fsType set to “restricted embedding”; the other Kaiti fonts are set to “editable embedding”

For the meaning of 0x0008 and 0x0002, cf http://www.microsoft.com/typography/otspec/os2.htm#fst The embedding problem with the regular and bold was with older versions of the OS.

If, when you copy text from http://zh.wikipedia.org/wiki/中國 and paste it in InDesign, Kaiti SC renders it properly, then the problem is not due to Kaiti but most probably to the encoding of the file you were given. For instance, I took the text http://typophile.com/files/text-gb2312.txt from www.bmo.com/s_chinese which is GB2312 encoded and saved it as plain text. If you open the .txt file with your browser, you will most probably need to choose view > text encoding > Simplified Chinese (GB 2312) so that it displays properly.

Added: the very same line of Chinese text can also be found in text-bm-utf8.txt; this time however you need to select the encoding Unicode (UTF-8) to view it. To get that new file, I just opened with TextEdit the previous GB 2312 encoded text and saved it in the UTF-8 encoding.
