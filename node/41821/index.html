This font has both ANSI and Symbol charsets enabled in its OS/2 table, so that MS Publisher probably recognizes it as a symbol font and maps typed characters to the U+F0XX Unicode range instead of normal Unicode values. To fix that, you should open the font in FontForge and go to Element->Font Info->OS/2->Charsets. Then do the following:

– Check the Unicode Ranges->Default flag; – Deselect the “Symbol Charset” line in the “MS Code Pages” list (only “1252, Latin-1” should remain selected); – Regenerate the font.

As far as I can see, this should help.
