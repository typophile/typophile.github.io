Which FontLab version are you using? 4.6? If so, move to v5.x if you can.

glyph class not defined (text was “@_o”)

In the Classes panel you can define (at least) two kinds of classes: OpenType classes, and Kerning classes. The name of the latter should always start with an underscore (_). That output suggests that you’ve not defined the class named ‘_o’. The Classes panel should have an entry that looks more or less like this: _o: o' oacute ograve otilde ocircumflex

2006-09-18 05:47:35 • Stephen Rapp

I first created it in the classes window the feature code didn’t seem to recognize it

Go to Preferences –> OpenType and confirm that you have the option “Add all glyph classes to OpenType…” checked. (I think it is unchecked by default.) [img:sites/default/files/old-images/ot_classes_6503.png]

I believe this will solve your problems.

BTW, the ‘kern’ feature created by FL4.6 is quite bad, since it divides the pairs in lookups. In some circumstances this will create kerning bugs, as it is described here: – Summing Kerning Pairs. That’s why I recommended moving to FL5.
