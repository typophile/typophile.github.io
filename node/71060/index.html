2010-06-05 03:47:43 • Frode Bo Helland
OT doesn't work on characters; it works on glyphs. There is no concept in OTL of 'end of selection'. Are you actually trying to reduce the size of the glyphs, as shown in your example, or is this just an illustration? In order to achieve this appearance using OTL, you font would need to contain sized alternates: you can't actually change the font size via OTL, and in any case a change of text size normally terminates a glyph run for OTL processing.

2010-06-05 05:34:30 • Frode Bo Helland
You could do it with a trigger glyph. So rather than having the context run backwards from the end of the selection you would insert the trigger glyph at the end of the selection and use that as the first context glyph. You would use a series of contextual lookups, something like this (using VOLT syntax; sorry, I don't do AFKDO): letter -> letter.smallest | trigger letter -> letter.smallerstill | letter.smallest letter -> letter.smaller | letter.smallerstill letter -> letter.small | letter.smaller and finally you would remove the trigger glyph from the string like this: letter.smallest trigger -> letter.smallest
