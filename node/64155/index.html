NUL is part of the old ASCII character set, (code point 00 in 8 bit hex). It's used in empty strings in programming languages. It basically means "no data". NULL is the name for the the equivalent code point in Unicode (0000 in hex). Not sure where .null comes from. Seems like an aberration. I was just reading something recently (in the OpenType spec, I think) that the only glyph name that is allowed to start with a period is .notdef. I would avoid .null.

I believe ".null" is the glyph name of the required second glyph in the Apple TrueType spec (after .notdef, before CR and space). FontLab has an option to automatically add ".null" "CR" and "space" glyphs when generating TrueType fonts. Cheers, T
2009-11-12 07:17:10 • Thomas Phinney
[Slaps forehead.] I actually looked at the FontLab manual where it talks about that option before I wrote the above. Somehow I only saw "NULL" and looked right past ".null". Thomas is correct. Also, it looks like ".null" and "NULL" are the same thing. This raises an interesting question, though. The FontLab manual states that only ".notdef" and "space" are required in Windows TrueType fonts, but that "CR" and "NULL" are also required in Mac TrueType fonts. Presumably, this means Mac TrueType resource-based fonts that are generated in suitcase files, a format that is effectively becoming extinct with the advent of Mac OS X, which can use Windows TrueType fonts (and OpenType fonts by extension). So, unless you are generating TrueType fonts as a Mac suitcase file, you probably don't need ".null" (or "CR" for that matter), since they are not required by Windows TrueType fonts, which work perfectly well on Mac OS X. And, according to Adobe's OT Feature File Syntax spec, which is the thing I mentioned above, ".notdef" is the only glyph name allowed to start with a period. So definitely don't put ".null" in an OT font.

Okay, so I've inspected a few .ttf fonts that Apple ship on modern Macs, Mac TrueType suitcase fonts that Microsoft includes with Office, and .otf fonts that Adobe bundles with CS apps. All except Adobe's .otf fonts include either "NULL" or ".null", but not both. Adobe .otf fonts I looked at did not contain ".null", "NULL", or "CR". I also check my own .otf fonts and none of them included those either. I would conclude from this that you probably should include "NULL" or ".null" in your TrueType fonts, but probably not both since they appear to be the same thing. Definitely not "NUL", though. I would just leave the option checked in FontLab and assume it does the right thing for the kind of font you are generating. Presumably, if the font already has a glyph named "NULL", it will not add ".null".
