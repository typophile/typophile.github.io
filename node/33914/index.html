Hi typophiles,

I’m in the process of porting Spiro curve editing to a Safari/Firefox canvas web app. I’ve just now got open Euler spiral splines working fairly well. You can play with it here. Just start clicking within the canvas window.

It won’t work in Internet Explorer. Use Firefox. Also, performance in Safari is weak. Use Firefox. You might find that closing your web browser and starting fresh helps a lot with performance.

There’s plenty more to do, including closed curves, corner points, UI niceties, and so on, but this should give people here an idea where it’s going. I will of course post again once it’s a usable curve editor.

2007-05-20 17:42:15 • hrant

William: yes, I did abandon the explicit tangent setting in the interests of keeping the UI simple. I found that I didn’t need it in my own work. If there is sufficient demand, I will implement it. Also keep in mind that there are other ways (in the full Spiro, but not yet in the web toy) to control the curvature in fontly ways, especially the one-way constraints.

hrant: glad you like it, and that the performance is close to acceptable.

how would you indicate that a previous segment shouldn’t change? Does that have to do with corner points? BTW, can you have a corner point with or without a tangent?

In the full Spiro, there are corner points and one-way constraints. The one-way constraints are there for indicating straight-to-curve transitions. I don’t have explicit tangents at corner points, but that would actually fit more neatly into the curve framework than wrangling tangents at smooth points. If I need a tangent, I usually just place a point near the end of the curve. Among other things, that technique lends itself more readily to interpolation between masters than does tangent setting.

The full Spiro also has both G2 and G4 smoothness constraints. Since those are mathematical descriptions of continuity, I think I’ll borrow terms from Erik von Blokland and call them “steel” and “rubber”, respectively. If people played with the C version of Spiro, they’ll notice that the G4 (rubber) points were the default, but you could get to the G2 (steel) by pressing the “6” key before drawing. I think the steel points are easier, especially for beginners, so they’re the default in the new Spiro implementation.

It will take a little while to get everything ported to the JavaScript version. I’m redoing the math to make it more numerically stable (not that you’d notice from the “diverge” error messages in the current version), and am also running into the fact that JavaScript is about 200 times slower than optimized C.
