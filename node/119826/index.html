2014-09-23 23:15:55 • JAB Creations
#xf049; etc. are Private Use characters. They are best being avoided, because their meaning is not defined, so when the specific web font doesn’t load, they can’t be displayed in a fallback font. You had the correct codes in your first post, you should definitely continue to use these (#x23ee; ...) The FontFont UI Icons collection contains all these symbols, designed to work together, but the fonts are not free: https://www.fontfont.com/news/new-release-ff67 (scroll down to ‘New Release: FF UI Icons’).

2014-09-26 05:21:52 • JAB Creations
BMP is going to give you access to the 65536 characters in "Plane 0", the Basic Multilingual Plane. Full will give you access to codepoints in all 16 planes, though I believe the total number of characters you can have in your font is still only 65536. The basic UI controls are all in the BMP, but if you ever want to expand upon that, there are some in Plane 1 too. I don't know of any reason not to go Full right away, except maybe that it's a bit easier to navigate the editor window when it has fewer character slots to scroll through. I only generate TTF and OTF, so I can't answer the question about WOFF. I do have an OFL font I'm working on that includes the UI characters you've mentioned, but it's not ready for release yet.

This script subsets fontawesome for me and saves the subset as WOFF: 
#!/usr/local/bin/fontforge -lang=ff

m = [61512, 61513, 61514, 61515, 61516, 61518, 61520, 61521]

Open("fontawesome-webfont.ttf",1)
SelectNone()
i = 0
j = SizeOf(m)
while (i < j)
  d = SelectMoreSingletonsIf(m[i])
  ++i
endloop
SelectInvert()
Clear()
Generate("fontawesome_subset.woff","woff",0)
Close()
