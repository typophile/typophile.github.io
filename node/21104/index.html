FWIW: Have just downloaded and tried that utility; seems to work as described on Adobe fonts. Mixed results on others, but that’s acknowledged by the creator. Gives you access to small caps, OS figures, ligatures and related features if they were included in the font. Doesn’t get at alternate glyphs. Because of the structured way it creates the name for the font containing the accessed characters, Word users could presumably then code a macro to swap out characters from one font to the other according to their own preferences. (Like, swap numerals to OSF except within a table, swap to small caps where a particular style’s used - or where faux small cap formatting is applied! etc.)

2006-07-08 16:28:48 • typequake

Adobe’s existing OpenType fonts ALL encode every single glyph in the font, with the small caps, oldstyle figures, etcetera at standardized Unicode positions (around U+F6xx-F7xx).

So there’s no need to hack apart the font with special tools. You can access all those glyphs now.

Mind you, Adobe is also abandoning this approach in new fonts moving forwards: glyphs without legitimate Unicodes, that are alternates or ligatures of standard characters, will be left unencoded (accessible by OpenType layout features).

2006-07-10 13:44:07 • typequake

Preferred answer: Use only applications that support OpenType Layout features.

Alternative answer: Word uses the Unicode codepoints. Any character with a Unicode codepoint can be entered into Word – through standard system mechanisms such as an appropriate keyboard layout, the Character Map application (through the clipboard), and also by methods specific to Word, e.g. typing the four-letter hexadecimal code and pressing Alt+X, or through a VB macro. [[Adam Twardoch]], in FontLab forum

N.B.: As Thomas said, Adobe will soon abandon the practice of using PUA codepoints, so the use of the method above is highly discouraged (even with fonts that contain PUAs nowadays).
