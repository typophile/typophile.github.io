2008-03-07 16:13:46 • Mark Simonson
You could also just reduce the size of the em, i.e. set the UPM value to a smaller size. If you are making a TTF or OTF, the UPM value can be any positive integer value. This approach avoids outline scaling rounding errors within your font source. If taking this approach, be sure to turn OFF the option to scale all glyphs to the new UPM in the FontLab FontInfo dialogue.

2009-02-06 21:29:12 • hrant
Michael, make sure you don't check "Scale all glyphs according to UPM size change" when you change the UPM. Otherwise, the relative size of the glyphs to the UPM will remain the same and the font will appear exactly the same size as before in applications. With the box checked, changing the UPM, in effect, simply changes the resolution of your font. With the box unchecked, changing the UPM changes the size of the glyphs relative to the type size.

My suspicion is that perfectly consistent stem widths are a non-issue at imagesetter resolutions and much more of an issue at screen resolution. But perhaps even then tiny inconsistencies can result in a more organic and less sterile typeface. That being said, I think the average would-be type designer would benefit from first learning the discipline of consistency and then later figuring out when, how, and how much to deviate from it. Sort of like learning how to balance letters, where first you need to figure out what makes letters seem optically balanced and then decide when it's okay to be a bit off-balance to increase the liveliness of the typeface.
