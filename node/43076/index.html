What Mark is alluding to, is a situation where the old hidden "resource fork" data has been stripped away from your font file by storing the file on a file system that doesn't support it. On your old system, make a .zip or .sit archive of the fonts and then move them over. At this point in history, font suitcases are just about the only type of file in common use that keeps critical data in the resource fork. In the old days most common document formats used resource fork data to identify the filetype, but you could still often get it to work by adding a file extension. Not so with font suitcases. Note: You can verify this by getting info on the file. If it's 0K in size, this is what is happening. This can be very confusing for people who recently switched to Mac because no analogous situation exists with other operating systems. Basically, Apple is providing backward compatibility with a 20yr old file format.
