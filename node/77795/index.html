2011-01-11 06:35:07 • Sye

Well, the word “binaries” is a technical term. It means files like the ones on your computer that end in .EXE or .COM for computer programs. You can run the programs, but you can’t read them; the files contain the actual pattern of bits that will be placed in the computer’s memory - possibly with instructions for relocation or linkage. (Actually, files like that are more likely to end in .OBJ, and for arcane technical reasons, while other computers had operating systems that did relocation and linkage when you ran a program, MS-DOS and Windows do without this flexibility, using other methods to link programs to external libraries.)

The other kind of file that isn’t a binary file is a “text” file. In the case of a program, that would be the source code - written in C, Pascal, BASIC, Assembler, or some other programming language.

So the “binary” is the final product - the program, or the font, all wrapped-up and ready for use… but not intended to be modified by the end-user. (In the case of fonts, though, while .TTF files are binary, they are what font editors work on; a font editor may have its own file format too, but there really isn’t a “source code” format for fonts.)

2011-01-11 22:46:28 • quadibloc

quadibloc: Of course DRM creates problems for law-abiding users of fonts, music, or whatever.

There is no DRM, per se, for web fonts. Digital Rights Management involves technical mechanisms to restrict use of digital content, e.g. to a particular user, with stiff legal penalties in some jurisdictions for circumventing such mechanisms. The minor protections available to fonts on the web do not constitute DRM; indeed, the majority of browser makers made it very clear that they were not willing to consider DRM for fonts. Fonts are protected by legal licensing terms, not by technical mechanisms.
