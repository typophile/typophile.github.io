2011-09-21 19:20:59 • BUROBRAAF

These are issues that you should probably take up with either Extensis (Suitcase not handling PST1 fonts properly) or with Underware (Dolly fonts not name-linked properly).

Or, the other possibility is that InDesign has a corrupted font cache.

The first thing to try in order to isolate the culprit would be to see if Dolly behaves this way in any non-Adobe software. Trying it in TextEdit is usually a good neutral choice. Testing in Word is often another way to distinguish an Adobe problem from a broader system-wide issue.

If Dolly behaves well in non-Adobe software, then it is a problem with InDesign. Delete your Adobe font caches (and restart for good measure).

If Dolly still shows the same mixed up problem, then take it out of Suitcase and put it in one of the system font folders. If it no longer shows problems in non-Adobe software, then the problem is with Suitcase’s handling of the fonts. Contact Extensis.

If the problem persists when Dolly is loaded via one of the system font folders, then the problem is probably with the fonts themselves. Contact Underware.
