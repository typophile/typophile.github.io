2009-12-17 00:46:54 • Tomi from Suomi
Unfortunately, RofoFog is based on Fontographer 3.5, a version which Fontlab Ltd. cannot maintain (we have not received the source code). Also, the RoboFog developers have long moved to RoboFab, a fully new font development environment, and they also established UFO as a new font interchange format. There are several RoboFab-based applictions available (Metrics Machine, Superpolator, Prepolator, Glyphs). We at Fontlab Ltd. have supported RoboFab from its inception, and we are now working on supporting UFO natively in our applications. It is likely that Fontographer 5.0 will be able to read UFO files, and we have plans to have full read-write support in Fontographer, TransType, FontLab Studio and other applications. Fontographer 5.0 is indeed under development. We are making good progress on the project. There will be a native Mac OS X version (universal binary) and a Windows version. The editing tools will stay largely the same, although some improvements have been made, especially in the user interface. Fontographer 5 will also include a streamlined Font Info dialog box (with much better family naming and linespacing management, hopefully much better than anything that we've made before), OpenType support (including quite smart, automatic OpenType Layout feature generation), better Unicode support. The output of OpenType (PS .otf and TT .ttf) fonts will be handled using FontLab's industry-standard output, so fonts made with Fontographer 5 will be technically fully up-to-date. In short, Fontographer 5 will be a pure type design application: stable, good, streamlined tools for type design. The type designer won't need to bother about any technical parameters, most of the work will be done automatically behind the scenes, producing optimized OpenType fonts (of course you'll still be able to make Mac and Windows Type 1 fonts). It will be more powerful than TypeTool and less complex than FontLab Studio. We hope that it will hit the "sweet spot" between simplicity and power. Cheers, Adam

2010-06-17 03:55:07 • Diner
Since it is now out, I have some time to comment :) UFO: not in 5.0, even for reading. We just needed some more time to test it and decided to not delay release even further for this single reason. Will be added soon: code is done, just needs some attention. Scripting: on our list for future update, but it will depend on how good it will go. Fog5 project already took much more time than we have planned, so we need to be very careful with our resources for some time. In any case, it will not be "live scripting" as in FLS, but "in-out", where font data is "out" to Python and then comes "back" from the script. Our plan is to make it Robofab-compatible. I/O: In Fog5 we have some mix of legacy and FLS code. Type3 input and output, MM output and .fog i/o (extended to support new stuff) is based on old Fog4 code. Everything else (OTF, TTF, T1, VFB, suitcases, export of BDF fonts) is hadnled by FLS, which is improved over 5.x. OTF export is done with Adobe FDK 2.5, so yes, you can add GPOS stuff there. Hinting: T1 glyph autohinting is done with Adobe algorithms. FLS code is not used. Font-level hints are generated by FLS algorithms in most cases, with one exceptions for alignment zones: in dialog box for their editing old Fog4 code is used (which is good and which is fixed compared to 4.7). Conversion of T1 hints to TT instructions is done by our code, the same that works in FLS5. Bitmap generation (if you need that): as Adam said earlier, Fog5 uses native rasterizers when you generate NFNT bitmaps on export of suitcases (T1 or TT). Internally, in the UI it will run old rasterizer, unchanged since 4.1. Windows and Mac: two versions are equal with one exception: you cannot import and export mac-format fonts (LWFN T1s and T1/TT suitcases) in Win version. That will be added later if all goes well (and if anyone will ever need it).

2010-06-17 15:41:04 • blank
1. The demo version for Windows has been updated now, sorry for that. Please re-download. 2. The main organization for the windows is largely the same as in FOG 4, i.e. a Font Window, a Glyph Window, a Metrics Window. No classes editor, no OpenType features editor, no Preview panel etc. You can specify OpenType Layout features in an external text editor using the AFDKO 2.5 syntax. We plan auto-generation of OpenType Layout features for common European scripts in a later version, but there won't be a "visual" feature editor -- this is something for FontLab Studio. I mean, you can use Microsoft VOLT or the demo version of FontLab Studio, or one of the DTL applications or FontForge etc. to develop the features if you want. Or a text editor. Or something else (I think DTL announced a GPOSMaster which they will present at ATypI Dublin). But Fontographer has rock-solid glyph drawing, convenient and easy Font Info, solid PS and TT autohinting, and can "do" OpenType, in principle. 3. The Illustrator copy-paste has been improved for format compatibility but the principle is the same. 4. The fractional-coordinate drawing is the same as in FOG 4. Generally, the drawing interface is largely the same, except antialiasing and zoom up to 1600% (in FOG 4 it was 200%). 5. The "add-on" basically means that as FontLab Studio user, you can buy Fontographer 5 for $99 and use both at the same time (so there is no special add-on application, you get the regular Fontographer 5 application). You can move your projects back and forth, you can e.g. design in FontLab Studio and then move to FOG and generate the fonts from there, benefitting from the much more cleaned up Font Info settings, the Adobe PS autohinting and the fact that it does AFDKO 2.5. So those who generated fonts using standalone AFDKO 2.5 could use FOG 5 now, then they'll be able to use Adobe PS autohinting and AFDKO 2.5 compilation, but also the FontLab-quality PS-to-TT hint conversion. Or, if you like FOG's simple drawing tools better, you can draw in FOG, then move to FLS5, use the class-kerning editor or Python scripting there, and then master your fonts in FLS. Whatever you like. FOG opens and exports VFB files. One catch: FOG won't import the OpenType features from VFB. If you have feature code in FLS, save it as a FEA file, save the font as VFB, open the VFB in Fontographer, then in Font Info / Encoding, point to the FEA file for feature definitions. It's not quite perfect but we didn't want to haphazardly clutter the FOG UI interface with new panels etc. -- that would make it the new FontLab Studio. However, there WILL be a new FontLab Studio. Fontographer is just an application that by definition is simpler, and should remain simpler. No manual fine control over some technical aspects of the font, but automatic algorithms instead.
