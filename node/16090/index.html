Organizing your font files

November 7, 2005 • souljacker

We mostly have much more fonts than we normally need, I’m no exception. Thus I was organizing my font files by splitting then apart in categories so I could enable just the kind of font I would like to work with.

After a while I became confused.

I really don’t know how to divide then. I don’t know whether to usa the habitual font categorizations or create my own system based on their usage rather then on their “academic” category.

Particularly I would like to split then in some kind of formal categorization but, since I’m not an expert, sometimes it get’s tricky to tell a Slab Serif font from an Transitional, for example.

So, I really would like to hear from you guys how do you deal with font files organization.

If you use Linotype’s new (and free) Font Explorer X (Mac only so far), you can make as many categories as you like, and the same font can be in more than one category. Other font management tools have this capability as well.

There is an old type specimen catalog that was put out by Phil’s Photo (now known as Phil’s Fonts) around 1980. It had what I thought was a very helpful way of categorizing its fonts. Here are the categories:

Evenly weighted sans serifs Thick and thin sans serifs Bracketed serifs Ruled serifs [e.g., Bodoni] Spur serifs Soft serifs Square serifs Extra light Ultra bold Condensed Italics Scripts Inline/Outline/Contour/Shaded Ornate Rustic

The actual categories they used are not important. The thing is to come up with categories that are meaningful to you and to allow the same font to be listed in more than one category. If you limit each font to only one category, you will never come up with a usable system. This limited idea probably goes back to font catalogs or reference books in which each font could only be listed once to save space or money, so they had to decide which one of possibly several categories a font could fall under.

If you know what you have, go with kris' idea.

Group your fonts by foundry and/or designer. Take Martin Majoor. In my Martin Majoor folder I’d have the Scala collection, Nexus collection, Seria, and so forth. When I think of Martin Majoor, I think of clean, amazingly legible, humanist slab-serifs with perfectly suited, accompanying sans faces.

These days we are blessed with plenty of serif faces with an accompanying sans. They are designed to work together, so you might as well keep them together under the designer’s name.
