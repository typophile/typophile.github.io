Help needed naming font family of four weights

April 9, 2008 • Vladimir Tamari

Hi. I have designed four weights of a new font family that I would like to appear in font menus with easily recognizable names reflecting their weight. For example

1-MyFont Light, 2-Myfont Regular, 3-MyFont Semibold 4-Myfont Bold.

Where the listed numbers are to simplify the discussion here.

Font #3 should be the bold for font #1 in applications such as Word Font # 4 should be the bold for #2 in applications such as Word All four weights should be accessible independently in the various font menus of different programs.

I have tried various naming schemes as suggested in discussions here and in the Fontlab MSN forums with little success. The name MyFont Medium seems to be generated automatically instead of the ‘Semibold’ for #3. Medium is an ambiguous term and it will not be clear to the user that it is heavier than ‘Regular’. I even tried installing extra differently-named versions of fonts #3 and #4 to appear as the independent bolds, but that conflicted with the already installed ‘linked’ bolds. This occurred even when I replaced the word ‘bold’ in the extra font with another word. I would really appreciate it if one of the kind type gurus here could help me solve this problem.

It would help if you told us what font format you’re working with, and what platform(s) you care about.

Assuming that you include Windows on the list of platforms you care about, “you can’t have your cake and eat it too.”

If you want to have the fonts linked as bold versions 1>3 and 2>4, then fonts 3 and 4 can NOT show up as separate fonts in typical Windows applications such as Word. Sad, but there it is. That’s just how fonts work on Windows (at least in most Windows apps).

If you pick which you’re willing to sacrifice, (A) style linking or (B) having four separate font menu entries on Windows, then we can give you more advice.

Also indicate what font format you’re generating.

Cheers,

T

I forgot to add another preference- that Word be prevented from generating its own “fake bolds”. These covered linking pairs of fonts so I now need more information on listing the four weights as individual entries,

If you want to avoid fake bold in Word, the style-linked method is your only option. This would be scheme A in the PDF (page 5, left column). Then, Word will show only Light and Regular in the font menu, while Semibold and Bold can be accessed via the Bold button. But you will see all four weights/styles in Adobe applications if you fill in “OT Family Name” and “OT Style Name” on the FontInfo’s “OpenType-specific font names” page – the “OT Family Name” should be the same for all four weights. Can you fill in the names and then post screenshots of the first two FontInfo pages, or send a link to a download of zip’ed screenshots? I think it’ll be easier to spot what went wrong.

Having the fonts “lie” about their usWeightClass is a little disturbing to me, but these wouldn’t be the first fonts to do so, I guess. Not a choice I’d make, but I can understand why Vladimir wants to try it.

I really don’t like the suggestion of just ignoring the OpenType names. That will cause the Windows-style names where every font is a separate family to carry over into OpenType. I believe there’s no good reason to punish savvy applications because of the limitations of typical Windows apps.
