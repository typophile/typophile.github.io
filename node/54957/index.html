Let me see if I understand -- You have 25 chapters. Each chapter might include multiple sections. Each of these sections, within each chapter, will begin numbering anew. So, you might have page 2.4 (Section 2, page 4) in Chapter 1, but also in Chapter 5? If that's correct, then InDesign might be choking on all the duplicated numbers (though I don't see why it should). Not sure why it's not keeping the associated page numbers with the entries. Normally a Table of Contents is not organized alphabetically, but rather in content/pagination order. Sounds more like you are creating an Index (even if you are putting it at the beginning of the catalog). Have you considered using the Index feature to generate your list instead? It's a lot more work to put in the markers, but it's probably more robust and might be better at handling your unique situation. Of course, at that point, it might just be easier to make up the list manually, the good ol' fashioned way. Unless you're planning on doing a lot of switching things around and want to have the TOC update automatically. Another solution might be to just generate the TOC in typical pagination order (which if I understood you correctly, InDD did just fine). Then copy the text into a word processing document, sort your list alphabetically, and reimport into your design file. If formatting is an issue, you might be able to export the text as tagged text and work around that in your sorting. -- K.
2009-02-19 15:32:21 • kentlew
António -- Try something: Check in the Book Palette flyout menu > Document Numbering Options and see if Document Chapter Numbering is set to "Continue from Previous Document in the Book". If not, see if checking this makes a difference in how the TOC feature treats your documents. You will need to set this separately for each chapter file in the Book palette. Then you may also need to select Book Palette > Update Numbering > Update Chapter & Paragraph Numbers. I'm not sure if this will make a difference, but it's something that might be worth trying. (You might want to back up your files: working with the Book palette and updating/synchronizing can sometimes have undesired effects and you'll want to be able to go back if this doesn't work and instead screws something up.) HTH -- K.
