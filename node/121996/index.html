2014-12-20 20:46:45 • andreaslarsen
Sorry, Andreas, forget what I said earlier - I was in a hurry and only looking at the basic characters, not the whole font. You'll need the WinAscent value a bit higher than the uppermost part of your tallest glyph, including diacritics. The tallest glyph is currently /A ring acute/ (Unicode 01FA) at 1085. So, the WinAscent value for Gidole needs to be a little above that figure. The WinDescent should be set lower than the glyph with the lowest node, which is currently /g cedilla/ (Unicode 0123) at -385. However, this glyph is usually represented as a /g/ with something like a /comma/ on top. See [[http://www.fileformat.info/info/unicode/char/0123/fontsupport.htm|this page]] and click on the 'view' link alongside each font to see how they treat this character. If you change your /g cedilla/ to match those in that list, your /A ogonek/ will become your lowest glyph, with a node at -202. You need to allow a little extra room, but enter the value as a positive one, not a negative one. As for setting up the metrics in FontForge, I'm afraid I don't use it so can't be absolutely sure about the 'General' value you're referring to.

Looks really good, Andreas, but I still think you have an issue with the shoulder of the /h/, /n/ and /m/. The right side is very 'flat' and I think the problem originates from using the /o/ in their construction. The /o/ has a slight diagonal stress, with the top-left/bottom right parts of the curve being thicker than the top-right/bottom-left. If you flip the /o/ before you start construction of the other letters, I think you'll find it easier to eliminate the 'flat' spots. I've made a rough mock-up to show what I mean.
