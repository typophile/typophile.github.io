Here is what I get from the Adobe Type 1 Font Format spec.

If two filled subpaths in a character overlap, there may be no problem when the character is filled. However, a Type 1 font program can also be stroked along its outline when the user changes the PaintType entry in the font dictionary to 2. In this case, overlapping subpaths will be visible in the output; this yields undesirable visual results in outlined characters. Always construct the character paths with outlined output in mind. I don’t know who uses stroked characters but let me tell you that in mathematical typesetting, if pieces of a large mathematical symbol do not overlap, the result may look like crap on screen, which I find that quite annoying. I am in a process of making a [[http://typophile.com/node/83674| connected script]] and I encountered no problems with overlapping characters. The only requirement is for the paths of your glyphs to be in the same direction.

Same goes for overlapping paths in an individual glyph.

There is indeed a problem if you want to visualize the outlines of the font, but any software that lets you do that must also provide you with an option to merge the separate forms into one (as in the Q&A example above). In Adobe Illustrator that is done with the “pathfinder” option
