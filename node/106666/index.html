Does anybody have any idea why is Microsoft doing this? Why can’t it render the font as Lucida and instead chooses Arial if the typeface has the characters?[img:sites/default/files/old-images/monte 1_3704.png][img:sites/default/files/old-images/monte 2_5445.png]
This is presumably due to RichEdit’s font fallback mechanism. I’m guessing there is some piece of metadata missing in the Lucida font that RichEdit is looking for as an indicator of support for particular Unicode ranges. So eveb though the font contains cmap support for the characters you want, RichEdit is switching to a fallback that it knows supports them. Yes, ‘tis frustrating.
