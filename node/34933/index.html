No kerning in InDesign but ok in Corel

July 6, 2007 • litera

Facts:
  • FontLab Studio 5
  • my own font
  • export to OpenType PS
  • kerning via classes
  • kern feature is the only OT feature and exists
When I use this font in InDesign I get no kerning info. If I use the same font in CorelDraw, my letters get kerned the way that kerning was defined. I've read somewhere that Adobe programs ignore kerning if there is only one (in this case kern) feature. What should I do and how should I do it?
For all those of you who run into this problem. I managed to solve it. What did I do? 1. created a "fl" ligature 2. created new OT feature "liga" 3. recompiled features 4. created my OpenType PS font VOILA. Two features and Adobe InDesign does kerning correctly.

Yes, it is a known bug in InDesign CS2 (you forgot to mention the most important "fact", i.e. which InDesign you're using). The number of features in the font is not important but the OpenType fonts must have at least one feature that does glyph substitution (so a GSUB table is present in the font). If only positioning is used in the font (e.g. in kern or cpsp features), the features will not work at all. A.
