2004-01-08 06:31:30 • Si_Daniels
I guess claiming that the user has broken the EULA would be difficult. First of all, you need to consider what the substance of a given EULA is. Often, it defines the object of licensing as "font software". The rendered bitmaps are usually product of such "font software". Bitmaps fonts derived from the rendered bitmaps are by no means font software, and if at all, they're being derived from the output of the font software, not from the font software itself.
Adam
2004-01-09 01:49:27 • twardoch
I believe this is the argument Bitstream originally used to say that TrueDoc didn't need to respect embedding permissions. Because they were simply asking the OS for glyphs at a given size and then packaging the results as a portable font resource (PFR).

I think you'd have a strong case for claiming that this particular tool is simply a font conversion tool - and that it's converting an outline font into a bitmap font. Although some EULAs allow for such conversion, few allow the derivative font to be distributed or bundled as part of a software product.

I'd like to hear how type designers feel about this. Do they feel it is okay for a games publisher (for example) to ship a bitmap version of their fonts without acquiring additional rights?
