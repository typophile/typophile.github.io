2011-05-08 21:00:05 • Florian Hardwig

@frode frank: Any digital fonts where the vertical metrics reflect this?

@hrant: How can there be more than one baseline?

In other words, to avoid confusing the innocent and naïve, while it may be true that in such a font, all the capital letters descend below the baseline (for the small letters) by the same amount, thus establishing, in effect, a second baseline, for the purpose of the vertical metrics of the font, there can only be one position that is presented externally as the “baseline” for lining up characters of that font with other fonts.

Now, in fact, I certainly could imagine a font format that does allow a second “caps baseline” to be associated with a font, and for there to be some complicated way to specify that “oh, by the way, on this line where I’m mixing these two typefaces, I’m using all caps, so please align the letters by their caps baselines rather than their regular baselines”.

And font metrics on Open Type are complicated enough that I wouldn’t guarantee that we don’t already have this feature.

Now I’ve checked. We don’t have it yet. However, as four lowercase letters make a baseline tag, all that has to be done is for someone to register a new baseline tag with Microsoft for this feature to become available.
