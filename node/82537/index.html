Guaraní character support?

May 26, 2011 • Andreas Stötzner

Andreu Balius mentioned in [[http://typophile.com/node/82501|this conversation]] an unsufficient Unicode support for the Guaraní language:

… languages like Guaraní that are not considered within the unicode system yet.

The [[http://en.wikipedia.org/wiki/Guarani_language#Writing_system|Wikipedia]] article didn’t enligten me about this.

Anyone knowing more details?

As I understand, based on review of the alphabet standardised by the Guaraní language congress in Uruguay in 1950, Guaraní requires eight diacritic characters, ã ẽ g̃ ĩ ñ õ ũ ỹ, of which one, g̃, is not encoded in Unicode as a precomposed character so requires use of the combining tilde mark character, and hence fonts with appropriate GSUB or GPOS support for the sequence g+◌̃. In addition, Guaraní has a number of digraphs that sort as distinct letters, requiring tailored sorting and collation algorithms, since these digraphs are not encoded as units but as sequences of normal Unicode letter characters.

So Guaraní requires particular kinds of software support, but this is not unusual for Amerindian orthographies or, indeed, for many of the world’s other languages. The fact that some precomposed diacritic combinations are encoded in Unicode, and indeed a few digraph combinations, is a technical accident brought about by backwards compatibility requirements. If Unicode were starting from scratch with no such requirements, no precomposed diacritics would have been encoded, and all languages would be in the same situation as Guaraní. Personally, I think that would have been a good thing, because it would have impelled software makers in North America and Europe to produce the kind of text processing solutions that Guaraní and many other languages need.
