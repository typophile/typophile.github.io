I may be wrong, but I believe that the reference point is not part of an exported font, but is only used (optionally) within FontLab as a reference for operations like rotation. If you export the font and then re-open it, you will see that FontLab puts all the reference points in at 0.0 . In other words, unless you are using it as a reference point in an operation--such as an instruction to rotate around the reference point--you can forget about it.

This is the macro I use in order to reset the reference point: #FLM: reset rpoint from FL import * f = fl.font for i in range(len(f)):   f[i].rpoint = Point(0,0) fl.UpdateFont()

>note to self I kept moving it back to 0.0 for months until one day I asked myself, why the heck am I doing this? This illustrates a pet gripe about the FontLab manual. It tells you that the reference point is useful for a symmetrical letter--though I'm not sure why as you have the option of rotating etc relative to the center of the selection--but doesn't tell you that it's only of any use in restricted situations. Basically, you have to figure out how its functionality or lack of functionality fits into the job of developing a typeface. A manual ought to give the user more guidance on this. Also because you never need to use it at 0.0 it makes more sense for it not to be always on the glyph window, but rather to be inserted optionally when you want it, at a desired rotation center, etc.
