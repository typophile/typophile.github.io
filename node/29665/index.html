Jean Baptiste,

  1. What do you mean, the font won’t compile? Are you just getting the warnings, or are you getting the warnings and the font does not compile? It’s a big difference.

  2. If you define your classes both in the Classes panel and in the OpenType panel, then of course it is logical that FontLab Studio will warn you that your classes are defined twice (hence, “redefined”).

If you prefer to use the class definitions from the Classes panel, then there is no reason of keeping them in the OpenType panel.

If you prefer to use the class definitions from the OpenType panel rather than from the Classes panel, set the appropriate option in Preferences (i.e. disable General / Unicode and OpenType / Add all glyph classes…).

A.

2006-11-23 11:49:30 • Jean-Baptiste Levee

classes in OT panel were a workaround in previous versions of FL, wasn’t it?

No, not really. If you use classes in the OpenType panel only, you can use all the aspects that the AFDKO syntax allows you, for example have classes that include classes, e.g.:

@LETTERS = [@UC @LC @SC];

This is not possible in the Classes panel. On the other hand, the Classes panel gives you integration with some UI elements in the Metrics Window (the little flyouts, the blue buttons in the Kerning table etc.) which you don’t get when you use classes in the OpenType panel. I don’t think it’s really possible to completely merge those two paradigms since the one is more an engineer’s, the other more a designer’s view.

I for my part work in the way that I use the Classes panel for Metrics and Kerning classes only, while I directly specify classes in the OpenType panel for use in other OpenType Layout features. Since there is no overlap between these two sets in my case, I keep the “Add all glyph classes…” option enabled.

I also copied the plist preference file

The supported way of backing up and exchanging preference files is by using the “Save” and “Open” buttons in the Preferences dialog. This will produce a .reg file which is cross-platform compatible between the Mac and the Windows versions of FontLab Studio.

Best, Adam

Jean Baptiste,

the .reg format is the Windows Registry format – something like .plist but on Windows. Since FontLab Studio is originally developed on Windows, some of the native data structures are tied to the Windows architecture and then replicated on a Mac through some translation.
