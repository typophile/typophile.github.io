2006-08-15 03:16:02 • Stephen Coles
Hi Billy Stephen's right. There ain't no magic number. The closest you might come to a pre-set formula is the (factory set) default value for auto leading in Adobe InDesign and Quark XPress – 120% of the type size. In other words if you set 10pt type the leading will measure 12pt from baseline to baseline. That's only two points extra to the body size – almost certainly too tight for sans serif faces, and not necessarily ideal for the majority of the serif faces either; I advise my students NOT to use auto leading.

Better than “what looks right” is to do “what reads right”. Typesetting is really all about producing text you can read and comprehend without distraction. If you don’t read and care about the content you’re setting you’re much less likely to set it well. So, experiment: try various settings using various texts (that you aren’t already familiar with) and see what you read most comfortably.

2014-10-01 12:07:20 • tomita
Language should affect leading. In German all nouns are capitalised, and I think “k” is more common than “c”. This means there are more tall letters, so the leading should be greater. In Latin literature as printed nowadays, capitals are usually only used for proper nouns, not for new sentences (which may mean extra space between sentences); “k” is very rare. There are fewer tall letters, so less leading is needed.

Something else affects leading. Suppose you have a paragraph of block capitals. If it is a copy of a Latin inscription you want to make it as easy to read as possible. It's certainly true that 120% is not going to be always right. However, maybe the poster was never going to use any other typeface but Times Roman, in which case a simple number, whether it was 120% or not, would work. Could one generate a magic formula if one knew the x-height of the typeface in question, in addition to its point size? I don't see why not. It still wouldn't yield perfect results in all circumstances, but as a rule-of-thumb for a first cut at typical body copy, with all the usual qualifications, it might work. I suppose the idea would be that Caslon and Jenson don't need leading, and other typefaces need as much leading as makes up for the missing height in their ascenders and descenders. However, if the x-height is given as a ratio between x-height and cap height, you still need the ratio between cap height and point size.
