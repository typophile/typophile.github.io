I wrote a bit about this last year. Since then, there has been a lot of copying of the soft/rounded and large type trends, but I think those are reaching the end of their life cycle now. Can't really say what's next, except that typography is definitely in a better shape on the web now than it was 5 years ago. Web designers, a group formerly obsessed mostly with imagery and effects, now generally regard good typography as an important part of their design process.

2007-09-04 21:16:38 • canderson
Do we really believe that the screens will become bigger and solve part of the web typography problem? No. Screens will have higher resolutions in terms of pixels-per-inch. The current problem is that a lot of what is displayed on a screen is not resolution independent. So if you switch to a 200 ppi screen, some things will look really small. Text will look sharp, but certain user interface elements with current operating systems will become unusable. Some OS components, like the dock in Mac OS can already be resized. But for the real revolution to occur, everything on screen needs to scale. Making the web-scale is another special case. One could imagine a superpowered browser rendering engine that would say, automatically resize (resample!?) bitmaps as they were being displayed. Images wouldn't look any better, but text would be super-sharp. My prediction on this: if anything like this is ever released, Apple will do it first. Why? Because they have the luxury of breaking backwards compatibility.
