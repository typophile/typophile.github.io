Major problems with kerning and classes in FL5.02

October 16, 2006 • Ramiro Espinoza

Hi There, I am having major problems with FL5.02 and kerning tables. I am dealing with a file originally made in FL4.6. When I work on the kerning or classes if I close the file and reopen it, the changes are gone! I also have a weird thing with the sidebearing of "N": both show '0' values. If I change them, after I close and reopen the values are set again to '0'! Is there any similar documented problems or bugs in FL5.02? Thanks in advance. Ramiro Espinoza.
When you do both kerning and metrics classes make sure that you go to tools in the menu and apply 'update kern feature' in kerning assistance, and 'apply and save' in the metrics assistance. Have a look at the metrics class in the classes panel. The N is probably referenced to a (or 2) character(s) that have a sidebearing value of zero. Nick Cooke
