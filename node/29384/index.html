Goran,

What do you exactly mean by “it doesn’t work”? Please be more specific.

If a font can be generated, it works in Quick Test. If a font does not work in Quick Test, it simply means that the font cannot be properly generated and installed on the system (because it’s all that Quick Test does, really). Therefore, you should verify how your OpenType features compile, if the font can be generated and if it can be installed on your system. (FontBook’s reluctance to install some perfectly valid fonts may be one of the problems if you’re on Mac OS X).

Goran,

the dialog box is normal. When you open an existing OpenType font, there are stored binary OpenType Layout tables, and you may choose which information you want to use – those stored binary tables or tables compiled from the code that is in the OpenType panel. Of course you can remove either the ones or the other, or you can control this behavior in Preferences.
