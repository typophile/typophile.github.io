consistent width symbols?

June 17, 2009 • eliason

In a proportional font with tabular numbers, would you expect other symbols to share the width of the figures? If so, which ones? plus, multiply, divide, etc.? equal, greaterthan, etc.? dollar, sterling, yen, currency, etc.? pound? asciitilde? logical not? percent? perthousand?!

There was a thread on this…

Anyway, I don’t think there is any need, and it really compromises the appearance of glyphs, especially the Euro symbol. The reason is that figure positioning in columns is now done by tabbing, rather than filling space with quads.

Having said that, I do try and make the Dollar and Sterling tab figure width.

Actually, it’s a good idea if you’re thinking about tab figure width, to consider designing all figures in a family to the same width from the outset – lining and oldstyle, different weights, roman and italic.
