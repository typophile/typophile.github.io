How to import small caps kerning pairs into FL

June 15, 2005 • filip blazek

I have a following problem (FL4.6, Windows).

I have a typeface with full set of small caps glyphs. I didn’t assign an Unicode index to these glyphs. I want to import a separate AMF file containing kerning pairs:

KPX d.small a.small -60

But FL refuses to add such pairs unless I assign Unicode index to small caps glyphs. Is there any way, HOW to import kerning pairs for all the glyphs, regardless on Unicode index?

I don’t want to make separate font, copy small caps, remove ‘small’ suffix, import AFM, add suffix and copy small caps glyphs to the main font. It is too complicated. I also don’t want to temporarily assign Unicode indices…

(Just came across this thread. It’s amazing how many threads include the word ‘kerning’ in the title.)

Filip, you’ve probably solved this by now. But anyway, here’s a small Python script that does the job. You’ll need Robofab installed.

#FLM: AFM Kerning Importer from robofab.world import CurrentFont from robofab.interface.all.dialogs import GetFile f = CurrentFont() PairsBefore = len(f.kerning.keys()) print ‘Pairs before:’, PairsBefore path = GetFile(“Open AFM file”) f.kerning.importAFM(path, clearExisting=False) PairsAfter = len(f.kerning.keys()) print ‘Pairs after:’, PairsAfter print ‘Pairs added:’, PairsAfter - PairsBefore

The script also gives information about the number of pairs added, just as a checker.
