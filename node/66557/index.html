This is an idea I’ve been mulling over for a day or two: could we as a web community design a font for Typophile?

This font/font family would be owned by Typophile, and all royalties would go directly to the same place donations would, and rights would be managed by the same people handling the funds for the site, and all designers would agree that this is all done for sole purpose of funding the community, and the administrators would agree to use the profits of the font to be used only for Typophile.

We could all participate in the design; swap font files weekly or monthly with willing designers, and have ongoing critique online.

2010-02-05 23:33:27 • Tomi from Suomi

  1. Do we make a list of who’s working on this, or just let people jump in?

  2. Since we’re not assigning characters, anybody can work on anything, right? The potential for disaster is great, but it could also be quite rewarding. So what we need I guess is: a public (or semi-public) place where work files are stored; and a simple versioning system where people can branch off, but a branch can be ignored. I suggest the following filenaming system: “tphf_n-n-n_xxx.vfb” where “n-n-n” (which starts with one “n” and adds an “n” when a branch splits off) is a branching path, and “xxx” is a version along the branch.

So “tphf_2-4_011” would mean “the 11th iteration of subversion 4 of version 2”. What you have now would be “tphf_1_xxx” where “xxx” is your current version number; and if somebody decided the change the weight contrast for example he would make “tphf_2_001”; after that if somebody adds/changes glyphs to that it would be called “tphf_2_002”, but if that person changes the vertical proportions for example it would be “tphf_2-1_001”.

We also need a shared text file where people clearly explain what they changed/added. Lastly, we would use a Typophile thread to show changes and ask for feedback.
