2010-02-02 23:31:18 • 1985
You can work at 2000 UPM, unless the fonts are to be generated as Type 1 (those need to be 1000 UPM). If you scale the UPM, you can also tick the ‘scale all glyphs…’ as well if you want to physically scale the characters themselves to fit the size. Although you have to remember that scaling up and down between UPM sizes with ‘scale all glyphs…’ turned on can introduce rounding-off issues into your characters. I.e. some anchor-points might slightly move, especially when scaling a 2000 UPM character down to 1000, because the grid on which the points need to align is more coarse. The changes are tiny, but if you’re working very precise you might want to keep it in mind.
2010-02-03 02:15:03 • Quincunx

do you know what the default setting is?

Not sure, you’d have to re-install FontLab to find out. In the user manual (p 77 in the Mac version) it says 100% & –40%, which is a good default value, I think.

You can lower the upper value to just above your ascender height, and probably raise the lower one as well. It depends on whether you prefer the glyph as large as possible or rather make sure nothing is ever cropped. It’s a personal preference, that’s why it’s in the options. It also has an effect on the Preview/Metrics window, that’s when I typically decide to change it.
