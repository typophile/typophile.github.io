How to embed fonts in PDF from InDesign

March 10, 2006 • maxack

Hello,

(I tried to find anything on the issue, but no luck, sorry in advance if it has come up before)

When exporting a PDF from InDesign (Mac OSX), how can I make sure all fonts are embedded? It only gives this uninformative notice about permission bits…

Can I check later in Adobe Reader if fonts are embedded, and if so, how?

Thank you for helping.

  1. In the box “subset fonts when percent of characters used is less then” put in: 100% The informative notice you’re talking about is a hint for the user to check if the font policy allows embedding, so if that’s ok, just click “ok”, that is: go on with exporting. If there is no appropriate permission, the pdf exporting will stop anyway. So generally it’s recommendable to check permission issues before using a font in a publication.

  2. In Adobe Acrobat uncheck “use local fonts” (Cmd + Shift + Y), you’ll find this option under “Advanced”. Possibly there is a similar option in Acrobat Reader.

2006-03-10 13:22:14 • maxack

It means fonts on your computer. i.e. if the box is unticked, it will only render the document using the fonts which are embedded. Other text will appear in a substitute font.

In Acrobat (or Reader) File >Properties > Fonts will give you full information whether fonts are embedded, subset embedded or if a subsitute is being used for display (in which case it’sll tell you which)
