Well for instance building a website for a government organization means you have to work with a different set of constraints then say building a personal design/portfolio site. On a government site there’s a strong emphasis on the lowest common denominator both in the content, which needs to be readily comprehended, as well as audience technology constraints (low bandwidth, older computers, limited or no Flash, limited JavaScript, older browsers, etc.) In such cases one may not be able to use such tricks like SiFR, or Flash, and even JavaScript DOM manipulation, as bandwidth and older computer issues make such technologies prohibitive. This makes it difficult to use anything but basic web fonts unless you resort to using images. However images can also run into bandwidth and accessibility issues. So like most design problems, it’s a bunch of trade-offs and you have to carefully figure out what works best and strike a good balance given the project requirements.

On a personal portfolio site though - you’re aiming for a different audience. Bandwidth is generally not a huge issue. You don’t worry about whether the user has Flash or uses an up-to-date browser because those who don’t are not your target audience. So you can create a much richer typographic experience because you don’t apply the same constraints. Ultimately, it really comes down to the project requirements and the target audience.
