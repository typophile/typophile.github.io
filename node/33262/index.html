Quartz rendering oddity

April 24, 2007 • guifa

I’m working up a post on this issue I’ve found in my design process as well as in some other (Apple-supplied) fonts on my computer to send to Apple’s bug reporter, but figured I’d get some comments from the build community on what other causes/symptoms/solutions might be.

If you take a font like Lucide Grande, comparing the double-acute and the double-grave at a low font size, the double-grave almost disappears, whilst the double-acute is very visible. I’ve run into it randomly on some of my fonts, but I finally figured out the cause (but not the reason for the symptom):

When doing anything with combining accents, it’s extremely important that the directions are the same for both the diacritic and the baseform. Otherwise, OS X has some odd rendering issues. I made a simple font to illustrate this. In the attached PDF and font, the character A as a long thick line with a square dot over it, and B being the exact same, except with the dot made counter-clockwise instead of clockwise.

[note: half finished with this write up and accidentally posted, bout to run to work but will finish it shortly].

This isn’t very surprising, path direction in TT fonts must be clockwise (black on the right) and path direction in PS fonts must be widdershins (black on left). While renderers do not necessarily have a problem with contrary path directions, some clearly do and the usual result is invisible or outline shapes.
2007-04-24 03:56:37 • John Hudson

Mainly when I run into this (and, I believe, though I’ve not opened up Lucida’s file to double check), it appears to be related to applying transformations to referenced glyphs. If the transformation is a flip, should the path direction be flipped as well, or should it be maintained? While it makes sense for the latter since a mirror image should go in the other direction, for font designing purposes, it makes more sense to have the direction corrected during transformation.

For now I guess I’ll just dereference the glyphs after transformations, but I was wondering to what extent I should consider this a “me” problem, a FontForge problem or a Quartz problem.
