2005-05-19 20:37:36 • Mark Simonson

It is possible but very, very unlikely that a trojan font could exploit a bug in a font renderer and propagate itself that way, but I think you probably got hit by something else. Viruses spreading via fonts would be sufficiently exotic to hit BUGTRAQ or vuln-dev, and I haven’t ever seen talk of anything other than the OpenType crasher on Windows to which Mark alludes, and that bug has subsequently been fixed.

The FNTCache.dat is merely camouflage, and has nothing to do with fonts (necessarily). Font caches are common system files, and using them as a place to store viruses or virus data is just a little bit of social engineering on the virus-writer’s part to keep you from noticing they’re there. Update your virus definitions and see what your virus checker tells you.

2005-05-20 05:16:24 • Eric_West

I was thinking about where one might go looking for exploitable bugs in type renderers during dinner just now, and I assure you, none of the specific attack vectors I was thinking of were Windows or x86-specific. OpenType is a complicated, new technology that contains a lot of other complicated technologies (like, say, TrueType hinting), and everybody’s got about the same level of experience implementing support for it. It’s just as likely that Apple’s overlooked a few boneheaded programming errors as Microsoft has – or that the Freetype team on Linux / *BSD have.

So, in other words, settle down there, buster. Just because the only high-profile OpenType crasher found so far was on Windows doesn’t mean that somebody might not find a bug somewhere else next. There’s a time and a place for OS-bashing, but in this particular fight, it’s those of us who don’t want viruses against those who want to give them to us, and other concerns are secondary.

One way to check the authenticity of a TrueType font in Windows is to download Microsoft’s free Font Properties Editor, which allows you to access a lot more information (several tabs worth) about the font when you right-click the .ttf file and select “Properties.” If, indeed, you have a real TrueType font, you will see that information; if it’s not really a font, the Properties dialogue box will probably be seriously confused.
