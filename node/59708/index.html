Web H&J Recommendations

July 7, 2009 • dberlow

I’m doing something, and I know there isn’t much H&J on the web, but I have a question(s).

First, if a type designer were giving recommendations for Justification of a particular font, do you all think that recommendation should be a binary decision, (i.e. Y/N)?

Second, if a type designer were giving recommendations for Hyphenation of a particular font, do you all think that recommendation should be a binary decision, (i.e. Y/N)?

Thanks in advance for any input.

Cheers!

Justification without hyphenation on the Web tends to look pretty bad, so I would discourage it. As to the second question, I guess it depends on how the typeface stands fiddling with its spacing. But being that there’s no hyphenation on the Web – yet! – it’s another N for me.
2009-07-07 16:37:32 • Antonio Cavedoni

On the one hand, I would say that neither recommendation should be binary Y/N because you can’t anticipate all the factors (different browsers, different H&J algorithms (or lack thereof), future-proofing, backwards compatibility, text size, etc.) that would influence the recommendation. On the other hand, I’d say that the recommendation has to be binary Y/N because you can’t anticipate all the factors….

Put it another way: the recommendation shouldn’t have to be binary, but I don’t see how you could make the recommendation more nuanced given that you can’t know for certain what the reader will be seeing.

And then, I’m also thinking of dedicated headline fonts, where H & J answers are also both fairly stark…
