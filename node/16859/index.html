2006-01-04 11:36:24 • dan_reynolds
If you’re troubleshooting a particular font, I wouldn’t recommend switching font managers around. In fact, I would recommend disabling any font manager you might have running and test the font in the system. Put the font in /Users/your username/Library/Fonts/. When I’m testing fonts in this way I keep /System/Library/Fonts/ with the default Mac OS system fonts and keep /Library/Fonts/ and /Users/your username/Library/Fonts/ empty except for the font I’m testing. If it shows up in Apple’s FontBook utility, does that mean that you already have a copy installed in the system? When you said that you “disabled all other fonts” does that mean you deactivated them in your font manager or that you disabled them in FontBook? I wouldn’t recommend using FontBook’s feature for disabling fonts while using any 3rd party font manager. How exactly did you clear the system font cache? Clearing Mac OS 10.4 font caches will re-enable any fonts disabled in FontBook since that is where that property is stored. The fact that the font works in other applications is a clue that the problem is InDesign CS 2 related. It’s a fine idea to take other software out of the mix to verify this. In fact, if you have to deal with Quark or Adobe support it’s important that they don’t have any other software to blame. You may also want to disable any 3rd party plug-ins as well, even though it’s unlikely that the issue is related to auto-activation. Both the OS and applications may cache font data so it could be that the problem is related to some CS2 cache. In the past, Adobe applications used “adobefnt.lst” files to keep track of what fonts were active. I don’t think that is the case for CS2. Does anyone know?
