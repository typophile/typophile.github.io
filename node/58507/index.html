Strange problem with RockoFLF

I recently downloaded a font, [[http://www.fontsquirrel.com/fonts/RockoFLF|RockoFLF]], and have been loving it, except that I’ve noticed one small problem with it.

Basically, when using the font in Illustrator CS4 (PC), I notice that sometimes the counters of the “a” and other letters go a bit crazy, and it makes the weight vary in some point sizes. But then the effect disappears in other point sizes. The screenshot below shows you what I’m talking about. The letters in black are how they should be, but the ones in pink show the errors. And the funny thing is it all corrects itself if I convert the text to paths.

I’m wondering if this is a hinting problem with the font, and if it’s possible to correct something like this in a free program like Fontforge?

Yes, bad TrueType hinting. Should be fixable in FontForge (but I have never used FF for hinting).

Daydream,

if you want to use a free tool, you could either use FontForge to regenerate the TrueType hinting, or you could do something else: use Adobe FDK for OpenType to convert the font into OpenType PS (.otf) and autohint it at the same time. After installing the Adobe FDK for OpenType from http://www.adobe.com/devnet/opentype/afdko/ just type in Terminal/commandline the two lines one by one:

makeotf -f RockoFLF.ttf autohint -nb RockoFLF.otf

and use the final RockoFLF.otf. The PostScript hinting model used in OpenType PS fonts tend to be more robust and less prone to “strange errors” caused by automatic hinting.
