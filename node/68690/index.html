Generally, I would say that using <- for the leftwards-pointing and -> for the rightwards-pointing arrows is a standard. For the verticals, either |^ or -^ makes sense. It is a matter of mnemonic. It also depends on the scope of arrows the font is going to support. If you want to cover significantly more than the basic four or eight directions and possibly allow for smooth combination and lengthening of the arrows (this is something Skolar does), you need to think hard. E.g. I like the /^ sequence for diagonals, but I could not use it because I wanted to support harps (-/ sequence). Etc. On the website mentioned above is a cheat sheet in PDF describing what combinations are used. It is by no means a standard, but as far as I know it has the widest coverage around. Best thing to let designers know about these things is to put it in a specimen. Also, these features are usually placed in stylistic set (any you choose) and/or discretionary ligatures (dlig). The latter is to allow support in Illustrator which did not support stylistic sets in CS–CS3.
