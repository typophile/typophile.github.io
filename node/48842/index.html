More kerning classes questions

August 27, 2008 • blank

The FLAB manual isn’t up-to-date regarding Preferences > Generating Opentype & Trueype > Kerning, so I have a few more kerning classes questions.

First, should I use the “Expand class kerning while building the [kern] table feature? I worry that there might be important programs that don’t support [kern] and that expanding the pairs will keep the font working in said programs.

Second, is the Generate OpenType “kern” feature if it is undefined or outdated box there so that I don’t have to manually update [kern] every time I change my kerns, or do I still need to do that manually?

The problem with expanding class kerning is that in a font with lots of language support, you will get massive kern tables of tens of thousands of kern pairs, many of which don’t occur in any language.

I’m presently tackling the problem by having only a very limited number of kerning classes and kerns in non-OT fonts.

I would not activate “Generate…feature if it is…outdated.” The reason is that you may at some point inadvertently lose your kerns, for instance if you have removed the basic kern table from an OT font before generating.
