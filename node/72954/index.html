Vietnamese accents: beyond the Bbox.

August 6, 2010 • Ray Larabie

When dealing with high Vietnamese accents such as: uni1EA4 uni1EA6 uni1EA8 uni1EAA uni1EAE uni1EB0 uni1EB2 uni1EB4 uni1EBE uni1EC0 uni1EC2 uni1EC4 uni1ED0 uni1ED2 uni1ED4 uni1ED6 There's a problem to deal with because the accents are vertically stacked. Usually when figuring out font metrics, I assume Aring is as tall as it gets and scale accordingly. But sometimes I later need to add Vietnamese accents. When I refer to Bbox limit I mean the highest point. In my case, it's usually the same as the Winascent and hhea Ascender. As far as I know, there are 3 solutions: 1: Reduce the size of the entire font to allow for the high accents which will make the font look tiny compared to other fonts and in some apps it will affect line spacing. Not a great option when adding to existing fonts. 2: Squashed accents which don't exceed the Aring height. This can be acceptable for display fonts but not so good for text. 3: Go beyond the upper Bbox limits (usually the Aring height). Some OS core fonts have Vietnamese upper accents which go beyond the Bbox limit. What happens to Vietnamese core fonts in various browsers? I assume the accents wouldn't get clipped. Do browsers pad the line spacing with Vietnamese the same way they pad superiors & ordinals? What are the negative consequences to exceeding the Bbox limit this way?
Don't forget (4): Increase the BBox size, along with os_2.usWinAscent and hhea.Ascender Of course, (4) would result in increased line spacing in many less savvy apps. But (3) will result in clipping in many less savvy Windows apps. Of course, you wouldn't "go beyond the BBox limits" exactly: you'd increase the BBox but not the os_2.usWinAscent and hhea.Ascender. Cheers, T

The 2004 context of the sudden urgency around the long practice of apps using OS/2 usWin values to determine default linespacing, although I couldn't say so at the time, was the MS Cambria Math font. We needed a way to get Word to scale to the Typo values in the font menu, because otherwise the Cambria Math font looked like it was about 3pt.
