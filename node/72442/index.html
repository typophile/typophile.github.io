2010-07-21 04:13:49 • russellm

It’s actually not a bad idea.

I think it’s a bad idea. Using GSUB to switch between different numeral systems makes no more sense to me than using it to perform transliteration, e.g. switching between Latin letters and Cyrillic ones. It makes much more sense for numeral conversion to be performed at the character level by applications than at the glyph level by fonts.

I thought about this feature (as a stylistic set) due to many people who don’t handle or understand properly Roman numerals these days. But even if one does not know what LXVIII means, it may have to face this is some areas, especially Law, History and Philosophy.

Although I’d prefer to use a GREP style in InDesign to manage this, user who has doubts about Roman numerals may not be able to adopt this solution. So, a stylistic set can be useful.

Interesting to not that a designer claims to offer this as an “exclusive feature” in a [[http://new.myfonts.com/fonts/isaco/monarcha|font]] released much later than Operina and also after I published the OT code in the thread referred by sii.
