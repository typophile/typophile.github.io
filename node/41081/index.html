OT Case Feature

January 18, 2008 • Michael Jarboe

I have a couple OT fonts I’ve recently finalized that utilize the OT case feature for punctuation.

Recently I’ve implemented and expanded my OT ‘aalt’ features (thanks to Typophile) using lookups with the ‘salt’, ‘ss01’ and ‘titl’ features to expand the range of access since not all UI’s are created equally.

I’m curious if there is any expansion of the ‘case’ feature similar to the ‘aalt’ feature that makes the ‘case’ feature easier to implement. I’ve noticed in Illustrator ‘case’ is only accessed via the glyph palette.

Does anyone here use any other feature tag and lookups with ‘case’ to expand it’s access with differing software interfaces?

Thanks.

Mike

In Illustrator and InDesign the ‘case’ feature is applied automatically when you apply “all caps” to the text. Maybe not obvious… The same is true of capital-spacing: ‘cpsp’ (not 100% sure about Illustrator here.)

Mike,

One thing to remember is that “calt” is on by default, at least, in InDesign. That means whatever you write will be applied unless the comp takes action to turn it off. If you want the comp to have to take action to turn it on, put the same features in a stylistic set. They will have to turn it on before the substitutions are made.

Not a huge burden on the comp BTW. If you base all paragraphs on one basic style, and turn on the stylistic set in that paragraph style, it will continue through all the paragraph styles based on it. And/or, it could, by action, be turned off in any paragraph styles the comp chooses.

While this may in some ways violate “proper” code, in my mind the decision should be based on whether you want the substitutions on or off by default. Remember, I only know InDesign.
