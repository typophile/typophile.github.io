i’ve had this happen and it’s maddening.

might be a situation with bridge and if there are multiple files on the drive named ‘document.indd’, revisions and edits and whatnot … if there’s an application error and it has to recover or start again at what it thinks is the correct version of ‘document.indd’ that it pulls from the wrong place, you can have those things jump up and bite you. especially since the last few edits on anything are small punctuation changes, a factchecking clarification or subtle design adjustments.

i’d recommend taking on a naming convention where you have the day’s date at the end. ‘document102009.indd’ so if you have to recover or compare it, you don’t have to go on an archaeological expedition to see the differences.

I once had problems with the line breaks when we began the book layout in Indesign CS3 and updated then to CS4, some parts had different line breaks in CS4.

Just FYI, that’s not a bug – it’s documented. The line breaking algorithm is revised for CS4, as the older versions (at least going back to CS) tended to kick out the last single word or syllable, leaving this more often on a last line than statistically likely.

The new line breaking routines are only applied to stories that are actually edited in CS4; if you don’t edit something, it’ll remain the same (and if you do edit something, the line breaks will be better!).

Whenever and whereever possible, I refrain from upgrading software across documents.
