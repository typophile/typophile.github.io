Cursive GPOS Vertical Position Not Resetting Between Words

November 11, 2010 • bunnyfly

Hi. I’m using Adobe FDK’s feature file to add some GPOS tables into a font I’m working on. The script I’m working with requires pretty complex cursive positioning of glyphs…each stroke flows into the next, sometimes moving upward, downward, forward, or backward. And I’m wondering how to reset the vertical position back to the baseline for each word.

If I use the following as an example [I had to replace the angle brackets with parenthesis to show up in the forum]:

position cursive r (anchor 0 0) (anchor 300 300);

And type “trrr tlll” each r shifts in position properly…but then second word, tlll should start at the same vertical position as the first word does, but it doesn’t…the vertical motion of trrr affects everything after it.

I’ve tried using both anchors, tried using the first and setting the second to 0s, vice versa, I’ve even tried setting the space character to null anchors like this:

position cursive space (anchor NULL) (anchor NULL);

Nothing works. Any idea how to solve this?! Is there a “vertical reset” positioning command?

Thank you! ] chloe [

# Script and language coverage languagesystem DFLT dflt; languagesystem latn dflt;
