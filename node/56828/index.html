Yesterday I was working on an OT font and double checking some kerning. In the metrics window kerning shows up as it should. It has a generated kern feature which I can turn on in the OT preview panel.

What I noticed that was weird is kerning worked everywhere except letters kerned to the cap A in the OT preview panel. I wondered if it has something to do with A being the default first character.

Check to see if kerning to A is marked in red in the metrics window or set to zero.

2009-04-09 15:52:56 • Stephen Rapp

There is probably a check box somewhere in the preferences that says, “Show kerning with cap A in preview window” that’s not checked. (Kidding.)

Are you using class-based kerning? If so, it may be that you have “A” in more than one kerning class. (Although, usually there is a warning about this when you compile the OT features.)
