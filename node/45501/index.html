FontLab Studio does not allow the creation of AAT Layout tables (which are required for Indic fonts to work in Cocoa applications on Mac OS X), only OpenType Layout tables. Indeed, FontForge or the Apple FontTools package would be more appropriate for you. A.

2008-05-27 01:56:23 • behnam
10.5 (and every other version of Mac OS) does not support the OpenType features needed to set Gujarati (or any other Indic script) properly. It can only do that with AAT-format fonts. If you want to make a Gujarati font that is OpenType, it will only be able to be used in Windows.
2008-05-27 07:24:25 • dan_reynolds
In addition to Dan's post, I'll just add that none of the Adobe applications support Gujarati either (neither on Mac nor on Windows). So an OpenType Gujarati font can only be sensibly used in applications that use complex script-enabled OpenType Layout text engines such as Uniscribe (e.g. Word for Windows), ICU Layout (e.g. XeTeX) and, possibly, Pango (in some Linux applications). A.
