Switching between PostScript and OpenType fonts

September 4, 2003 • Bald Condensed

Since I started using the OpenType version of the Neutraface family in InDesign (this is what digital typography always was meant to be, although the OpenType features of Neutraface are VERY basic unfortunately) I constantly need to switch back to the PostScript version when I want to work in other software. For example for a congress book I'm currently designing, the cover is done in Adobe Illustrator 10 (no support for OpenType), but the book itself is in Adobe InDesign 2 (OpenType supported). I can activate only one version at the time in ATM, because it sees them as conflicting duplicates (quite logical IMO). So when I want to switch software, I have to deactivate and remove one version of the fonts and then import the other version. This is a bit tedious. Does anyone know a more elegant way for doing this? (And please don't give me any "try designing your cover in InDesign too" nonse, smartypants!)
If you dont need to work on the files outside the studio, I would see no ethical problem in taking the PostScript version into FLab (or a lesser editor) and making a new version with a different name / id?
