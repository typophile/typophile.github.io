2010-02-06 23:50:14 • Tomi from Suomi
Simplicity is useful. But I think the simplicity of taking turns working on the font would be fatally prohibitive. So I suggest we have a central, semi-public repository of primarily* font files and text files. Each font file would be named in a way that indicates what existing font it's an extension/modification of (see that other thread), and the text files (which could actually be judiciously Subject-ed emails) would mostly explain who did what. * It would also be useful to have EPS/PDF files for evaluation. So a person could look through the font files and the text files, decide where/how to jump in, and make new font/text files. And the whole time we'd be showing our work-in-progress on Typophile for feedback on which branch(es) to pursue. But this is your baby, so if you have a totally different idea that's fine, and in any case I'll help with whatever time I can find. hhp
