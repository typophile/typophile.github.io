2010-09-01 20:51:48 • riccard0

.bmap is not a Windows file, it’s a bitmap cache representation of the ‘real’ font. The AFM file is useless on its own, and so is the .lst file (that is not a part of a standard Type 1 Font distribution and can be anything).

You cannot use the .bmap font in InDesign because that needs real fonts, not just bitmaps. It might work in other programs, though – I dunno, always avoided them like the proverbial plague.

I’m wondering about the untyped file – OS X doesn’t know what it is, so it presumes it’s an ‘executable’, but you can be pretty sure it’s not one. Can you open it with, say, Textedit, and see if it contains readable data, or just a jumble of strange characters?

If it starts with something akin' to

%PS-AdobeFont 3.0 %% All rights reserved bladibladibla %% Some more stuff…

it’s the one file type I’m missing from your list, and you can try again after adding the proper .PFB extension to it.

2010-09-01 21:19:36 • Bo

It’s possible that you have a Mac PostScript Type 1 font in which the resource fork has been unintentionally stripped out of the .bmap file. Mac screen font files often have that file extension. A Mac Type 1 font can’t be used on a Mac if the screen font file is missing or corrupt.

If you “get info” on the .bmap file, what is the file size? If it’s 0k (zero k), then the .bmap file has become corrupted. This can happen when emailing or ftp’ing Mac font files without first compressing them.

zip compression sometimes strips out Mac resource forks. It depends on the specific program/version; the zip utility included with MacOS X should do fine.

Stuffit compression ([[http://www.stuffit.com/]]) handles Mac resource forks more reliably.

2010-09-01 22:26:19 • cuttlefish
Starting with Mac OS 10.3, the Finder includes compression, under the File menu as “Compress (whatever is selected)”, or if you right click on a folder or file. This will create a .zip file. If you compress a Mac file with a resource fork (such as an old style Mac screen font file), it will be preserved when the file is decompressed on another Mac. No need for StuffIt unless you or the other party are using a version of the OS older than 10.3.
