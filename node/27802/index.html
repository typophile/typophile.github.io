August 21, 2006 • gmoudry

Dear Typophile, apologies for newbie questions.

A font designed by my someone else (‘Domino’) looks pixellated at size 12 - See a screenshot here. I can see that sizes 18 and 24 are anti-aliased and look decent, but size 12 is not. Under pixellated I mean that some vertical lines are 1 pixel wide, some 2 px wide, and it looks ugly.

First I thought that perhaps some glyphs have a problem, so I used FontCreator and TypeTool2 to paste a suspect glyph ‘h’ into a known good font file (Lucida).

It seemed to work, I saved it as a new font file (‘DominoNew’) where the ‘h’ has my shape and is not pixellated like into the source TTF file.

However, when I copy another letter (‘i’) from Domino, it suddenly looks pixellated DominoNew, whereas Domino’s i was not!

My question is: does this problem (pixellated font) have a name? Under what words can I search for problem on google? How can I check if a raster image for a certain size is embedded within a TTF file? Does anyone know how to edit the TTF file to fix it? Thanks, George UK

After reading more about TTF, it seems that the problem of ‘inconsistent line widths’ lies with ‘hinting tables’ within a TTF file.
