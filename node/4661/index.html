Windows XP is handling Arabic as Unicode text, but it is possible that some of your applications are not. Question marks generally indicate that an application is unable to recognise the pasted character code.

You may also have another problem in that some of your Arabic fonts may not themselves use Unicode encoding. This would explain why changing fonts from Arial, which is a Unicode font, to another font might not work.

In order to handle Arabic in Illustrator, you would need the Middle East version of that application. The regular version does not support right-to-left text or Arabic script shaping.

Since you don't need to handle text per se, one easy option might be to make a custom font using a hacked 8-bit encoding, e.g. putting Arabic glyphs in ANSI Latin decimal positions, and then enter the characters manually in Illustrator. You could do this with the relatively cheap TypeTool.

Maybe you could just copy paste the outlines from the glyph windows into illustrator, and then just assemble the words there. You could typeset the text in Word just to see what it looks like (that is if you're not familiar with the language). You'll also need to watch out for the alignments of certain characters since some of them don't sit on the baseline.

I think this would be simpler than building a new font but I wouldn't know if you would be violating any copyright laws with this or not. You will probably need to make certain adjustments to the outlines (especially the baseline strip) if you want the text to sit on a circle.
