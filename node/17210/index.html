The simplest cross-platform way to something like that would be to use FreeType, possibly in conjuction with an image manipulation library such as the one that comes with ImageMagick.

I’ve used FreeType in the past to write a full-fledged text renderer for the TiVo Series 1 platform. My application took arbitrary input text and presented it in a “dialog box” on the user’s TV screen. Here’s a screen shot:

I had to implement some of the fancy stuff (such as word wrap, kerning, and input text parsing) myself; fortunately, FreeType makes all the necessary metrics available to the programmer. Some of the hardest parts – parsing the TrueType files, and rendering the actual glyphs – are done by FreeType itself.

Register.com’s use of JPEG compression instead of PNG or GIF is just silly. Saving the rendered image to a proper PNG file can be done pretty simply by using libpng.

I’d be happy to test your TrueType font on my TiVo setup if you want. I’ve actually been following some of your progress, and I’ve been looking forward to seeing the results in action.

Note that at the glyph level, the output will essentially be the output of the FreeType rasterizer.

The FreeType rasterizer has many internal options that can be tweaked; for my specific application, I spent a lot of time experimenting in order to determine out the combination of options that best approximated the text rendering of the native TiVo software (which, by the way, is probably using FreeType itself). My goal was seamlessness rather than optimal subjective rendering quality. That’s the reason that, for example, the tracking is too tight (in my opinion).

Therefore, if I test your font under that particular setup, it may not look as good as it possibly can. Additionally: I’m not sure what you mean when you say that your TrueType font is “special”, but if it uses esoteric TrueType features, it’s possible that FreeType will choke on it in some way. The only way to know for sure is to try it.
