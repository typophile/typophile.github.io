SeeFont for Windows [Preliminary version]

December 14, 2008 • Theunis de Jong

For your delight & entertainment: a Font Viewer. “Not another one!” I hear you cry. Well, this one I wrote especially to examine OpenType features. Drag any font onto it and it will tell you if it found an OpenType substitution table. If so, a mere press of the F5 key pops up a window where you can browse the features at leisure. It uses FreeType to display glyphs, so it should be able to load just about any file type there is; however, I’ve dismembered it to read only PFB, TTF, OTF and TTC (first font only for that one).

Download the tiny zip (282K) from [[http://www.jongware.com/binaries/seefont_1_0.zip|my website]] and start a-draggin' those fonts onto it.

A few caveats follow right after these pretty pictures.

The main screen. Move Cursor to see all glyphs, with glyph names if available, AGL names if not, unicode point if there isn’t one, and glyph id as last resort.

Press F5 to browse through OpenType features. Those numbers “#188” and so on are glyph ids (this is Calibri, which doesn’t contain names).

Press F2 to see which Unicode blocks are covered (partially) by the font.

Okay, here come the caveats.

  • It works just fine on my old Windows machine. Do not try on a Mac. I'm not sure about Vista.
  • No GPOS support. Difficult to program in. Check your kerning and spacing case somewhere else.
  • Not all substitution types are supported, specifically Contextual ones. A pity, because these are always interesting to check out, but it is hideously difficult.
  • Some fonts take ages to load. Arial Unicode, for example, takes many many seconds. Apple's LastResort.ttf takes a quarter of an hour. Really!
  • Some fonts are displayed at what seems double size, mostly CJK fonts it seems. I have ta think about that.
  • At startup it tries to locate the Windows/Fonts folder for its interface font: Calibri, if not found Arial. If neither is found, you don't have text in the main view.

The good news is you can also use it as a font browser: press Page Up and Page Down to browse the folder the last font was found in (i.e., first drag any font onto it, then browse). You can also right-click a font file once (not in the Windows' font folder), select “Open With…” and browse once for the proggy. That’ll insert it in the font type’s “Open With” list. Don’t set it as default – I can’t imagine what’ll happen. Perhaps your hard disk will be erased.

If you are browsing, you can press F4 to see a list of all supported font files in the current directory. Don’t press F1 – it does not supply help (who needs that anyway) but pops up an old dialog I forgot to take out :-). Press F3 to sort by Glyph Id; as mentioned before, press F5 for the OpenType features. Press Control+G to jump immediately to a glyph number; press Control+F to find a glyph by name.

No worries. No problem. No guarantees.
