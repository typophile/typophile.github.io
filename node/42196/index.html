CS2 applications have a bug that require at least one substitution feature in an OpenType font in order for the kerning to work. So you could add something like

feature ornm { sub bullet by bullet; } ornm;

to the OpenType panel. It doesn’t really matter so much what substitution feature the font has, but of course it would make sense if the feature would be somewhat meaningful and somehow related to your typeface.
