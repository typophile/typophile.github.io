2008-03-03 22:54:44 • Dan Gayle
I don't think Windows (or any other OS) running in Parallels has any access to the host OS or resources like fonts. It's not like Classic running on OS X. Parallels virtualizes the PC hardware, not the software running on it. You can access files from the host system, but the host system appears to the virtual machine as just another computer on the network.

2008-03-04 02:23:26 • edison
With Bootcamp, when you boot up in Windows, you cannot access any of the files from the Mac partition of your hard drive. But if you boot into Windows, you can access ALL of the files from your Windows partition. So, in Mac mode, you can move files to and from your "PC," but not vice versa. This means that if you want Mac fonts in Windows, you have to copy them to the Windows font folder from Mac OS X. However, I don't think that any of the Mac system fonts will work in Windows. Windows doesn't support AAT fonts, right? Are all Mac system fonts AAT format, or just some of them? Anyway, even if they do work, it is a big workaround to view the same fonts you use on your Mac side in your Windows side. But if you want to copy any of the PC fonts over to your Mac side, that is easy. Si, as an aside, does using the Windows fonts in Mac OS X violate the Microsoft EULA? One might assume that it would not, since both Windows and Mac OS X in this case would be installed on the same workstation.
