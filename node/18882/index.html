2006-03-30 15:25:11 • hrant
If I remember correctly, Type 3 allowed practically any PostScript function to be used, including strokes, grayscale, conditional statements--anything pretty much. This is how random fonts like Beowolf were possible. It was also used for clip art fonts with complex images. The downside: no hints, so if all you wanted to do was to make a normal font, it looked like crap on a 300 dpi laser printer. I do recall a company selling Type 3 fonts based on strokes. The advantage they touted was that you could squeeze or stretch the fonts without the kind of distortions you would get with outline fonts. Unfortunately, since the fonts were composed of unvarying strokes, the designs were limited and lacked optical corrections. I also seem to recall that the original version of Courier that was built into the original LaserWriter was stroke-based.
2006-03-30 17:39:01 • Mark Simonson
If I can add a small comment. Type 1 fonts permit stroke-based outlines. Most of the type 1 fonts have a PaintType defined to be 0, but if you set it to be 1, and define StrokeWidth to be the pen thickness (30 is thin, 100 is thick), then outlines are stroked. This can be used to thicken fonts (not recommended though!), but it can also be used to render fonts as strokes only. The change in the PaintType value and the addition of StrokeWidth parameter can be done in a standard text editor on the "pfa" version of a type 1 font. If this sounds too technical, I apologize.
