Malicious TrueType fonts

March 3, 2008 • Gus Winterbottom

[[http://heasman.blogspot.com/2008/01/sunsolve-103024-cross-browser-cross.html|This website]] describes a way to use the TrueType bytecode interpreter to attack the Java runtime environment and hence any web browser on any platform: “The bug in question allowed an attacker to craft a malicious TrueType font that could execute arbitrary native code when processed by a Java applet, thus compromising the browser.”

I wonder if this also affects TrueType-flavored OpenType fonts, TrueType fonts embedded in Word or PDF documents opened in a web browser, or the FreeType bytecode interpreter. And web embedding of fonts via CSS @font-face would seem to be dangerous until the problem is fixed.

Does anybody know if Adobe and FreeType are aware of the problem and working on fixes for their bytecode interpreters?

As the problem (reported by Sun as fixed) was described as cross-platform then we can assume the exploit relates to Java’s font rasterizer.

I’m no expert but from reading press reports I think Sun’s Java Virtual Machine (JVM) currently uses the Freetype rasterizer developed by Google’s David Turner - it used to use the T2K rasterizer developed by Sampo Kaasila (which is now owned by Bitstream) but that was replaced recently in an effort to rid the platform of proprietary software. Open Source solutions are so much more secure - except when they aren’t ;-)

So although it’s technically possible that Apple, Adobe, and Microsoft’s TrueType rasterizers (along with iType, FontFusion and T2K etc.,) have the same vulnerability, it’s unlikely.

If it’s indeed another Freetype vulnerability you may be able to use it to gain control of an iPod ;-) All UR iTunes R belong to FreeType!
