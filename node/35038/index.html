The pixel width of each character is different for each point size, it's not absolute, as font units ("Funits") are. For more info, see this thread: http://typophile.com/node/14929 and scroll down to (Adam) twardoch's first reply. Jason C
2007-07-11 20:11:25 • jasonc
The UPM size always corresponds to the pixel size on screen, and the pixel size corresponds to the point size at 72 dpi. (At other dpis you need to divide the dpi by 72 to get from point size to pixel size). For example, if a font is rendered at 20 pt with 72 dpi, then the UPM size is scaled to 20 pixels. If the font has the UPM size of 1000 units, then each pixel is 50 units large. A.

There are 6 factors that can effect the width of a glyph: 1. the glyph's width in unrounded high resolution units. 2. the resolution of the output device. 3. the point size of the output font and any other scaling that might be applied to it. 4. the presence or absence of hints and how they are interpreted. 5. the layout method used by the application composing the glyph. 6. the rendering used to draw the glyph.... To determine the pixel width of characters in certain cases, or if your pixel widths remain constant for more than four hours, consult your physician immediately. Cheers!
