2006-05-29 16:38:15 • Chris Keegan

The Designer And The Grid by Lucienne Roberts and Julia Thrift is not so bad (especially if you are just starting with grid systems), because it shows some nice designs AND the grid system used for it. Cheap (there is a paperback version available), colourful, well illustrated.

Typography: Macro- and Microaesthetics by Willi Kunz is a littlte bit more serious: Basel/Swiss style moderism, with some essential information about typography and graphic composition

Grid Systems in Graphic Design
by Josef Muller-Brockmann is a classic, because it was one of the first descriptions of a grid system and it shows the connection between the modernists ideas (clear, simple information as a prerequisite of a democratic society)
and the grid systems. However, I rather see it as a historical document, less an useful guide how to use grids nowadays.

If you want to design with other grids than a modular grid system, have a look at this book: The Typographic Grid
by Hans Rudolf Bosshard In this book you can find some quite interesting individualistic (and hence complicated) grid systems. Refreshing stuff. However, for a beginner it is easier to work with a modular grid system, I think. (Modular means the smallest units in a grid - in most cases the width of a column - are equal in size.)

Grid Systems: Principles of Organizing Type by Kimberly Elam is a cheap book, but it is even cheaper to buy it not

The way to develop the most unique grids would be to create your own I do not understand - you ALWAYS create your own grid. If you use a grid system that was designed for a different purpose and fill it with your content the result can only be miserable. You can check existing grids as a role model, but you always create the grid from the inside, from the characteristics of the main body text (leading and column width of the main body text), and probably also from the proportions of the pictures you want to use. From the leading of the main text you get the baseline grid. From the length of the text columns and the width of the picture you get an idea how many columns might be useful. Too few horizontal units - say 4 columns only - , and the layout will be stiff, too many, and the system will become too confusing for you. A tip: 12 columns are not a bad idea, cause you can divide it by 2,3 and 4 (and 6, if you need it) - but of course it always depends on the nature of the material (texts, pictures) what is appropriate.

There are a lot of designers who are doing great stuff and don’t have any idea what a grid is. But if you have to deal with complicated layouts, grid systems are necessary. (Also when you need to specify design systems for other people.) Probably you can compare it with music: As long as you play together with just a few other guys, you don’t need to know about composition techniques - your ear will be more important. There are a lot of great musicians and song writers who do not know how to read musical notation. But if you want to write music for big orchestras, you’ll have to learn composition techniques.
