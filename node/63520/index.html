Getting the text formatted into ID is one thing, making it stick is another. Tiff, your local overrides will show the dreaded ‘+’ appended to these paragraphs, and if you ever do something like a search-and-replace using a paragraph style, they will all be flattened to its base definition.

The recommended way to make bold, italics, and so on “sticky” is by defining them as character styles. These won’t disappear with an override; and you can easily change the definitions, for example, from bold to semi-bold, without touching anything else.

Replacing every possible combination of text attributes with an appropriate style is a bore, especially since ID does not allow nesting of character styles (in particular, you might need a superscript, superscript italic, superscript bold, small caps italic, … endless possibilities…). So I wrote a script to create them on the fly. It got mentioned on InDesignSecrets! Read (well, listen to) [[http://indesignsecrets.com/indesignsecrets-podcast-108.php|podcast 108]] for all details.

Gregers,

Your bullets disappearing happened because they were added in the paragraph style in Word. InDesign also allows bullets as part of paragraph styles, but if you override the style they will disappear. Not visible in Tiffany’s screen shot (version?), but the Word options should also include sth like “Convert Bullets and Numbering to Text” – use this if you don’t use ID’s own bulleted styles.

The script can be downloaded directly from [[http://www.jongware.com/binaries/preptext.zip|my site]]. Unzip, copy into your Scripts folder. With a right-click on any script in your scripts panel you can select “Reveal in Explorer/Finder”; copy the script into there.

A more recent podcast discusses various ways of cleaning up Word before/after importing – well worth a listening as well.

On the subject of Discipline:

Before I run my own script, I always check first what the Find Fonts list says. The script doesn’t behave properly if a font’s styles are not called plain “Bold” or “Italic” – it’ll fail miserably on “Light Italic”, “56 Italic”, “Bold Condensed”, and such. I replace those fonts with Times New Roman analogs. It also lets me check up on mis-used Symbol (since most fonts I use have their own Greek characters) and kill off any stray Wingdings arrows. And then I’m just starting! Next is to check each and every possible text attribute for mis-use – I hate superscript spaces in the middle of a regular paragraph, italics that include its trailing space, paragraph returns that are formatted with bold and underline, less-than with an underline standing in for less-or-equal, plus with an underline posing as plus-minus.

Cleanliness … Oh, and then I run my script.
