What do you mean the kerning is lost? If you are generating PFB files, also generate an .AFM file. You can open that with almost any text editor, and I'll bet the KPX commands are there. If you are using only InDesign as your applications program, there is a bug in InDesign that has been covered many, many times on this forum. For OpenType fonts, make sure you have at least one character substitution feature so that a GSUB table is built, build your kerning tables, compile the features, and generate an OpenType font. If there is no GSUB table, kerning won't work in ID through version CS2. They say that is fixed with CS3.
