Fontlab tip ( minor one ) for feature scripting/testing

December 3, 2005 • ebensorkin

As tips go it’s a minor one, but…

I have come up with a way of working through & testing my Calt feature script & keeping track of where I am. I put a space in the code and move lines code up or down to indicate if I am done with them (depending on the direction I am moving throgh the list). Happily you can change code & recompile with the space there & it doesn’t break anything.

I don’t have 5.0 yet so I can’t say if this will work in that version.

Anybody else have practical/process tips to share?

Or a somehow better ways of doing this?

That’s a handy tip, actually. Sounds good to me. It will work fine in 5.0 as well (the properties of whitespace in the feature code have to do with the Adobe-defined syntax for that code, picked up by Fontlab).

I often put comments at the beginning of a feature to indicate exactly what I still need to do with it. Because I’m only doing about 10-15 hours a week on type design, it’s the only way I can keep track sometimes.

2005-12-03 05:26:04 • Thomas Phinney

You can use the “#” character at the beginning of a line to mark the whole line as a comment, or at the end of a code line. For example:

More swashy glyphs

sub @a by @aswsh;

or

sub @a by @aswsh; # More swashy glyphs

Okay, it wasn’t too hard to do a quick-and-dirty module for BBEdit after all.

http://www.marksimonson.com/otfeature_module.plist.zip

Unzip it if it doesn’t happen automatically, quit BBEdit if it’s running, and put the the file into ~/Library/Application Support/BBEdit/Language Modules/ . (If you don’t already have a folder named “Language Modules” in there, make one.)

Now when you open a .fea file in BBEdit, you should see some basic syntax coloring. The colors will depend on your preference settings. Nothing fancy, and some character names (like zero) are also feature tags, so they will be highlighted. There is probably some way to fix this, but, as I say, this is a quick-and-dirty module. If anybody gets ambitious enough to improve it, be my guest. But let me know, too.
