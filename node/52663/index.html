font rental application (dreaming of)

December 14, 2008 • dogmatheque

Hello Guys,

i’ve been wondering about how a font rental system could work nowadays. Here’s how i see things. i’m curious to have your feeback.

1/INSTALL You install the free font software, let’s say something like fontexplorer or font agent pro.

2/REGISTER You create an account to access the online library.

3/CHOOSE FONTS You choose your rental options. Which foundries do you want to have access to. Which packages. How many months (weeks?) do you want to subscribe. Of course according to the packages chosen and so on, the price would change.

4/ ALL SET I’m ready to go with quite a few fonts for a start. Besides i’m legal.

How would this work?

As long as you’re connected to the internet you can use the fonts on your system, as if they were locally installed. Once you’re offline, the fonts are considered deactivated and you cannot use them anymore.

What are the advantages of that system?

1/ Promote typography / Less piracy

a/ This solves the problem of paying something before being sure you can use it. Thus encouraging new typographic experiment and variety.

b/ Young designers will have equal chances with bigger companies with bigger budgets and bigger fonts libraries.

c/ You can still buy the full set if you want to work offline and use them on a regular basis (in the end it remains the most convenient option).

d/ Some people have never paid and will never pay for their fonts, but i guess that a starting fee of 15$ (or 12€) with access to a decent library would make people think twice.

e/ Fair sharing of the font for the people involved in the workflow.

For example : I’m working on a website. We make an introduction video with the font. The site is partly in flash and we use the font in it. We’ll send flyers and documents using the font too.

So who has to pay for it? I guess it’s unfair to make the flasher pay (but it’s illegal not to pay for the font in most cases as far as i know) Same for the video guy. Well for the print we can still make PDF, but in some cases the print guys does the last minute text corrections… so it can happen that this guy needs the font as well.

Some kind of approval system with a time limit could ease everyone’s task.

Like that : i’ve got the rental agreement for a font package. With that i have a code that i can give up to five people so they can access the font to work with. Once the job is done, you clear them of your list and they cannot access the fonts anymore.

I’m done, i guess you got the picture. In an ideal world, this application would only take a few month to become real. Everything that we need is already here. What do you think? Anybody interested in making this happen?

2008-12-14 23:09:35 • E. Scrooge

I’ve spent a great deal of time thinking about systems like this, and they aren’t workable.

First, how does one handle sending the font to the printer? You can’t just convert everything to outlines. But if the user can just send a font to the printer, the user can just keep the font and use it over and over. Sending a PDF doesn’t work for everything.

Second, this is a DRM-based system. It has to be, otherwise nobody even needs to use torrent sites to get complete foundry collections, they just need to get the crack for the rental system. But end-users hate DRM systems, and a system this heavyhanded will irritate the hell out of people.

And what you’re suggesting is DRM that works with all software across platforms. To make the DRM system work across apps software and/or the OS will need to play along. Adobe, small software developers, and open-source developers don’t want font DRM. Microsoft loves DRM, but will inevitably screw it up or try to turn it into a monetized standard. The DRM issue is a disaster all around.

The best option along these lines is an honor-system that allows users to buy single-use licenses. That’s not going to happen because font designers have already learned that about half the graphic designers in the USA are font pirates, and the number is much higher in developing nations. So designers have no reason to assume that the users are really trustworthy.

2008-12-15 00:59:29 • dogmatheque

All right. Let’s take into consideration such kind of system.

First, support of major OS’es. In Windows XP it can be somehow done, but it requires a decryption mechanism for using them locally. Encryption adds one more API level of abstraction, which leads us to… well, a whole new heap of versions of any graphics software. Because if decryption is done on OS level, it is obvously bypassable. And if it has to be done in specific applications, they need to be rewritten to take this level of abstraction into account.

Note that for Vista such scenario (like injecting plugins for such encrypted fonts) is next to impossible, at least without Microsoft incorporating this technique in WPF. Otherwise, the user will be haunted by UAC messages everytime he tries to use such fonts. Or the users need to deactivate UAC (not always possible), or use graphics software with Administrator privileges. Too many problems to sort out, with too few real advantages to justify the effort.

About Mac I have serious doubts this can be done without OS integration, which even if Apple agrees to add, leads us to a serious problem: what should the users with outdated OSes do? Upgrade or get lost? I am by no means a Mac OS X programmer, but still have doubts such changes in OS architecture are justified.

And last but not least, Linux (for the sake of GIMP and Scribus users). OS integration is possible, but reversable (open source, anyone?). Same goes for app-specific modules; if the app is opensource, it is reversable, and if it is not, users are forced to use specific proprietary software, which somehow defeats the purpose.

To put the long story short: such DRM systems are hard to implement system-wide, and most users simply refuse to use them, avoiding the manufacturer altogether. The most obvious example is Chuck Davis’s Letterhead DRM implementation, which forced most users not to use his typefaces. If not extremely shortsighted, I do not expect any foundry to pay big bucks to develop a rental system, which brings less income than the present sales system, and drives the users away.

Just my 2¢.

2008-12-17 09:03:43 • James Arboghast

Here’s another approach that I’ve thought about from time to time and wondered if it might work:

This is based on the idea of the way that the old type houses used to work in the days before computers were everywhere. In those day, if you needed some display type for magazine titles, book headings, advertisements, etc., you would send a type house a list of the text you needed with specs for the font and font size; the type house would set the desired text and proof it; and then deliver the proofs to you to cut apart and paste onto the final artwork for your project. (If you needed to adjust the letterspacing/kerning of the type, you cut the individual letters apart and letterspaced them yourself.)

The updated version of this would work something like this: An online typesetting company is set up – let’s call it “NewTypeHouse”. They buy a LOT of display fonts (not cheap, but doable for a business). I e-mail NewTypeHouse with the display text I need along with the specs for it. They copy and paste the text into their in-house typesetting program, and then convert it into outlines and save it as an Illustrator file. Then they e-mail me the file and charge my account. I open the file in Illustrator, adjust it further, if needed, to meet my needs, and then insert the outlines into my project.

I claim no expertise in the legal side of these things, but my understanding is that once NewTypeHouse converts the type to outlines, it ceases to exist as type, and the company would be no longer bound by the typeface license. Likewise, my understanding is that one could not use the outline art received from NewTypeHouse to reconstitute the typeface (except perhaps by cutting and pasting individual letters into FontLab, which would be so much work that I doubt that it would be considered worth the effort by the type buyer.) But perhaps I am wrong about this. I would be interested in others’ thoughts about this approach.            ~~~~~~~~~~~~~~~~~          When going from A to Z,          I often end up At Oz.
