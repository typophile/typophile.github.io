Here is the KA dialog box for an all caps font which has duplicate caps in the lowercase positions:

[img:sites/default/files/old-images/ FontographerScreenSnapz001.png]

(This is in FOG 4.7 (demo), but it works the same in any FOG 4.0 or later. I don’t know why there are some strange characters in some of the rows. Just ignore them.)

Note that in the first field are a bunch of A’s, both upper and lowercase. The design of these characters is identical for kerning purposes, so they are “equivalent first letters”, the left-hand letter in a kerning pair. The field the two T’s is another set of equivalent characters for the right-hand position. Don’t worry about the “link all” check boxes for now.

What this first row says is that every pair composed of a character from the left and one from the right will have the same kerning value. If you change the kerning value of any of these pairs (using the Metrics window), the others will change automatically, as if they are all marching in lock step. So, if you change AT, you are also changing At, aT, at, áT, Åt, etc. The reverse is also true.

The “link all” check boxes let you take a shortcut in certain situations. Let’s say you have the following pair equivalents set up:

F    [x]    AÅÁÀÃ   [x] P    [x]    AÅÁÀÃ   [x] T    [x]    AÅÁÀÃ   [x]

(Pretend the x’s are the check boxes.) F followed by any of those As will have the same kerning value. Same with the P and T. This is a bit redundant, and it would seem that you need to do it this way unless you want FA, PA, and TA to have the same kerning value. BUT, by unchecking one of the “link all” boxes you can combine these into one row:

FPT    [  ]    AÅÁÀÃ   [x]

The F, P, and T are not linked to each other and will have independent kerning values with regard to the As. So, if you change the FA kerning pair, the FÅ pair will also change, but not PA, PÅ, TA, TÅ, etc. The limitation is that it’s an all or nothing proposition. If you want the F and T to be linked, but not the P, then you can’t put them in the same row. You would need one row with the F and T, and another row for the P.

Note: At least one of the two check boxes should always be checked. If both boxes are unchecked, then nothing in that row will be linked, and whatever you put in that row will have no effect on the kerning, as if the row did not exist.

Normally, you will want both boxes checked and not bother with this unlinking option. It just makes things more complicated and less flexible for very little gain.

Also, normally you will leave the “make new pairs” option checked. If it is off, KA will only affect existing kerning pairs and not generate new pairs, which is usually why you use KA in the first place. There are, however, situations in which you want to, say, clean up inconsistent kerning pairs without generating new ones. Also, using KA with “make new pairs” on can potentially generate a lot of pairs which will never be used in the real world. After you’ve used KA to generate all the pairs you need, you can turn this option off and delete the extraneous pairs. The pairs that are left will still be linked.

One other thing: Certain characters (single-straight-quote, exclamation, comma and hyphen) have special meanings in the KA window. As a result, you can’t include them in character lists just by typing them. There is a simple way around it: Just use a single quote before the character ('' or ‘- or ‘, or ‘!). You can see an example of this in the third row, second field (’,). [edit: Typophile’s formatting genie is messing with my straight quotes.]

A further note about the special characters:

You may notice this line in the screen shot above:

AaÄÅ á-ã ÀÃÂÁ

This was originally entered something like this:

AÄÅÀÃÂÁaááåäâã

When you click “OK”, FOG rearranges the characters in ASCII order and compresses the list (if possible) using the special hyphen character. The hyphen means “through”. The run of a’s with accents I typed happen to be consecutive characters in the Mac Roman encoding when listed in ASCII order. The spaces separate the shorthand notation from the rest of the list.

The straight quote is the escape character. I’m not sure what the comma or exclamation point are used for.
