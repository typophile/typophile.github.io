2005-03-21 16:32:30 • Miss Tiffany
Well, I can't give you any specifics, but I've always been attracted to forms that take into account visual entry and information grouping. This also creates opportunities to go beyond type and rules.

Visual entry, how the person sees the form within the first second or two and sorts the visual data into information about how to use the form, is something that often gets overlooked. What's the at-a-glance impression? Then, start examining each bit individually and ask yourself what's really needed. Also, use a clear, open, and high-legibility font so you can use smaller type without fear. I agree with Darrel as well.

And semi-related, check out Airbag's redesign of an intellgence brief to see how strong, simple, and purposeful visual hierarchy can inject a lot into a document that doesn't have much to begin with.

James, not only do forms need to work in the collection of data, but they also need to work in the extraction and entry of the data into some kind of database (presumabaly) or other persistent record.

So part of your considerations should also be the workflow of the clerical task: what order does the software require the information in? Is the interface something that requires mouse clicks or is tabbing/shift-tabbing possible? Is the software changeable by you or a developer to improve this work? And if it's not going into a database in a computer perhaps the record book or file could benefit from your typographical design expertise as well.
