I used Veenix Font Tools to great success for printing out roughly 1,400 fonts for use as sample books at a college art department. You can print 10/page (two lines), 20/page (1 line), 40/page (1 line, 2 columns), cascade (6 pt. through 44 pt.), smaller cascade + body sample (at 10/12), and an ASCII reference sheet. You can customize what the lines say, too. It handled opentype, postscript, and truetype fonts equally well. My procedure:

  1. Deactivate all fonts possible in FontExplorer X.
  2. Activate Veenix. Categorize all remaining active fonts as “System” or whatever tag works for you. (create the tag first, then go to Actions→Categorize All Fonts in Current Sort As→)
  3. Group fonts into historical/stylistic categories in FE X if you haven't. I would do this if you have more than a couple hundred fonts. Veenix leaks a bit and will start making mistakes after parsing several hundred.
  4. Activate only the set I wanted to print in FE X, deactivate all other fonts.
  5. Restart Veenix.
  6. Flush the font database in Veenix (command-K) which forces it to only index the active fonts.
  7. In the Current Sort option, select "Uncategorized fonts" (to filter out system fonts).
  8. Open the type book creator in Veenix (command-6).
  9. Set it up as needed.
  10. Click print all.
  11. Instead of printing, save as .pdf.
  12. Print .pdfs via Acrobat.
All in all, it went pretty smoothly. I customized my two lines (I used the 10 fonts per page option) and separated the fonts into three largish books: Serif, Sans, Display/etc. Each book had subcategories: Serif had Venetian, Garalde, Transitional, Didone, Egyptian. Sans had humanist, grotesque, geometric. Display/etc. had Headlines, Blackletter, Handwritten, Brushscript, Calligraphic Script, Chiseled Stroke, Non-Western, Pi/Symbol, and Misc.

Perhaps this is way too much info, but perhaps it will help you too… I hope the latter.
