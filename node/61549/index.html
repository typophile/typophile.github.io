Problem with windows in Fontlab

September 1, 2009 • milne.ross

I’m running Fontlab with two monitors and have noticed previously that all windows not recognized as proper windows by Mac OSX (ie: macros, opentype, classes, preview) do not default back to the main monitor when the second is disconnected. When this happens, these windows become totally inaccessible. I have found that reconnecting the second monitor and manually dragging the windows back to the main monitor solves the problem.

Still with me?

Recently, a problem has arisen where all of these menus have become lost in monitor land. I’ve tried a bunch of things but nothing allows me to access them again. As you can imagine this is becoming a real problem. And no, restarting Fontlab does not fix the problem.

Has anyone encountered a similar problem? Anyone know of a fix to this issue?

I apologize if this is vague. Trying to find a succinct way to describe this strange problem proved as difficult as the problem itself.

There are panels positions stored in the workplace setings:

~/Library/Application Support/FontLab/Studio 5/Workspaces/CurrentWorkspace.rc

Maybe just deleting this file will fix it. If not, some panel and window positions are in:

~/Library/Preferences/com.fontlab.fontlab5.plist

Just delete the pref file, and recreate your prefs.
