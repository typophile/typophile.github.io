2008-09-04 01:18:02 • rosem
When a text file displays Hebrew as question marks, this means the text is not not recognized. The customer's program that saved the text file exported the Hebrew text data as Unicode encoding for Hebrew, but in a file format (i.e. .txt) that does not support Unicode. The customer must resend the file to you in a different file format. The customer must resend the file to you taking into consideration two factors: 1. The file format must support Unicode. 2. You have a program that can read the file or import it correctly. .doc from MicroSoft is a very good file format to transfer Hebrew text, as most people have MicroSoft Word XP or later. .doc files from MicroSoft Word XP or later support both Unicode encoding and Hebrew. After you finally get the file, to create an animated gif or a .jpg with the Hebrew text, it must appear right-to-left. You can use Adobe PhotoShop ME (it must be a ME -Middle East- edition). Contact www.fontworld.com for more information.
2008-09-04 01:49:17 • gohebrew
The ISO-8859-8 hebrew encoding is still in use (I regularly receive emails with that encoding). If you have an ISO-8859-8 encoded file and need a utf8 input, you can use the free recode facility (that is a Unix command but there are non unix ports). I installed recode on my mac and if file.txt is ISO-8859-8 encoded, then the command recode hebrew..utf8 file.txt results in file.txt being utf8 encoded (as you see, recode overwrites). To see all the encodings that recode can handle, type recode --list in a shell window. Michel
