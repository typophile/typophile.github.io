Everson Mono & InDesign

January 2, 2008 • Frode Bo Helland

I’m having problems using Everson Mono with Adobe InDesign. Only a few glyphs show up. (I’m on a Mac, if that’s relevant.) Using it with Illustrator works like a charm, but due to some wierd reason InDesign won’t play along. I contacted the designer, Michael Everson, who told me Adobe apps in general aren’t to happy about Unicode, but didn’t bring me any closer to a solution. Does anyone know what the exact problem might be, or even better, how I can get it working properly?
InDesign might not yet support all aspects of the Unicode Standard, but I’m not sure that’s what’s causing the problem. Running the font through MS [[Font Validator]] outputs a few errors that seem to be serious (image below), specially the one that says that “The subtable length is zero”. The font, BTW, has 4899 glyphs (according to FLS5, at least).

Indeed, with Illustrator (mac) no problem; InDesign CS3 (mac) + InDesign CS2 (windows) = problem.
2008-01-02 22:59:21 • david h

Word (Mac) doesn’t like the font either.

The problem is definitely a font bug: Miguel got one of our engineers to do a big of spelunking, and he found that there is an error in the header of the 32 bit Unicode cmap subtable. The length of this subtable is set to 0, even though the rest of the subtable contains valid data. A variety of programs fail to deal well with this error, while many others are just fine.

That being said, it would be nice if InDesign dealt more gracefully with the font bug than it does, so Miguel logged a bug on the InDesign side as well.
