2011-12-02 00:28:15 • dezcom
Okay, this is interesting. The 'Unicode Data' provided by InDesign is misleading, if that is truly associated with the sign following the tau. U+0384 is not the COMBINING ACUTE ACCENT. U+0384 is the spacing GREEK TONOS character, which was encoded for backwards compatibility purposes to enable encoding of uppercase diacritics by entering the spacing tonos before the capital, e.g. ΄E. [This encoding is not encouraged.] Now, U+0384 has a compatibility decomposition to 0020 0301, i.e. the space character followed by a combing acute accent. So it is possible that InDesign is reporting Unicode Data for this decomposition, which is why you are seeing it reported as COMBINING ACUTE ACCENT (U+0301). As to how this ended up in the text, it seems likely that some Greek users mistype the spacing tonos instead of the apostrophe in the same way that some English users type ` instead of '.
