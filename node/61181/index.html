Trouble with numbers

August 20, 2009 • typeluv

I copied a piece of text with numbers from a file since I wanted to use the font to create another set of numbers. The trouble is when I type the new numbers, they look different (“2”) and the numbers fall below the baseline (“7”).

Bearing in mind it is the same font with exactly the same properties (baseline shift, character styles, paragraph settings, etc.) I cant seem to understand the problem.

I can use the original text of numbers to create the new set of numbers but I want to understand what the problem is. Please help!!!

In the image, the numbers above is the result and the numbers below is the source from where I copied the numbers

Font: Prelo Sans Light 7pt

You are using an OpenType font with oldstyle numerals enabled (the top example). These are often used in running text, since they tend to blend in better. The bottom example would be called lining numerals, in which the numbers are all the same height. More and more typefaces nowadays include these two sets of numerals and more.

Switching from one set to the other depends on the application you are using. If you are in InDesign, in the character palette click the flyout menu, then OpenType, and then (at the bottom) select the numeral style you would like.

Read this article on I Love Typography for more information.
