Greek terminological question

December 6, 2007 • charles ellertson

I suppose this is silly, but with all the programmers around, it seems important to get the terminology correct.

So, using Fontlab Studio 5.0.4, when I enter the correct Unicode index for the terminal sigma in Greek (U+03C2), and hit the little green diamond by the “name”, I get “sigma1” for the name. I had always termed it “sigma.terminal”. Now I know at some level the “name” doesn’t matter, but it seems wise to get it right.

The same thing happens with the script theta (U+03D1) and alternate phi (U+03D5) – FontLab names them “theta1” and “phi1”.

There is also a terminal iota (used in some ligatures), which I’ve named “iota.terminal”, which would seem at variance with calling the terminal sigma “sigma1” – should it be “iota1”?

What’s going on is that we’re making up, for our own use, an OpenType version of Wilson Greek (with Matthew Carter’s permission). Since there are a lot of ligatures in Wilson Greek, it would be nice to get the names consistent so I can figure out what in the world I’ve done wrong as I go back & fix all the mistakes I make in the features code.

& BTW, just from a historical perspective, since this Greek is more of a script, I wonder why the script theta isn’t the “default.”? I notice the Greek Font Society used Wilson Greek in their English volume, but used the U+03B8 theta.

I’m putting a lot of the ligatures in Stylistic sets; Carter indicated that the ligatures that were used originally were the alpha_iota combinations, alpha_upsilon combinations, alpha-final sigma, and epsilon-iota combinations at the beginning of words.

Following an idea of Nick Shinn’s, lower-numbered stylistic sets will have these; higher numbered sets the less-used ligatures.

Thanks in advance,

Charles

Names like sigma1, theta1 and phi1 are old Adobe Glyph List names, which means that they are mapped to the corresponding Unicode codepoints by Acrobat glyph name parsing, but you can also use uniXXXX form names for the same result.

I usually call the terminal sigma ‘sigmafinal’ during development, and switch to ‘uni03C2’ for font generation. ‘sigma.terminal’ is not a good idea, at least for font generation, because Acrobat will parse that as the regular sigma character, not the final form, since it treats the dot as indicating a glyph variant.

The final sigma is a distinct character in Greek text. But the variant forms of theta and pi, etc. that are included in Unicode are symbol characters, for mathematics etc., and not intended for use in regular Greek text. So my advice is to treat these forms as glyph variants, accessible via discretionary or contextual OpenType Layout features. In that case, you would name them e.g. theta.alt, phi.alt, etc.

For ligatures, use the _ convention, e.g. delta_alpha_uni03C2.

2007-12-07 11:05:26 • charles ellertson

As to the theta: Well, yes, the script theta is used in mathematics, but there was a time when it was the favored for text

Yes, I’m not saying that the form should not be used, but that it should be encoded as the theta letter character code, not as the symbol character code, and any variation in the form should be handled at the OpenType Layout level, not at the encoding level.

According to Yannis Haralambous, there is a tradition in French publishing of Greek to use the open (scribal) theta at the beginning of a word and the closed form within a word. As far as I can tell from my analysis of older Greek books (e.g. Aldine or Stephani publications) the forms are mixed pretty freely. Indeed, I wonder if they might have been mixed in the same case compartment and the typesetter just used whichever he grabbed.
