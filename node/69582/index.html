Since leading is something that is set in applications like Photoshop by the user, you have only limited control over this in the font. The best thing to do is to scale your border elements so that they are correctly spaced when the font is ‘solid set’, i.e. when there is no leading between the lines, e.g. 20pt text size on 20pt leading. This is the only predictable basis on which to work. What this means, in FontLab, is that your border elements should be equal to the UPM height of the font.

For applications that make use of font metrics to define default linespacing, you will want to set the OS/2 Typo and Win metrics values to also sum to the UPM value (treating negative descender values as positive for this calculation).

You can either sit the border elements on the baseline and have all the height above the baseline, or you can give them some descender space. This decision depends on whether and how you want them to align to other fonts.

As an example, presuming your fonts are on a 1000 UPM height. You might scale the border elements to close to that height, and sit them 300 units below the baseline. Then you would set the OS/2 metrics as follows:

TypoAscender = 700 TypoDescender = -300 TypoLinegap = 0 WinAscent = 700 WinDescent = -300

The hhea metrics should match the corresponding OS/2 Typo values.

Of course, then you have to communicate to the user that these fonts should be set solid in programs like Photoshop.

This is the approach we took with Restraint.
