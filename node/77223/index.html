2010-12-15 03:37:24 • WeAreLoa

You could look into a server based/browser based manager. That would allow users to still have system and some application fonts on their computers, but the main collection is all on one server. You can create specific user groups, manage licenses, etc.

When I worked for a large studio we used Font Reserve (that had problems) and then switched to Universal Type Server (UTS) by Extensis. UTS seemed to perform well once you got used to it. Fonts with limited seat licenses can be restricted to specific groups. Font Reserve, also by Extensis, I had problems with and wouldn’t personally recommend.

For myself at home I use Font Explorer also.

2010-12-15 05:15:23 • Stephen Rapp

Be careful with Font Doctor and all those fonts. An Extensis rep came to my former workplace and did a font migration of our Font Reserve server to a new one dedicated to UTS using Font Doctor. It was a complete disaster.

As a preface I should mention a big problem Font Reserve had. Font Reserve would allow users to manage fonts on their own computers as it well should. Only problem was it was also copying these fonts onto the server. I was not managing fonts when that was all set up, but started about the time we got UTS. So when I went to go through the new migrated collection instead of a couple thousand fonts there were close to 10.000. More than half were unlicensed fonts pulled from users computers that were either corrupt or PS orphans. Font Doctor also merged versions of fonts with the same family name even if they were from different sources. I spent way too much time fixing that mess. The only good out of all of that was I got a large studio to finally pay attention to licensing.

2010-12-15 22:01:49 • Thomas Phinney

re FontCase: Although the program has been a one-person endeavor it has the backing of FontShop International nowadays (they assisted with the plug-ins if I recall correctly).

My program of choice is FontAgentPro, which is simple yet powerful and well maintained by the developer (example in case: they are quite forward with putting out new versions of plug-ins for new CS-versions, which counts in my view as a major plus). For a single person shop it is just the thing, imo.

My tip for your friend/client: Backup all the font files onto a external drive. Erase from the main computer. Install just the necessary ones and the ones that come with Adobe apps. Search and install fonts as and when needed, preferably OS-format. And always test before giving newly installed fonts the right to live on.

I did the same a couple of years ago and it’s been plain sailing ever since… : )

(I also have tried FontDoctor to sort out things, but it is — or was — quite finicky about certain characteristics which I never could place in context; maybe a misplaced byte by a font designer or such… The result: loads of font files classified as Reject/Repair and such, blah!)
