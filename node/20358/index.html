Earlier versions of FontLab (2.5 and 3.0 I think) had UI support for PS subroutines, which meant you could define a section of a glyph as a subroutine and then apply it an other glyphs. If you edited the original subroutine source, the changes would be applied to other glyphs that referenced the same subroutine. Subroutines are a PS-specific function, so I think they were most likely dropped from the FontLab UI when the program began dealing with TrueType in a serious way. Now 'subroutinisation' is something that happens on the fly when a CFF font is generated, i.e. it is applied after the design work as a font size efficiency measure, rather than as a design tool.
