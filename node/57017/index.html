I work in journals production so get a lot of PDFs to check, of work done by our typesetting colleagues on another continent. Some of this was done in InDesign, and some in 3B2 (version 9). I often have trouble with them failing to set characters beyond the basic Latin alphabet (eg, accented characters such as Å, and even bog-standard non-alphanumerics such as the © symbol), and have queried this with them. For example, I’ve seen a regular A kerned onto a degree symbol (yes, not even a floating ring-diacritic, but a °), or a lowercase c kerned onto a circle taken out of some symbol font.

The response I have received is kinda hard to interpret, partly because it’s techie but also because I suspect it’s non-native-speaker English.

So if there’s anyone on here who’s really conversant with 3B2, please let me know, and then I can show you more detail and see if you think their response is reasonable…

Thanks in advance,

As far as I ever got in understanding 3B2, it uses a font format of its own – much like (LA)TeX. PostScript fonts had to go through a conversion phase into the proprietary format, before the program could use them.

As with TeX fonts, it’s not unusual to see “original” solutions to accented characters, unhindered by any standard such as Unicode.

I found a recent PDF created w/3B2 (v8.07f/W).

Adobe’s Digital Printing Preflight reports nothing unusual – some images in RGB. Some lines as well, although they appear on screen in black. And, come to think of it, all text is in RGB as well, leading to tons of warnings and absurd CMYK values. Ptoei!

The fonts have interesting names: AdvGreekM, AdvMT_SY, AdvPi1, etc., none of them has a recognizable regular font name. The plain text font, however, clearly is MS Trebuchet.

I see a single ‘©’, but copying yields an ampersand. So no Unicode, but it is a real glyph, not a manually faked one. Copying an ‘=’ gives a ‘¼’. A degree gives ‘8’. Well, perhaps those are symbol fonts.

No trademark sign–superscript ‘TM’ instead. No plusminus but underlined ‘+’. Beta’s have been slanted manually to fake italics, and they’re not from Trebuchet.

All in all, yeah, some surprises. But at this level it’s hard to differentiate between (possibly) program limitations and (probably) operator stoopidities.

Hi,

yes your correct that originaly 3B2 used converted type 1 postscript fonts in its own fnt format to enable all it clever stuff with character setting. Version 8 brough truetype font support and 9+ added open type support.

Pre unicode the basic character set was 256 ISOLAT1 using the font table as set in the users own font. Unicode came in for version 7+ but the fear of change and lack of knowledge and availability of unicode fonts, and the fact it would change behaviours of kerning and ligatures (therefore line endings) made few typesetters want to change over existing templates. New templates however should have made this a breeze! :)
