Where's the kerning in Arno Pro?

November 15, 2007 • Harbs

A colleague of mine owns Arno Pro. He’s doing work which contains transliterations, and needs an h with a dot above. He wanted to modify his copy of Arno Pro and add it in. When he opened the font in fontlab, he discovered that none of the kerning pairs show up, and when it gets saved again, all the kerning (and who knows what else) gets stripped from the font.

I didn’t quite believe him, so I took a look myself, and sure enough, FontLab shows no kerning. Not in the metrics window, and not in the OpenType window. (The feature shows up, but there’s no lookups!)

Is Adobe using some new kind of system?

Take pity on an old print guy. The only way I know to do this is to set something in InDesign, export a PDF file, open that up in Photoshop, then save the file as a GIF so I can post it. That is a lot of work.

I don’t even know how to control the quotes in this forum. Consider:

“‘Oh’ he said ‘don’t do that.'”

Now typographically, that is rendered

/quotedblleft{}/quoteleft{}Oh/quoteright{}/space{}/quoteleft{}don’t do that/period{}/quoteright{}/quotedblright

But that’s not what gets displayed.

In any case, in all the Adobe fonts I know, there is no kern between /quotedblleft{}/quoteleft{}, and there should be – anything from about 75/1000 to 150/1000, depending on the font & word space value. The same between /quoteright{}/quotedblright{}. And in the UK, those sequences are reversed, but the kerning should still exist between the two quotemarks.

/quotedblleft{}/quoteright should also have a kern, as in

“‘Tis of the I sing.”

For a different example, using Warnock Pro Light, set the string A/quoteright{}/comma{}. The kerning is so tight the comma winds up looking like some strange diacritic on the “A”.
