I second Eben’s suggestion. I almost grabbed my defibrillator. ; )

For a quick fix, you could try “correct path direction” on everything (select all at once), and maybe they’ll show up. Otherwise I think you can select a preference to show open path ends, which are allowed in Illustrator, but not in FOG.

Though it may not be very satisfying in solving this issue, Sally, I suggest something else: In the future, don’t use Illustrator to digitize your type; use the type drawing tools built into Fontographer. Fontographer always had a difficult time importing EPS files, and there are pitfalls that may not be apparent: FOG scales outlines at different percentages on import, and there’s this issue with path integrity that you’ve encountered.

Illustrator’s tools are not as useful or intuitive as FOG’s for drawing type. Even if you’re used to using Illustrator, I suggest you only use FOG for digitizing type. Since you already use Illustrator, it’s a tiny learning curve, and it saves a bunch of time. You can also put scans in the background and autotrace or hand-trace them in FOG, so there’s no loss of functionality. You can trust FOG.

One other note: Sally, please note the difference between spacing (the permanent, built-in sidebearings of each letter) and kerning (the additional tightening or loosening of specific letter pairs to compensate for gaps in default spacing). Space the whole font completely, and when that is done, THEN add kerning, only where it’s needed. Kerning is only meant to fix problems that are unavoidable when the spacing is complete, such as LT, AV, or To.

For tutorials on spacing a font, see Briem, Underware, or Anatomy of a Typeface.
