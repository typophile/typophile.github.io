The future of typography on handheld devices?

June 26, 2007 • TBiddy

I was recently flipping through the type issue of Step Inside Design magazine, where Jared Benson offered his expertise on the subject. Looking at the case studies, and with the iPhone being introduced this week…it had me thinking…

What will the future of typography on cell phones and handheld devices be like?

Are higher resolution handheld displays changing how people think about designing type for these devices?

(* Note: Please do not use this thread as an excuse to rant about how much you may hate the iPhone.)

It’s a good open question. I’ll help the discuss with a breakdown of the 3 types of instances where it will show up (which loosely defines the players responsible):

1- Mobile phone UI. The UI that’s built into the handset (as with Punchcut’s Qualcomm Sans in the article you mentioned). // Falls to the manufacturers and carriers.

2- Mobile web sites (like what you will see when using a mobile browser (Opera and Opera Mini, IE for Windows Mobile, Safari Mobile Version, etc., etc., etc., The list is long, tedious and the results ugly.) // Falls to the web site and content owners.

3- Fonts embedded into “after market” games, applications, themes, and content (like e-books). // Falls to the app developers and sometimes affected by the carriers who certify them.

2007-06-26 06:05:16 • hrant

“Image” isn’t necessarily a bad thing. Saying that image isn’t an important part of user experience just doesn’t make sense. The bigger problem, as I’ve seen it, is that device manufacturers/purchasers don’t understand the value of typography in creating “image”. As an industry the device makers are still a little engineering focused, as opposed to experience focused. Making type look good can be expensive from a processor/memory standpoint, not to mention from a development/design standpoint. The iPhone will raise the bar in this category, whether it’s successful or not.

As for the power of the Treo: it’s the only mass market phone that I know of where a user can write and compile software without having to sign up for a crazy developer program that requires each application to be signed by the phone manufacturer and lots of money (I’m not sure if windows mobile requires signatures, but I’m pretty sure that windows mobile devs can’t use free compilers - only pros with $$$ can play). A good example of why this is powerful: how many phones have a great pixel font editor? Only the Treo, as far as I know. In the bitmap font editor category, my years old treo is light years ahead of the iPhone with its closed platform. I know that hackability doesn’t always equate to user-friendliness, but user-friendliness is not the same as power.

What good is pixel font editor if the device is high res and uses ClearType like technology? It was probably more important years back with monochrome displays. All future handhelds (we’ll see how long we’ll have to wait) will use technologies like ClearType, which means that fonts will be vectors there too. No need for pixel editors. Processor power is doubling every 18 months, memory prices are falling down, so doors are opening here too. iPhone BTW doesn’t use the highest res of all the handhelds I know about. AFAIK there’s a much better display around on an iPaq. (coincidentally named with starting i?)

The bigger problem that I see here is the size of the display. Handhelds may become smaller and have higher resolution displays and being able to display FIRST CLASS type but that will definitely restrain our eyes. So the biggest advancement will still be in displays not technology behind them. How to pack a bigger display in a device of a smaller size? Or even don’t pack it but project the image in space/surface somehow.
