How do I replace end characters in an OpenType script typeface?

April 4, 2007 • Nick Cooke

I’m making an OpenType handwriting font for a client, so all characters have long lead-out strokes to flow nicely into the next character. However, the client doesn’t want these long strokes at the end of every word. So I’m wondering how I can replace the regular characters (eg. y) by end characters (eg. y_end), so in the word ‘every’ only the y would change. I’ve looked through the needle-in-a-haystack manual with no luck. How does one specify just the last letter of each word to change to an alternate form?

Any help would be much appreciated.

Make a ligature? y + space = y.end y + period = y
2007-04-04 18:06:28 • peterbruhn

Use the contextual alternates. Below is an example with a terminal “f”. Note you should use the “period” construction, as in “y.terminal”, not a ligature (underline) construction as in “y_end”.

feature calt { # Connection Forms

Latin

sub f' space by f.terminal;

} calt;

2007-04-04 21:07:56 • paul d hunt

Note that the ‘fina’ feature is on by default and in some applications may be impossible to turn off, so you only want to use it if the substitutions should always happen. ‘fina’ feature support presumes character level analysis by the layout engine, so you don’t need any context for it. The layout engine identifies the circumstances in which the substitution should occur and applies the feature.

If using the ‘calt’ feature, I second Paul’s suggestion to use the ignore syntax or, in VOLT, the EXCEPT syntax:

f -> f.terminal EXCEPT |
