Typophile: Feature request / wish list

April 17, 2006 • Miguel Sousa

I was going to add a new entry to the Typophile Beta Bug Report, however, since this isn’t really a bug but rather a request for a new feature, I decided to start a new thread. Hopefully others will add some more ideas, and we’ll all make our beloved Typophile even better.

Here’s mine: Is it possible to add a permalink to each post so that they can be referred directly within the thread? I know it’s already possible to do something like this http://typophile.com/node/17011#comment-103291 but not everyone knows about it. In addition, one needs to loggin (or dig through the source code) to get the comment number. Thank you :^)

(BTW, I noticed that this ‘technique’ doesn’t work for threads with more than one page.)

[Edit: A List Apart does this too. Notice how the page number is included. I see now that we could do http://typophile.com/node/17011?from=50&comments_per_page=50#comment-105491 but this is human-unreadable.]

Sorry, one more thing. Could the URL of the new page(s) be a little more user-friendly? Something like: http://typophile.com/node/17011?page=2 instead of http://typophile.com/node/17011?from=50&comments_per_page=50

(A List Apart does it)

Thanks again!

2006-04-17 01:55:53 • Miguel Sousa

One thing that bugs me is how, when a topic extends to multiple pages, the “new” flags get wiped out before you can get to the last page of the topic. My workaround has been to add “?from=0&comments_per_page=500” to the link url so all the threads appear on the first (and only) page.

Related to this, I would like the topic page links (e.g., “1 2 3 … next page last page”) to appear both at the top and bottom of topic pages. This would be especially helpful when there are lots of images to load on a page. For example, the only way to get to, say, page 3 of a topic is to got to page 1 first, then jump to the end of the page and click on “3”. But if page 1 has a lot of images, the bottom of the page keeps jumping down as the images load making it hard to hit that “3”. My workaround is to stop the page loading (leaving out any images that were loading). It’s tricky to time it right–if I do it too soon, I may stop it before the bottom of the page has loaded. If the page links were also at the top, it would be much simpler.

Just my 2¢.
