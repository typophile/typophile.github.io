David,

The whole point of “Ship Font” in VOLT is to strip the private VOLT tables from the font. If you don’t want to do that, just use Save.

VOLT strips existing OTL tables from a font when opening it because the purpose of VOLT is to create new OTL tables from the private VOLT data that the user edits.

I suppose we don’t do a good job of explaining that one of the cool things about SFNT is that the font file is made up of tables, and you can define and add your own “private table” to contain useful information such as high-level hinting source code, or OpenType Layout source code, or comments, that can be used by a font development tool. That way you don’t need to keep multiple files hanging around, all of your ‘source’ can be contained in a font file.

In most cases, you’ll want to remove (strip) these sources prior to distributing a font to a customer, mainly because they add additional file size to the font. In other cases, say you’re making an open-source font, you might decide to keep them in.
