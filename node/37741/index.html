2007-10-04 22:21:56 • afonseca1974

“The body in minion will be fine with the header in Arial and you could try to place the indent in italic helvetica, etc”

I think if you are doing the body in Minion, then you should choose either Helvetica or Arial for heads and indents. Use the regular italic and/or bold for the heads, contrasting or matching the indent depending on the tone you need for the content. E.G. if it’s a softer story, use Helvetica regular heads and regular or italic for the indent, or when you’ve got something harder, use the Helvetica bold for heads and sightly condensed italic for indents. If you want, you can also get a Minion Bold Condensed of some sort for heads. If you get enough Minion, you could easily do the whole thing in Minion for a Modern/Serious/Traditional/Formal magazine for medical articles.

Cheers!

2007-10-10 00:02:43 • afonseca1974

Two columns sounds right for the line width you describe. You might find 1 cm gutter between to be a bit much. 1 pica is traditional and I think 1 cm is over 2 picas. Do a test and see, you might want to cut the gutter to 0.5 or 0.75 cm.

Many articles in scientific works start with an abstract, and this is traditionally set in a single column. Use a larger type size and an indent so that the line measure does not get to be too long.

Illustrations can be either one or two columns wide, as appropriate. A 1 column illustration in the middle of a two column layout can be problematic, as you have been told, because the word spacing around then (where the type goes to a half column) can be too loose.

What you really need to do is to build a dummy page now, to see how things look. Post the PDF and we will be better able to comment on it.
