I encountered a similar issue but under the Windows version. Apparently some preferences got corrupted and apparently FontLab does not deal with those very well. I would try quitting FontLab, then removing the com.fontlab.fontlab5.plist file which resides in ~/Library/Preferences and start FontLab again and see if it helps. Be advised that doing this may (...will) reset a lot of other preferences that you've set (global options, etc.) so you may not want to pemanently delete that file...just move it out of the prefs folder onto your desktop or something, temporarily. Or, If you're comfortable with editing .plist files you could probably just edit the specific key, which I think is under /General/ClassListWidth. Mine is currently set to "142". There may be other similar values that need updating for your specific case.
