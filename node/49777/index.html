Chris, I’m familiar with this problem, but I’m still not sure what series of actions causes it. I’ve only ever ended up with half a dozen or so such glyphs, never hundreds. What happens is that multiple duplicates of GID 0 (usually the .notdef glyph) are created. These are only visible in Index mode. The way to remove them is to select and cut the individual glyph that has been duplicated. This either removes the duplicates or allows them to be deleted – I don’t remember which, it has been a while since I last encountered the problem –, but there is a risk of a crash, so it would be a good idea to copy the glyphs you need to keep into another VFB if there are changes you have not saved.
