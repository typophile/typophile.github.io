Problem with Ligatures on Fontlab

April 14, 2015 • Mauricio Villamayor

Good morning, everyone. I’m new at this forum, although I’ve been reading it since a very long time. My name is Mauricio Villamayor, I’m designing a calligraphic typeface for my thesis and I have a problem in doing it. I hope if you can help me with this situation.

Basically, the problem is with the ligatures. The typography has a lot of them, since the calligraphic characters are more complicated to bond harmonically. The ligatures work in all the combinations, except in the accented (acute accent) letters. For example, a specific ligature exists for the combination “yo” (which works), but when I create the combination “yó”, it doesn’t work, indicating an error when I compile the ligaduras.

This is the specific error that shows: “[FATAL] invalid token (text was “ó”) [C:/Users/Maui/Documents/FontLab/Studio5/fontlab.fea 4]”

Is it possible to create ligaduras with accented letters? Or am I doing something wrong? Maybe an error in the language configuration of the font? I searched a lot on the Internet about what could be the problem, but nobody seems to have a problem like mine (maybe because in English there are no accented letters). I would be really thankful if you could help me, I never had to do ligaduras with accented letters and right now I’m in a hurry since this work is for my thesis.

Thank you so much!

You need to use correct glyph names. The error implies that your Opentype code contains “ó” where it should say “oacute”. Every glyph name must start with a letter and may only contain A-Z, a-z, numerals, underscore (for ligatures), and period (to suffix alternates). If you give a glyph a name that does not fit these criteria, it cannot be referenced in the code. So, for example, the yó ligature glyph should be named “y_oacute”.
