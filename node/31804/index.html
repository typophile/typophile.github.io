2007-02-26 22:26:15 • Robby Woodard

All the OT fonts I am using for reference have tabular as their default.

This is not a must (but may be what users expect). Microsoft’s new ClearType fonts however use oldstyle proportional numerals as default.

Are there set unicodes for glyphs zero.tab through nine.tab and tabular currency?

In OT fonts you should select one set of numerals as default, name them “zero” to “nine” (no suffix) and assign standard numeral Unicode values. For additional sets of numerals attach the according suffix to the glyph name, but do not assign Unicode values. Substitution is handled by OT features onum, lnum, tnum, pnum. There is a very recent thread about this.

And how important are the unicodes for OT fonts anyway?

Most important. :)

Won’t everything work if I set up my OT classes using glyph names?

In FontLab Studio, as well as the AFDKO, both classes and OT feature code are defined via glyph names.
