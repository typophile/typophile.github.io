Adding ordinal caps in TypeTool 3

January 23, 2008 • olho

I'd like to add ordinal capitals to my font, but I can't find where they are put in TypeTool 3's font table. They can either be set glyphs - ST, ND, RD, TH etc. or a full set of superscript letters - A to Z. I only need uppercase as my font is a titling face. I've tried all the default layouts. For the most of the work I've been using Adobe OT PS standard names (CFF) as I want to output to OpenType. Perhaps it's not possible to add the ordinal caps in TypeTool which has basic OpenType table support. Can I not add custom glyphs? Anybody else come up against this, or overcome it in TypeTool?
TypeTool doesn't allow to create new empty glyph slots, but it can be done using a little trick. Copy and then paste any existing glyph. TT will ask if you want to keep the replaced glyph. Do so, and the glyph will be added to the font, the glyph name being suffixed ".001". Then rename the new glyphs as needed, in your example you would need d.ordn, h.ordn, n.ordn, r.ordn, s.ordn, t.ordn and/or their uppercase counterparts D.ordn &c. You cannot write the OT feature code to switch to ordinals in TypeTool, but you could add the features using Adobe FDK later. Cheers, Jens.

Of course you can add new glyph slots in TypeTool. Just open your current .enc (Encoding) file in a plain-text editor (or make a new copy of that file first) and add the glyphnames you need. Restart TypeTool and the glyph slots will be available. You can use the free Microsoft VOLT or Adobe FDK for OpenType tools to add OpenType Layout features to your font. The FontLab Studio 5 user manual (and I think also the TypeTool manual) contains a section on glyph naming and encoding. Regards, A.
