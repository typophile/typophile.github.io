Need Workaround Assistance

May 25, 2005 • Bryan Narendorf

I am setting a book that includes the polytonic (ancient) Greek alphabet. The Word document that I am pouring into InDesign has the Greek set in Hellenica, which has really large word space and unacceptably offset diacritical marks. I am setting the book in Minion Pro, which contains the appropriate glyphs, but when I selected the Hellenic text to change it into Minion, the Greek letter forms are replaced by Roman letter forms. It looks like the Unicode values for equivalent glyphs in each font, e.g. the Hellenica alpha and the Minion alpha, have different unicode values.

I’m about to start setting a Greek-heavy chapter and would prefer to not have to manually change each glyph.

Can you offer any suggestions?

Many thanks in advance,

Bryan

I’m guessing that the Hellenica font uses a hack 8-bit encoding for polytonic, rather than Unicode, and the Word doc uses this hack encoding. You are going to need to re-encode that Word document. The easiest way to do this is probably in Word, prior to copying/pasting to InDesign, using a macro. The macro will need to search/replace each hacked character in the Hellenica encoding and convert it to the correct Unicode character.

2005-06-01 06:43:07 • Bryan Narendorf

Actually, the later releases of Word do handle Unicode – you don’t really have to have OpenType in the Word documant. See, for example,

http://www.myfonts.com/info/opentype-support-in-applications/

As you can see, for everything to work out, both your platform and Word version and the authors platform & version of Word must support Unicode, and the font must use Unicode indicies, which right now means TrueType.

Your problem was the fonts from Linguists Software – All of their fonts I’m aware of use, as John suggested, the 8-bit format with “inappropriate” names, not Unicode indicies. This “inappropriatness” isn’t an error on their part, it is caused by most composition systems not allowing the use of a custom encoding vector.

We’ve taken Word documents with polytonic Greek straight to comp when the font used in Word was Palatio TT (Unicode). I believe the customer wanted the job set in Palatino, but with the Unicode there, we could have set in in Minion OpenType CFF with no extra effort.
