...you can only access half of your RAM, and half your CPU in each OS. I'm pretty sure it doesn't work that way. Mac OS X uses dynamically allocated virtual memory. Each app (including virtual machines) can take as much memory as they need. The hard disk will be used if there isn't enough physical RAM available, mainly for inactive apps, but even for the active ones if necessary. You can't allocate more RAM on your virtual Windows machine than is installed on your Mac, and you will definitely run into performance problems if you allocate the maximum (a couple hundred MB less than the installed physical RAM), but short of that, it's no different than using any other app. Also, unless you are using some other app that is hogging the CPU, Windows will get all the CPU it asks for, again, just like any other app.

Perhaps new versions of Parallels are different, but I'm fairly certain that older versions did split your CPU in half. One core for OS X, the other core for Windows. Though I haven't looked into running Windows as a VM under OS X in 2 years or so. No doubt other VM solutions don't impose this restriction, and I wouldn't be surprised if newer versions of Parallels didn't either. Nor would I be surprised if I was totally misinformed about older versions of Parallels. In retrospect, the 'half your RAM' crap I was told was probably just a rule-of-thumb for standard 2GB RAM MBPs. 1GB for each OS to avoid going into the harddrive swap that Mark mentioned. At any rate, doing 3D modeling on a VM doesn't make much sense to me -- nor does re-licensing all my software for OS X -- so it's been Bootcamp all the way. And I will say that the only bad experience I've had with Bootcamp is related to the battery driver. Again, no doubt fixed in newer releases. But if you decide to go with Bootcamp, and your CPU usage in Windows jumps to 100% and everything goes to pot -- before you're tempted to throw your computer against the wall, disable the ACPI-Compliant battery driver. If that 'fixes' things, re-enable the driver, reboot into OS X, reboot into Windows, and everything should be coming-up-roses.
