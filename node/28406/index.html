Here’s an update for anyone interested in the gory details. First of all, apologies for the recent outages.

In order to enable a database cache, we upgraded MySQL to version 4.1x. At the same time, we added an autostart script. What I neglected to do was to name the said script starting with a capital S (RedHat likes it that way).

In the mean time, our server is being relocated to a really sweet data center, so for the last few weekends the nice folks at Media Temple have restarted the server a couple of times. Since the startup script was failing, MySQL didn’t come back online with the rest of the server like it’s supposed to. The problem has been fixed, and hopefully we won’t have any more sad Typophile-less weekends.

The disk space issue has been solved and isn’t an issue.
