Assuming a 1000-unit em square,set your Ascender Height to 1000. Prepare your characters for import with spacer marks at the Ascender and Descender extremes. Import all of your characters into FOG. [img:sites/default/files/old-images/Erasmus_Mediaeval_6204.png] Then, find a reference point for your baseline. Zoom in and click the "Guides" checkbox. Drag a guideline to the baseline and make a note of its vertical position. Under FONT INFO, Dimensions, enter this number as your Descender, then subtract that number from 1000 to get your Ascender. Then select all of your glyphs and MOVE them DOWN this same value. Then RESET YOUR BASEPOINTS to 0,0.

As far as the multi-tonal aspect is concerned, most modern font formats only support pure black and white. I'm pretty sure Type 3 was the last format to support grayscale values and most modern software won't be compatible with Type 3 fonts. What you might want to try is making a different font file for each color you want. Then you can layer them and change the fill as you see fit. A lot of type designers use this technique for drop shadowed, 3d, or pattern filled fonts.

There is one caveat about using this import method in FontLab: for some reason, if your spacing marks are not part of a solid figure, FontLab imports the EPS at less than full em-square height (the top image in the screengrabs below). A solid full em-height figure, such as the bar glyph in the bottom grab) does import full height... [img:sites/default/files/old-images/FL5_anomaly_4162.png] So, if you want to maintain consistent dimensions throughout your importing, for FL you need to use one method or the other, but not both...
