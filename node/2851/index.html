Different groups with Adobe are responsible for defining the feature set of their apps. The <salt> feature is something of a peculiarity as it may be implemented with GSUB type 1 lookups (one-to-one) or type 3 lookups (one-to-one-of-many). Apparently the InDesign team decided not to support the feature at all until they are able to support the type 3 lookup, while the Illustrator team decided to provide support by treating the first enumerated glyph of a type 3 lookup as if it were a type 1 lookup. So Illustrator provides partial support for the <salt> feature (so, by the way, does InDesign Middle East).
