January 20, 2003 • capthaddock

There seems to be a lot of interest here in designing or discussing low-resolution bitmapped fonts. OpenType fonts have a feature that allows bitmapped glyphs for specific point sizes, optimizing on-screen display. Has anyone tried doing this? Paul
I've experimented with this a little bit Paul, but I'm no technical wizard, so for your own safety, doubt everything I say here. As far as I'm aware, your OpenType font has to be TTF flavored - not CFF. I work in FontLab, so if you use that, try the following: 1) open a glyph 2) Tools>Operations>TrueType Hinting 3) turn on the Bitmap tool (see below) 4) right click in the glyph window 5) select Generate Bitmap You have to generate bitmaps individually for each glyph, for each ppem you want to support, as far as I know. Hope that's helpful. Nate

2003-02-03 21:58:06 • hrant
I've had a little experience with embedded bitmaps. My (free) font Anonymous, a Mac/Windows TrueType font, uses them for the Windows version. (The Mac version has them in the suitcase file.) It works, but it depends on OS support (and application support sometimes) for it to work properly. On MacOS 9 (and earlier) and Windows they are used instead of the TT rasterizer at the sizes that you would expect. MacOS X uses the bitmaps only when they fall below its anti-aliasing threshhold, otherwise it ignores them. Flash will use them (yay!) if the font is installed on the client system for dynamic and input text, but not (boo!) for static text and not for anything when you embed the font in the swf. Also, the size at which they are used on Windows differs from Macs in the usual way. In other words, if you have embedded a 12-pixel tall bitmap font, it will appear at 12-point on Macs but at 8-point on Windows (assuming that the Windows default 96-dpi setting has not been changed).
