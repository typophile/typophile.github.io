2008-08-23 15:57:12 • Stephen Rapp

Linotype has the habit of producing same-named fonts in two variants: “family-linked” and “single-linked”. The single-linked fonts appear as separate families with just one style, while the family-linked fonts appear as multiple styles grouped under one family. The confusing thing is that the filenames of the fonts are the same so you need to be careful which fonts you actually install.

I don’t know if any other vendor is doing this, but still, family grouping conventions definitely differ between font vendors.

Sandy, existing Windows GDI API has a limitation of only supporting up to four basic faces (regular, bold, italic, bold italic) per font family in its font selection logic. In addition, many applications allow one to apply additional algorithmic bold and italic simulation to a native font face. These factors contribute to what you are seeeing.

This is something we are working on improving. For example, Windows Presentation Foundation (WPF), a relatively new client programming platform built on top of .Net Framework, supports virtually unlimited number of font weight/width/slope combinations in a single font family, which allows an application built on top of WPF to present a single font family instead of four different ones in your example. Here is a relevant thread discussing this issue: http://typophile.com/node/33288

Best regards, Mikhail Leonov Microsoft
