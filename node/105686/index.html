Hi all,

I’ve created a typeface that has a swash on the uppercase. I want to create alternate uppercase glyphs without the swash that will be used as a default after an initial cap. Problem is, I’m not sure how to create them so that they’re interpreted by a design program in the correct way.

I’m still learning Fontlab but have created three weights, set the metrics, kerned and included ligatures – this is my last stumbling block so I’d really appreciate the help.

Thanks everyone

Stew

From your description it sounds like you have swashes on the default form of the caps? If so, the following would work.

feature calt { sub [@Swash @NoSwash] @Swash' by @NoSwash; } calt;

Where @Swash and @NoSwash have been defined to to include your default caps and your swashless caps, respectively.

I’d suggest, though, that it would be better to use your swashless forms as the default and then include

feature swsh { ignore sub [@Letters] @NoSwash'; sub @NoSwash' by @Swash; } swsh;

Where @Letters is the set of all upper and lowercase letters defined in your font. This would provide a (IMO) more intuitive interface where swashes are turned on by activating them rather than turned off by deactivating contextual alternates.
