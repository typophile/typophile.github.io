2005-06-06 03:01:29 • paul d hunt

Classes can be tricky. I’ll tell you how I do it, then I’ll add what doesn’t seem to work.

For your A class, I will set up a class called _KERN_A_CAP, and in the classes panel it will look like this: _KERN_A_CAP: A' Aacute Acircumflex Adieresis Agrave Aring Atilde Abreve Amacron Aogonek.

Then in the opentype panel you have to also add a string so that the OT features know about that class. (Apparently this can be done automatically, but I do it by hand.) There you write the class like this: @_KERN_A_CAP = [A Aacute Acircumflex Adieresis Agrave Aring Atilde Abreve Amacron Aogonek];

There are a few things to keep in mind. One is that you cannot have a space after the semicolon. Another is don’t forget to remove the mark after you add the key glyph in the OT panel. The third is not to put the same glyph in more than one class of the right or left groups. I.e., don’t reuse glyphs in different classes from FL. You can probably do this if you’re using Adobe FDK to generate fonts, but I have no idea how to do this. From FL, it just seems to mess things up.

Another note about the left versus right classes. It’s important to remember that Right classes are the ones that go on the right, not that kern on the right. So if you want to kern everything like Ta, To, Tq, etc, then the class containing a o and q will be a right class. As I’m sure you know, you make a class a right class by checking it off in the Kerning Assistance dialogue box. (Don’t forget to check on “Apply and Save” every time you change the kerning, or the changes will not be made in the OT features.)

I hope this helps.
