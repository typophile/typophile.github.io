Goran, the Indic scripts are complex writing systems that require not only correct (Unicode) text encoding for the basic letters and marks, but also many glyph variants (e.g. half forms, conjunct ligatures) and smart layout for mark positioning. The Devanagari fonts I am working on at the moment each contain more than 650 glyphs, not including Latin characters. I use OpenType Layout for the glyph substitution and positioning. One could also make Mac-only AAT format fonts, but in any case this isn't something that can be undertaken as an 'extra bonus' in a Latin display type. Unlike some elements of Latin typography, the complex glyph handling of Indic scripts is not optional: you have to provide complete and correct shaping in order for the text to be normally readable. As an example, here is some Devanagari text shown (top) with the raw Unicode characters before subject to shaping and (below) the correct shaping as applied using OpenType Layout. Note that in addition the letter ra becoming a subscript mark attached to the preceding letter (in other instances, it would become a superscript mark attached to the following letter), and the combination of consonant sequences into conjunct ligatures, some glyphs actually change order. All of this needs to be handled by the font and an Indic shaping engine. [img:sites/default/files/old-images/DevShaSam_6589.gif] Regarding keyboarding, one would use an appropriate Indian keyboard driver, e.g. Hindi or Marathi for the Devanagari script. I used the Windows Hindi keyboard for these samples.
