I find it very helpful to partition hard drives, not merely into two or three, but really carve it up. This allows certain data types to be grouped easily, and you end up with multiple small directory trees, one on each partition, instead of one great big cumbersome tree on C drive. Much easier to navigate and find things, automated searching is quicker. For example: C: Sys Vol (the OS and system apps only) D: Documents (.doc .pdf .rtf .qxd .pm6 .p65 .txt, email) E: Applications F: Admin (desktop, shortcut to IN drive, out folder, personal stuff, project folders, shortcuts to key utility proggies. This drive contains any data type) G: Work drive (.ai .avi .bmp .eps .indd .pcd .psd .psp .gif .jgp .png etc) H: HTTP 1 (saved webpages from the interwebs) I: IN drive (my in basket, accessed by shortcuts on F:) J: Anime (a library of anime artwork I collect) K: D'Arts (artsy-fartsy artwork of all kinds for reference) M: Femmes (pictures of attractive women) N: Webdev (web-based work projects: .html .htm .php .css .txt) P: Pics (massive library of stock images, different from artsy-fartsy stock) T: Type (whole drive devoted to fonts and EULAs) S: Spare (large storage drive for dumping giant-sized files, software, DVD movies and the like) A place for everything and everything in its place. Having one drive devoted to fonts only makes finding a font manually quite easy, provided you create an extensive hierarchy of category folders. The IN drive has only a few folders, each assigned to whatever clients I'm working with at the moment. I used to have an OUT drive just for outgoing items but found an out folder on the admin drive more efficient :^) j a m e s
