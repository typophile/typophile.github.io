Kenneth, I would strongly suggest you learn about the 2 types of kerning, the newer class based kerning and the old-time Type1 kerning table and how they interact in FontLab. It’s pretty cool/well done, actually. It helps to understand things better and allows you to check your settings for both opening existing fonts and generating fonts. Word 2003 does NOT support the class based kerning but it should use the older style kern tables and you have to tell Studio (not sure of the default there) to include that when you generate an OTF or TTF. Tools, Options > Generating OpenType & TrueType > Kerning and turn on the “Export old-style non-OpenType “kern” table.”

That said, I beat my head against the wall for a week or more but was never able to get Word 2003 to use the kerning in an OTF font. Generating a TTF DID work in 2003. Word 2010 DOES use the kerning in the OTF fonts I’ve generated.

If you’re opening old Word files in 2010 (I don’t know about 2007 or after 2010) you’ll need to take it out of Compatibility Mode. By default Word 2010 opens documents created in older versions of Word in Compatibility Mode to help minimize reflow of text. At least, I THINK that’s why they did that.

Hope this helps, if you find a solution please be sure to post back.

Good luck! Ken
