The logo has to perform in any realistic scenario. Clearly it is going to need to be reversed.

Say, for instance, in the future the company sponsors an event. The event host wants to include all the sponsor logos on the bottom of a brochure. That logo may need to be reversed. It may need to be really small.

It is very common to reverse a mark. I would say that if a mark was designed that couldn’t be reversed the designer would have needed to strongly advise the client about the issue before they made their selection.

Logos have to be used for all purposes. (I know two others have said that already.)

An example is the CBC (Canadian Broadcasting Corp) who spent the better part of a million dollars to come up with an animated logo that looked wonderful in the TV commercials. Then they tried using it in their printed media, and had a great deal of trouble getting the animation to work well.

In the end, the logo was simplified considerably, and now works well in print and animated.

Not as big a fiasco as NBCs, but still notable.

Some good things to keep in mind when doing a logo are:

If you’re using a spot colour and CMYK, make sure you do a straight 4 colour version as well. Make a 2 colour version, if you use a pantone colour Make a reversed (white) version and set your Illustrator file to ‘view outlines’ or set it on a dark background so people can see it when they open the file A 1 colour version (generally black), sometimes it’s ok to go with different shades of black to show different areas, but be careful if you do so. make RGB versions of the above coloured and 1 colour logos. Include the correct values for PMS, CMYK and RGB in the standards guide

Make sure the logo will scan and fax well. Check legibility at small sizes – it might be worth it to create a second version for use when scaled under a specific dimension. Set a minimum size in inches and pixels tho logo can be scaled down to, to ensure the best possible representation. Create a guide for the amount of space left around the logo.

Also consider the use(s) of the logo - you might want variants. For example if it’s a stacked logo you might need to do an all-on-one-line version. Try to anticipate every use the logo might have so that you can - as best as possible - prepare guidelines for it. If you leave matters to chance your client or their web designer may do something you don’t like.
