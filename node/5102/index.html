Early CRTs used vector-based character displays. This would be in the 1960s. In the 1970s, raster displays were introduced. The characters on these displays could be considered the first bitmap fonts, but they were not fonts in the sense that we mean now. The patterns for each character were stored permanently in the computer memory and would be called something like "resident character set" or "character set display map". The size of the characters on the display was also fixed and every character was the same width, so video dispays were typically specified by how many characters they could display (e.g., 80 x 25). These early character sets did not have names that I know of.

The idea of a "bitmap font" (as opposed to a resident character set) as a discrete system resource of interchangeable sizes and styles for a computer display, probably started at Xerox PARC in the 1970s. I don't know what they called them, but they generally appear to be modeled on Helvetica, Times, and Courier. (This is not too surprising since the founders of Adobe, who chose these same fonts as the basic PostScript set, came from Xerox PARC.)
