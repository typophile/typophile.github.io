August 16, 2008 • krdoughty

Apologies if this is not the site to ask these questions, but I have a user moving working font files from one PC to another - both running XP SP3. She has even tried installing from clean copies with the same results, per below: - Open Type Fonts generate a 'file is missing or damaged' message - OTF fonts show up in /fonts/ folder, but are designated as TTF - trying to open these files also generates 'file is missing or damaged' message - The icon associations are all over the map: .ttf files that call themselves 'True Type' have a true type icon, .otf files that call themselves Open Type (and don't work) have an open type icon; and .ttf files that say True Type but say they are an 'Open Type Font with True Type Outlines' when I open them also have an open type icon. This may be normal, I don't know. It is unclear to us whether this is a Problem with the font files or with Windows. Any advice/referrals much appreciated.
Try to open and view the OTF files with a program that does not use any of the Windows APIs. If that fails, it's the files themselves that have been damaged in the transfer. If not, your Windows installation might need a refresh ... The icon oddities come from the fact that Windows looks at the file extension -- otf is OpenType, ttf is TrueType. Then it checks whether the file is digitally signed -- "yes" shows an OTF icon, "nope" shows a TTF icon. Go figure.
