There are plenty of critical system fonts that are TrueType fonts (.ttf). I wouldn't touch Tahoma, for instance.

Additionally, I don't know anybody who thinks it is sensible to convert Windows system fonts to Type 1. Why on earth would one do such a thing? You'd break the extended language support and the TrueType hinting. In any case, it is not permitted under the Windows license.

2004-05-19 07:52:53 • pixelmaster
ATM Light doesn't really "manage fonts" in the sense of turning them on and off and handling sets. For that you need ATM Deluxe (or one of its competitors). ATM Light just provides rasterization services, and most of that is now build into the OS (except for multiple master support).

Changing the format of the font won't help system performance. For that you need to deactivate some fonts, which you can do just as easily regardless of font format.

These days Windows can handle having hundreds or even thousands of active fonts. The main thing that gets slowed down is that certain applications enumerate all the fonts when the app is launched. Plus anything with a WYSIWYG font menu may have some extra work to do. But all this is pretty much linear; there's no magic number where things suddenly get much worse. 300 fonts isn't that much worse than 200.

If (cough, cough) there's any worth keeping, only install those. Then with ATM Deluxe or Suitcase you can turn them on and off individually.
With dreamweaver, you won't be using fonts other than system fonts or default sans or serifs, unless there's something I don't know about. Nobody will be able to see anything else, unless they have the same font installed and active.
Atm deluxe can handle both ttf & ps. Also, if you need type reunion, it comes bundled with atm deluxe.
I'm not sure atm light handles ttf, and if it does, deluxe isn't so expensive anyways. And, like Thomas said, windows rasterizes most everything already.
