2003-10-31 19:59:30 • aluminum
Joseph, spec'ing type in pts or another resolution-independent measure is certainly the best idea. Yes, the results are going to vary across platforms, but they are not going to vary as greatly as fixed size pixel type across different resolutions. Personally, I think type size should not be spec'd at all, except as relative to body type, but should be determined by the user-side preferences, and all other measurements, e.g. text block width, should be spec'd in ems so that these too are relative to the body type. That way you have perfectly scaleable web pages. Above, Darrel says that he thinks 780 pixels is 'simply too wide for comfortable reading lengths for a lot of people', but this observation is itself based on the assumption that the type is spec'd in pixels. If the type were spec'd in pts, then the number of words a on 780 pixel measure would depend on the resolution: it would probably be a bit narrow for comfort on my screen. But if the type were spec'd as pts and the measure spec'd in ems, then the number of words on a line would be the same regardless of the resolution.
