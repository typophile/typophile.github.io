Postscript font corruption

January 15, 2010 • Stephen Rapp

I’ve seen this happen a few times now and am unsure what is causing the problem. So I could really use some help here from some of the brilliant typophiles here. :-)

We have had several PS fonts that have become corrupt over the last year. They used to work fine and now in Illustrator come up at the bottom of the list enclosed like < font name >

Our work environment changed this year and I suspect that is part of it. We use a server based font management system. Previously we were using Freehand for layout and Font Reserve for management. Now we are using Illustrator for layout and Universal Type Server for font management.

When I was asked to look into the problem for one of our proprietary fonts I first tested it by bypassing the font manager. I put the font in my home library collection (Mac) and tested in Illustrator. The results were the same as reported. I then reopened the font and the thing that seemed out of place was in the font info under encoding. For some reason at the bottom pull down menu for encoding where it says “Mac script and FOND ID” it was switched from Roman to Arabic. I’ve seen a couple other times where this has happened even with some commercially licensed fonts. Since Alphabetically Arabic would be the first entry I’m wondering if the encoding went haywire and so Arabic came up as default.

Any ideas or similar experiences?

The FOND ID controls what writing system (“Mac script”) the Mac OS, and Illustrator, think the font belongs to.

Because there is a relatively limited number of FOND IDs available, Mac OS can automatically re-assign FOND IDs to fonts in memory so as to avoid collisions. However, it shouldn’t change which range the font is in (Latin vs Arabic).

Additionally, there are tools that can be used to manipulate or correct FOND IDs.

I’d suggest first opening the font up and checking what the FOND ID is in the physical file. If it’s in the Arabic range of 17920-18431, change it to something in the Latin range of 00000-16383 (probably in the upper 3/4 of that).

See also http://www.evertype.com/software/apple/fontnums.html
