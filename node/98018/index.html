This is a very old problem. Some older FontLab distributions were missing an entry in the Standard.nam file. on a PC, you can find the file in c:\Program Files\Common Files\FontLab\Mapping\ Edit the file and insert the line 0x0025 percent between 0x0024 dollar and 0x0026 ampersand The line may be missing in other .nam files in that same folder (Mapping) Is this the same font you're having the icon problem with? - Herb
