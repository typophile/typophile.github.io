Sorry, this isn’t entirely accurate. It has nothing to do with whether fonts are ‘homemade’ or not. An units per em (upm) other than 1000 is perfectly valid for OpenType fonts. In fact, it can have any value between 16 and 16384. There are a number of fonts from FontFont and Adobe which use 2000 upm.

Changing the upm in FontLab will scale the glyphs and lead to distortions in the outlines. (There was probably a good reason why the type designer needed a higher resolution in the first place.)

I can’t imagine what effect ‘rebuilding the names’ should have on this issue.

Application updates can’t change this (unless applications switch to their own font rendering engine). It’s clearly a bug in Mac OS X and will probably be fixed in the next system update.
