the halving in InDesign is only suggested in the panels, not the layout, so a kerning value of 100 in FontLab will show as 50 in InDesign type panel but 100 in the actual layout Yes. Adobe by default apps express all kerning and tracking in 1/1000 units per em, regardless of the actual UPM of the font in use. This is sensible, because it means that spacing is not radically scaled when you switch fonts.
