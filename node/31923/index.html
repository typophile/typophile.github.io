2007-03-02 18:21:01 • Quincunx

if i open the new generated font with fontlab, kerning is ok. But if i use the font on any application (photoshop, illustrator…) kerning doesn’t work…

My guess is you have no substitution feature(s) defined in your font.

This comes up about once a week. I think we need a wiki entry until Adobe fixes the bug in future releases.

Anyway, the GPOS tables (kerning) will not work in many Adobe applications unless there is also a GSUB (ligaturing, fractions, etc.) table as well.
