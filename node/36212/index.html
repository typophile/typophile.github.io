How perfect must it be?

August 17, 2007 • kosal

In my neverending quest to complete just one set of glyphs into a sellable font, I become overwhelmed when I see the pros. Usually with scores of fully equipped font families under their belt. Foreign chars, ligatures, and all. How does one man (or woman) do so much in a lifetime? Especially when it takes me so damn long to finish a measly ONE. Am I missing some big secret here? I think my main problem is I don't know when to stop. This question comes from looking at existing fonts. For example, the > was virtually identical to the mirrored <, but a copy + paste-on-top reveals minor differences. Is this due to casual carelessness, or just the opposite? Since I've attributed it to being intentional, I now draw as if my glyphs have to be mathematically point perfect in order for it to be supported by the foundries. How good is good enough? Oh yeah, and I haven't even started hinting.
How good is good enough? That will be determined by the buyers, will it not? Gerald

... it takes me so damn long to finish a measly ONE. Am I missing some big secret? ...my main problem is I don’t know when to stop... Time at it, is a big part. I assume you are attempting this digitally? If this is the first "font" you've designed, maybe You are not "being a good client." I find it's easier when I have a who, telling me what, why, where and especially when. In the absence of an actual client, I make one up if I have to to perform the vital function of worrying needfully. Then, if you think about the word font literally you can be a better client by only looking at and judging the font used for one precise intended purpose. Practically speaking this makes the definition of "mathematically point perfect" apply only to a single use of the digital outline, and not all possible uses. This focusing of your eyes on an actual target, hopefully frees you to use the tools, hit the mark, and stop right there, as in — that looks good, and it works, next glyph, or...next font. Cheers!

I think Ricardo's comment is most crucial; save literally months of plodding by resolving problems with the test characters. Measure twice, cut once. I didn't actually do that with my last release, but I did start it while an intern, and I learned what Chris is now learning. Now, when I test out designs, I never go beyond keyword stage until I've made up my mind and solved basic design issues, and anymore, I don't even start unless there is a brief or specific intended use. Not sure what your process or tools are, but the enormous character sets in Latin typefaces are composites, and the accented characters can be assembled in a very short time when the parts are finalized. That step should not drag on, and niether should kerning, if spacing is resolved and you use time-savers like kerning classes.
