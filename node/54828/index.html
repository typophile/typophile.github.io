Here is my thinking, and the why – the “why” because you may decide something else makes sense.

I make up separate superiors, “supers” and denominators and numerators. The superiors are for direct entry, as with an equation, NOT to be used via an OT feature. The reason is that the text file is not changed via the use of the feature; if you want 10^2, that is what should be in the file. “Supers” in my fonts are for things like footnote calls.

I work pretty hard on my footnote calls (“supers”). They usually ride higher than a numerator (or a superior, for that matter), as the sequence all)" and a note call should not create a lot of visual bounce along the top (ascender, parenright, quote, footnote call). “Supers” are usually a touch smaller than numerators, too.

Numerators and denominators should balance with the figures, both oldstyle and lining, in size, weight and vertical position. Denominators will usually be different than subscripts, as superscripts/subscripts should have the height/dept for use in an equation; equations are different than a string of just figures.

As for encoding, I don’t. I use “one.super”, “one.numr”, and “one.dnom”, etc. They have no Unicode number. But I work on fonts only to be used in our shop, with InDesign. If you want to give them PU Unicode assignments, that seems OK.

It is extra work to make up good sets of these numbers, but it pays off. Sidebearings differ for the sets, to keep them amenable to class kerning, with minimal kerning exceptions. As I said, I think size, height and weight are slightly different for the different uses. And by all means remember what will remain in the text file should it be “repurposed”; OT features don’t change what is in the file.
