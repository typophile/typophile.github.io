Don, I can tell from that test that you're simply using the wrong point size. A pixelfont has to be set at a single given point size (at a certain dpi) to work. UPFs are pixelfonts - and they need to be set at 8 point (no matter how many vertical pixels the font spans).

If you're still unconvinced by the Ultrafonts site (or my sample), you can choose a font and go into "Sample", and set your own text. In this case I guess you'd also screen-grab then reverse-field it, which however brings up the possible need for gamma correction*. So don't despair, I think you're almost there. :-)

* Note that Mana comes with two versions (for each size & weight), one for positive setting, and another for higher-quality negative setting. The latter version can also be used as a native Windows font (like in a word processor) to great effect.

The font you choose depends on a few things, including the importance of legibility versus style.

It's conceivable that many people will be looking for their favorite racer, and since your tables have a lot of entries, you might like to facilitate that by having a font that makes it easy to scan-and-find names quickly. This means a font with: structures that people are used to seeing (considering that most reading is done in print where fonts have a certain "traditional" look), so for example having a "y" with diagonal arms helps; sufficient interletter spacing (which is actually where most pixelfonts fail); and capital letters that have some presence (since this is all about proper names).

Very tight and "geometric" pixelfonts are great for buttons and short headings, but not reading tasks that are performance-based. You don't race a Miata on a oval, dude. :-)
