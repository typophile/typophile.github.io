Yes, Mac OSX supports the following font types:

* OpenType (both Mac and Windows formats)
* PostScript Type 1 (Mac only)
* PostScript Multiple Master (Mac only)
* TrueType (both Mac and Windows formats)
* dFont (data fork TrueType) (Mac only)

OS9 did not support all of these, so you are now free to use your windows truetype files.

PostScript Type 1 fonts were for a long time, the industry standard for professional typesetting. These fonts were, and still are, platform specific. For example, a windows PostScript font uses two files with .pfm and .pfb extensions. This is the one format not currently supported by Mac OS at the system level. There is a way to get these fonts to show up in Adobe applications only, but I would only look into that if you have a real need. Also, to be clear about your first post, you should not rename anything. It probably wouldn’t cause a problem, but it’s certainly not necessary.
2008-06-17 18:59:40 • canderson

Just to clarify, there do not exist separate Mac and Windows versions of OpenType fonts. There exist OpenType fonts with TrueType outlines, and OpenType fonts with PostScript style outlines; both flavors are cross-platform font files.

As a side note, the dividing line between OpenType fonts with TrueType outlines and Windows TrueType fonts is very thin indeed, and one often doesn’t even need to make a distinction between the two.
