What could I have done to break the verticle metrics

That could happen if you’re allowing FontLab to recalculate some OS/2 and hhea values, such as TypoAscender/Ascender etc.

and where do i fix this?

Using FontLab’s “Metrics and Dimensions” –> “TrueType-specific metrics” panel, make sure you’re using the same values as the previous version of the font.

Alternatively, you can get those values from the older font with [[TTX]], and change them in the new font, using [[TTX]] as well. The values you’re looking for are in the tables ‘OS/2’ and ‘hhea’, so you can use this command to get them:

ttx -t OS/2 -t hhea MyOldFont.otf

Then you’ll just have to edit the .ttx file that got output (MyOldFont.ttx), and integrate that file with the new font by typing:

ttx -b -m MyNewFont.otf MyOldFont.ttx
