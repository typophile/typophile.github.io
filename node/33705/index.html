Well, I guess the one-million-dollar question is... Does it show when you print it? It looks like your screeen shot was taken at 100%, which usually means one is seeing a fairly accurate representation of how the document will print. But I'd rather see a raster image of the .pdf transported out of Illustrator and cropped. If it still looks ugly when you print it then there are one-million more questions that need to be answered, such as: Exactly what you mean when you say "vector object?" Was the logo created in InDesign using the vector drawing tools, or was it created in some other program and then transported in somehow? If so what was the logo's original format? I don't mean "vector", there are a bunch of different vector formats, .ai, .eps, .wmf, .pic, .myd, even .pdf itself (just to name a few). Each have varying support in Acrobat. Which one was the file originally saved as? Which version of InDesign are you using? Which version of Acrobat are you using? Which version of Distiller are you using? There are probably more where that came from, but see how it prints first :) Good luck!

I've noticed the same thing happening in PDF's created from Illustrator. So far I've not had a problem with the final printing of any design that's done this on screen though. I'm pretty certain it's a result of Adobe Reader's raster engine's preference to whole pixels for straight edges. Overall it's probably a good thing as it's possibly the only reason the address is still readable in the image you posted. I guess it's similar to font hinting in a way, but for screen pixels instead. If it changes at diffent zoom levels then I dont' think you have anything to worry about. If you are using PDF's to proof the job with the client and you don't want them to see the glitch then I think converting the logo to high res pixels is the only option.
