To kern MATT correctly, I would do this:

  1. type NATN or HATH (whichever has both sides vertical and is largest),
  2. kern AT so that the visual distance between MA and AT is the same (other way of seeing it: so that you get the same white in-between)
  3. use the AT kerning value in MATT, chances are you need not change TT BUT if you do
  4. type HHTTHH or NNTTNN so you get an evenly spaced block.

IMVVHO your name does not look good with the shortened TT bars; it might look better with either a ligature TT (joining on top) or a TT where only the inside of the ‘cross’ bar is shortened.

Still, I would keep it as in the font (or change font).

PS. My name is DAVID and in 99.99% of the fonts, it looks horrible when spelt in all-caps, I need to respace everything manually, sadly (working first on HHDAHH, then HHAVHH, HVIH and chekcing HIDH, though this is rarely a problem - then it all assembles magically).

All-cap (and small cap) settings are traditionally letterspaced, and to this day, many foundries add extra “cpsp” as an OpenType feature, as a matter of course.

I concur with Ilene. When I first started making fonts, I put a lot of kerning in them, but now I consider that less is better. If a font has kerning between A and T, and A and C, as this one appears to, it only worsens gappy combinations where kerning is not possible.

The exception is the “tight but not touching” style, where the spacing is all about proximal distances, not the balance of light and dark.

IMO, the proximal style makes sense for Syndrome, as it is a feature of the wordmark, where the stencilling leverages the proximal effect. So I don’t have a problem with MATT MACDONALD; however, I find the letterspacing in Syndrome to be too great, in relation to the stencil gap, and the tightness of MATT MACDONALD, which is also much tighter than the U&lc text. So, the card exhibits three different spacing strategies–loose, tight, and default– which seems wrong.
