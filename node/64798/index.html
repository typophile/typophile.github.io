2009-11-28 00:29:57 • Nick Shinn

If your version of InDesign happens to be the latest – CS4 – you can simply use a GREP style to toggle a font for digits only.

In short: create a character style, only defining the font for the alternate digits; then, in the appropriate paragraph styles, create a new GREP style. Coincidentally, these are predefined with “\d” – i.e., “all digits” – so all you have to do is set the character style.

And for older versions of ID, I’d suggest creating a character style and simply use search-and-replace to find all digits – use “^9” as search code – and replace with no text, and in formatting: your character style.

Note that both options heavily rely on the fact that the digits should have the regular character codes (as you are only changing the font). If you have a font with both lining and oldstyle digits, you cannot use this trick and have to manually replace each digit one by one.

Another option is to have someone (or ask the foundry) to generate a custom version of the font for you with the oldstyle figures in the U&lc font. I used to do this for some of my book work – after either ascertaining that the standard EULA allows such modifications (i.e. Adobe fonts) or contacting the foundry for permission to do so.

It was precisely this frustrating experience that led me to develop Whitman with two different figure style fonts and convince FB to release it that way. (This was prior to OpenType.)
