2010-05-31 10:13:48 • David Waschbüsch

There are several formats commonly used for web graphics – JPEG (normally used for photos), GIF (normally used for images just containing a few solid colors, like some logos), and PNG (a newer format with some advantages but not supported by older browsers).

I’d suggest saving your logo as both a JPEG and a GIF – using the highest quality settings – and see which one looks best.

Start with the highest-quality original image when making your web image; don’t make a copy of a copy of a copy.

If you’re using Photoshop, use the “Save for Web” command. Once again, use the highest quality settings. This will create a larger file but it will also be better quality.

I don’t see the artifacts when the image is uploaded to tinypic either.

It has to do with the way that the site compresses the image. If you look at the size of the image you posted its 8kb, while the one you uploaded is only 4kb. The site may be trying to compensate for the small file size and that’s creating artifacts.

If you posted a .png the site re-rendered it as a .jpeg. I would suggest making a 400x400 px version and save as a web format. Choose jpeg and put the settings at maximum. Post it again here and use the preview to see how it looks. If its still bad check the image information and see how the site is handling it.
