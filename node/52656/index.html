A little searching brought this up from an Adobe Technote:

Type 1 fonts don’t list in Creative Suite 2 and 3 applications (Windows XP Professional x64 and Vista x64) Issue

Type 1 fonts installed in the Fonts control panel on Windows XP Professional x64 or Windows Vista x64 do not list in the font menus of Adobe Creative Suite 2 applications.

Fonts display as expected in other Windows applications (for example,Character Map, Wordpad). Solution

Move copies of the Type 1 fonts to the Common Files\Adobe\Fonts folder.

  1. In the Fonts control panel, select the Type 1 fonts that you want to move.
  2. Select Edit > Copy.
  3. Open the Program Files (x86)\Common Files\Adobe\Fonts folder.
  4. Select Edit > Paste to copy the font files into the Adobe\Fonts folder.

Additional Information

Type 1 fonts installed in the Fonts control panel on Windows XP Professional and Vista x64 do not list in the font menus of Adobe Creative Suite 2 and Creative Suite 3 applications. Creative Suite applications can access Type 1 fonts installed in the Adobe\Fonts folder.

OpenType fonts from Adobe, both TrueType- and OpenType-flavored (.TTF and .OTF) display and work as expected in Creative Suite 2 and Creative Suite 3 applications when installed in the Windows Fonts control panel.

Type 1 fonts come with two files (at least): a PFB file that contains just the outlines, and a PFM file that contains metrics and kerning. It seems one of the two is deleted/moved/managed and the other stays put. I can imagine your system getting upset about that.

Installing/removing fonts under XP (my system) copies and deletes both files at the same time.

If you intend to use the fonts exclusively for CS4, you can locate the CS4 Fonts folder – somewhere inside the CS4 shared stuff – and put the Type 1’s there. They won’t be manageable, though.
