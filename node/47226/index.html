2008-07-10 11:35:32 • Tim Ahrens

I just remembered a workaround I once did:

First, do the autobold effect to a lesser extent that you want. For example, give the parameters for weight in(de-)crease half the value.

Hopefully, this retains the node structure because no overlaps occur.

Then, use MM-extrapolation to double the effect. You would generate an instance at -1000 or at 2000, depending on your setup. This should lead to exactly the same result you initially wanted, but without removed overlaps.

If halving the values is not sufficient (because it still creates and removes overlaps), try a third, or a quarter, amplifying the effect accordingly in the end. The more extreme your amplification the more rounding errors might come into play, however. Note that when extrapolating you should always try to work with full numbers (i.e. thousands) because then at least this step is lossless.

Adam, that is not what I was proposing but it is quite similar.

While I suggested to remove the weight to a lesser degree (say, –5 instead of –10) in order to avoid overlaps and then amplify the effect, you would do the effect in the opposite direction and then reverse it. Both using MM-extrapolation.
