Because I’m not sure I want to use the Th ligature in my custom version of Adobe Garamond Pro, I tried making it a “discretionary” ligature. That is, I cut the substitution lookup for T_h from the liga feature and then pasted it into the dlig feature. (The same for T_h.caron and T_h.commaaccent.)

The features then worked the way I wanted in FontLab’s own Preview/OpenType panel – and also, when I had generated and installed the font, in TextEdit.

But in InDesign 2.0.2 – I know, very old! – the Th ligature still shows up when only the “standard” ligatures are on (Basic Character Formats/Ligatures).

Anyone have a clue why this is happening? Is it a bug in this old version of ID? (I’m gonna have to update the program pretty soon anyway.)
