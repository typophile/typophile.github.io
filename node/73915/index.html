At font layer any style is possible for a letter thus instead of the k/K shape क/क stylisation can be put. The font designer connects the new shape to the same code point. Agreed ... apart from the 'let's replace a random glyph with a totally new shape', that is. There is a historical reason the glyph K has the form it has, going back to (approx.) the 7th century B.C. Could this method be expanded to other non-caps keys ('non-priority' ones) also by allowing other letter styles besides only lower and upper cases of the caps key? Sure. Historically, the Caps key is only used to toggle between capital and lowercase, and thus behaves slightly different than, for example, the Shift key. But, for example, the Alt key (Opt on a Mac) serves another purpose -- accessing character codes that cannot be entered on a regular keyboard. It's also possible not to get 'a character' at all when pressing one or more keys, but instead perform a system command (whatever Ctrl+Alt+Del did, it did *not* send an ASCII code). Assigning character codes to any key, or key combo, or key sequence, can all be done in software. It's the basis for custom keyboard mappings such as, oh, AZERTY -- French, QWERTZ -- German, and Dvorak. These are examples of (basically) moving the standard ASCII keys around, most allow access to a few additional characters (ß, ä), and some have additional software support to easily create accented characters. There is no reason to stick to ASCII keys either -- Cyrillic, Arabic, Hebrew and InScript keyboards send totally different codes. The caveat here is the connected computer (actually, its Operating System) must be set up to expect 'other' keycodes -- it's no use sending a command 'फ़ोल्डर बनाना' to a Western computer, and in reverse, typing 'mkdir' may simply not be possible with 'another keyboard'. The command 'mkdir' will not be understood if it is 'spelled' 'mकdir' -- where spelling means, for this discussion, the actual character codes sent in sequence to the computer -- U+6D, U+915, U+64, U+69, U+72. To be properly recognized, the codes should be U+6D, U+6B, U+64, U+69, U+72 However, this is totally independent how these codes are represented on your screen. If you hack your fonts to see "エ❤の♕芠" instead, nobody will care.
