2006-07-29 21:33:14 • Miss Tiffany

I would not recommend any kind of auto-tracing task for creating the vector-based parts of any glyph.

Perhaps one can make an exception for type designs that have an intentionally rough look, such as a dry-brushed script or “antiqued” letter.

Ultimately if you want the minimum number of points possible then you have to hand-digitize them. Items like extrema points (where the direction handles extend in absolute vertical or horizontal directions) need to be used whenever possible. You’re not going to get that sort of thing through auto tracing and auto reductions on points.

2006-07-30 02:29:57 • thierry blancpain

When you use extrema points, while also keeping point count at a bare minimum, it actually makes it easier to tweak and alter glyphs and stay consistent about it.

Using extrema points will also give you better results on autohinting within FontLab or Fontographer. The generated fonts will be interpreted better in Postscript or TrueType standards.

Font editors will allow you to “recalculate” a glyph to change point location to the extreme edge of a curve and have direction handles pulled to extrema horizontal or vertical directions. But that process, like auto-tracing, can be hit and miss. It’s better to draw the points in extrema when the original glyph is being designed.

Naturally you’re not always going to be able to use extrema points. For certain curve shapes you’re going to need more anchor points to get the kind of shape you want.

The best drawn fonts all use extrema points whenever possible and use the least amounts of points as well. This standard of drawing is one of the easiest traits in telling the difference between a professionally digitized typeface versus some low quality font available from many free font websites.
