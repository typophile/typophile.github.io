Those different Font window options are just showing you subsets of possible characters you can add to your font. When you select one of these views, you can build characters in that subset. If you switch the view to a different subset, you can build characters in that subset. Any characters you have already made that are not part of the current subset will be displayed at the bottom of the Font window in the order you created them. (You can change the order in the Index Mode view.)

I saved these instructions from an earlier Typophile thread, but not the name of the author, so apologies to him/her for the lack of a credit:

Select all the glyphs and hit copy. Then paste into a text editor, which will dump all the names with slashes in front of them. Then find/replace all of the slashes with line breaks in Indesign, Textwrangler, etc. After that add a header and you’re done.

**

You can always rearrange the glyphs in Index Mode and then Save the Encoding file. Relaunch FontLab and select the new encoding scheme under Names

In Index mode, you can drag the glyphs around and rearrange them. Once you have an ordering you like, choose Glyph > Glyph Names > Save Encoding. (This is only available in Index mode.)

By the way, if you want to start with a particular existing encoding before you start rearranging glyphs, switch to Names mode, select the encoding, then choose Glyph > Sort Glyphs > By Encoding. When you go to Index mode, the glyphs will be arranged in this order.
