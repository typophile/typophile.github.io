asciitilde: tilde or swung dash?

June 16, 2009 • blank

I’m confused about what should actually go in the asciitilde glyph. Bringhurst claims it should be a swung dash, and some type designers seem to have done this, but in most fonts the asciitilde is either the tilde diacritical mark, sometimes modified to be a little taller and wider. Do people using it for mathematical similarity expect to see a swung dash there? Are people actually using it as a diacritical? Do end users really care at all?

Swung dash or something like it. It shouldn’t be raised like the tilde character U+02DC, and needs to be large enough to stand out in technical text, but may or may not be as wide as the swung dash ⁓.

The character is used in programming languages, in which usage it is best if it aligns vertically with =.

I can’t think of a use for this character in which it would be desirable for the glyph to be raised to the height of a diacritic mark.

From the MS character design standards:

* ASCII tilde Unicode: U+007E

  Design : This character is used in mathematics as an operator for 'is proportional or similar to'. This character is also commonly used in text as a sign of approximation. The double tilde U+2248 is the correct mathematical operator for ' is approximately equal to'.

      Example of common usage : One inch is ~72 point in traditonal typography.

  This is not the same design and character as the lowercase spacing tilde diacritic.

  Alignment : Vertically centers on the plus sign.

  Advance width : Advance width should be the same as the figure space width.

  Spacing : This character should space between figure zeros. </em>
