Just to warn everybody, after I had a lot of troubles with this and deleting the font cache innumerable times, which did not make the bug disappear, I want to share this. Mac OS 10.6.7 has problems with OpenType PostScript fonts. It will print them with errors and produce broken PDFs no matter if the internal PDF function is used or if Adobe Acrobat is used. These PDFs are broken forever. http://www.tuaw.com/2011/03/28/10-6-7-update-causes-opentype-font-issues/2 http://reviews.cnet.com/8301-13727_7-20048314-263.html

Now, there are two workarounds here. One can reinstall the system and install the 10.6.6 combo update from the apple support website. OR There is a work-around, to install some parts of 10.6.6 upadte onto the 10.6.7. This is mentioned here: http://i.tuaw.com/2011/03/28/10-6-7-update-causes-opentype-font-issues/ direct link: http://azlist.info/kj/downloads/ the link in the bottom of the green area.

I personally experienced following errors:

  • Fonts are replaced with bullet signs are not displayed at all.
  • Adobe PDF crashes when trying to handling the OT PS fonts.
  • The PDF with Adobe can be created, but then the print-server or the laser printer couldn’t handle the PDF and I couldn’t print the documents.

How about just installing the »Snow Leopard Font Update« via Software Update?

http://support.apple.com/kb/DL1377
