2006-09-04 22:50:05 • garyw

Manual font management is not that difficult in OSX. On your desktop, place aliases of the destination folder (in my case, Library/Application Support/Adobe/Fonts because I don’t want/need fonts to be available in MS and other apps) and your fonts folder.

Drag between them as required. Crude (perhaps, although I prefer “simple”) but effective.

Mike, one thing to consider is upgrading to an older Mac. If you look at Apple’s system requirements page for Tiger (10.4) you’ll see it runs on some pretty old iron, some models going back over five years. A few of them don’t have DVD drives, but most do. The biggest issue with older Macs running newer software is hard drive space and memory, but both can be upgraded relatively cheaply.

Lots of resources on the Low End Mac site for running older Mac hardware (including which old Macs are the best value).

2006-09-06 19:58:37 • Mark Simonson

This thread has made me nostalgic for some of the early, “bad old days” of Mac OS X. I actually tried out 10.0 and waited patiently for the (free) upgrade to 10.1 which Apple distributed on CD. Each version from 10.0 through at least 10.3 exhibited real performance improvements. It was one of the only times I can think of when OS updates actually made things faster. With 10.4, features like Spotlight and Dashboard probably eat up some resources to overshadow any optimizations. For an older Mac, I think installing 10.3 off CD would be reasonable. If I remember correctly, it takes two CDs. The current state of the art software(FontLab 5, CS 2) will be compatible, and the next generation (like Adobe CS 3) will probably have hardware requirements such that it would never run on your old system anyway.

For a lot of professionals, the trick is to wait long enough for any major issues to be worked out with new versions, but stay current enough that you don’t get left behind as support drops off for older products.
