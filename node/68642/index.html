2010-03-22 21:55:02 • dtw

Arial Unicode MS has further design errors. The double-width diacritics are positioned incorrectly, meaning that a correctly encoded string with a diacritic mark that is supposed to go over or under two characters will display with the diacritic mark shifted one letter-width to the left.

One can also point to numerous design flaws in the non-Latin components that render the font unusable for serious typography in those scripts, but the s-cedilla/comma issue and the double-width diacritic issue are egregious outright errors. I had the impression Arial Unicode MS is basically abandonware now, with no plans to correct these errors or update the font at all.

This is probably all for the best. It’s better to extend Arial’s character coverage gradually in careful, thought-out steps than to try to fix something that is fundamentally broken, namely Arial Unicode MS. Rather than duplicating Arial Unicode MS’s approach of including everything from Arabic to Chinese in a single font, the current approach seems to be developing a suite of Arial-compatible non-Latin fonts each having Arial glyphs for Latin coverage. This makes more sense. The logical strategy would be to at least match the character coverage of Arial Unicode MS through the extended Arial and its family of compatible non-Latin fonts, so that we can leave the Frankenstein that is Arial Unicode MS behind. Just how realistic it is to expect all those Arial Unicode MS fonts on all those computers to be replaced with a new and improved Arial-led family of fonts remains to be seen.

I admit I don’t know much about this situation, so please do come correct me if you know more about this.
