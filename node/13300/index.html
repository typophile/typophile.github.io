An upgrade to Tiger was the straw that broke Suitcase's back for me. It wouldn't render common fonts in my browser, took forever to activate a font, and finally locked up my machine. I've been giving FontAgent Pro a try and, though I don't love its interface, it has not caused me any problems. Unfortunately, when I walked away from Suitcase, I also had to walk away from the font sets that I created for it. I spent many hours sorting thousands of fonts into sets according to (my best assessment of) their general category -- slab serif, gothic, geometric, symbol, script, etc. This upfront work paid off every time I went to the well to make a new font choice. All this got me thinking... Why don't font files include meta-data that specifies their category?! Or how about a database of fonts and categories to which font management programs could automatically connect, as iTunes does with CDDB to identify tracks on a newly imported CD? Pipe dream? Or maybe I could create this myself and make thousands of dollars?!
There has been an initiative along these lines, the Panose System, so I guess you have to come up with something more clever to become a millionaire (sorry). More info: http://en.wikipedia.org/wiki/PANOSE

I tackled Panose properly for the first time last year, for the MS ClearType Font Collection. It isn't the easiest thing in the world, so I'm not surprised that few font developers take the time to do it properly. And it does need to be done properly: you can't cheat with the Panose system and, especially, you must not make guesses about the correct values based on interpreting the Panose classification terminology. The terminology used by Panose is largely unrelated to the ways in which similar terms are used by most typographers and type designers; I'm not sure why, and I think it would have been better if Panose had stuck to a purely numerical system and avoided 'descriptive' terms completely. Panose is based on careful measurement of various aspects of key glyphs in a typeface, and calculations based on those measurements. Sometimes, the resulting values may correspond to Panose descriptive terms that are not at all how you would describe your typeface, but it is the values that are important to the whole Panose idea, not the terms.
