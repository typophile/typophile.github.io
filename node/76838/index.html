Crisp Text, a Google Chrome for Mac OS X extension that turns browser font antialiasing off and on

December 1, 2010 • twardoch

A bit of shameless self-promotion: I’ve just written a small extension for Google Chrome for Mac OS X that can aid Mac users to select webfonts that work well on Windows. It’s called Crisp Text and installs a button in Google Chrome that allows the user to toggle the text rendering on any web page between unhinted antialiased (the Mac default display) and hinted black-and-white (which is what Windows users will see with smoothing turned off).

It works well with webfonts, so you can navigate to any webfont-offering service that uses live webfont previews and check the screen quality in the crudest of all environments (black-and-white aliased).

It’s a one-click install at the Google Chrome extensions page (visit the link with Google Chrome, then click on Install).

After installation, you’ll see a small icon: next to the Chrome address bar. Clicking on it will turn font antialiasing off and on again. The extension is not perfect (you need to click twice on the button sometimes, and you need to activate the black-and-white rendering on every page), but it works well.

It only works on Google Chrome for Mac OS X. Windows users should use the system-wide font antialiasing switch (in Control Panel / Display).

Enjoy, Adam

David,

I’m only exposing what’s in the OS. No magic :)

Rob,

the black-and-white rendering does indeed use hinting to its full extent. It’s switching to the “old” Apple TrueType rasterizer. Verdana and Georgia render pretty much identically to what they look like on Windows. There may be minimal differences due to some potential differences in how Apple and Microsoft treats some instructions, but really really minimal.

Actually, I was rather surprised that the CSS -webkit-font-smoothing:none property (which my tool simply applies to the “body” element of the HTML page – there is no magic involved) works, and what results it delivers. I did not know that hinted TrueType display is possible in modern Cocoa apps, but well – it seems that it works.

As I explained in the Typographica thread, I do not consider the tool an “ultimate solution”, more a little helper that helps the lazy Mac user who does not own a Windows license to get a peek at what Windows rendering with ClearType disabled might look like.

Best, Adam
