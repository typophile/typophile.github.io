Keep in mind that since the bitmaps are embedded into the font at a particular ppm size, you'll only see your bitmaps when you call that ppm size. When I use the MS-Windows preview (Win XP)on a font I have with bitmaps, I can see bitmaps at 12ppm in the 3rd section that shows the Ascii characters, at 16ppm for the 12 point sample, and at 24ppm for the 18 point sample. (Remember that bitmaps are embedded at ppm size, and the sizes listed in the preview are point sizes) If your bitmaps are embedded at another size besides 12,16 or 24, you won't see them in the Windows preview. You should install the font in Windows, then try to type a few characters in Notepad or Word, using your font. As Si noted, "don’t forget to turn off ClearType and all antialiasing when testing the font" and remember that you're specifying a point size in the app, so you need to spec the right point size to hit the ppm size you embedded: (point size * 96) / 72 = ppm size used.
