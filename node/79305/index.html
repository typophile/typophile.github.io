@Tim Ahrens: Practically speaking, what does “source” mean when we talk about “open source fonts”? It doesn't mean what you seem to be thinking. Compiled TrueType fonts will do just fine - a textual source listing or a font-editor format file is not required. It means that the license has to allow me (perhaps with the restriction that I allow the same privilege to the next guy - GNU versus BSD) to put the font in a font editor, make changes, and distribute the result. That is the generally-accepted distinction between "open source" and "not open source". It's true that some software where nearly all you can do with the source is look at it has also been called open source. And this does have some benefit in maintaining the software. (You can fix bugs and recompile for your own internal use, perhaps.)
