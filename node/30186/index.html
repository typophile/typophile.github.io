Choz,

for OpenType (PS or TT) fonts, you can use arbitrary styles names in the OT (preferred) Style Name field (which is used in the Adobe and Apple font menus) but you should not use any name except the standard four (“Regular”, “Italic”, “Bold”, “Bold Italic”) in the normal Style Name field (which is used for the Windows style-linked groups in typical Windows applications).

In the OpenType preferred naming, you could have wither the family name or the style name including the cut name, i.e. it would be “Cunningham Soft”-“Oblique” or “Cunningham”-“Soft Oblique”. It really is a matter of how you want to present the fonts in the Apple/Adobe menus (three entries in the family menu with 4 subentries for styles, or one entry in the family menu with 12 subentries for styles).

Since Windows style-linked groups can only have up to four members, putting the cut name into the family field may be a better idea because then you can use the same naming for both the Windows and the Apple/Adobe menus.

Regarding font menu names and Vista:

Microsoft’s new Windows Presentation Foundation (WPF) handles font names in a completely new way, based on trying to shoehorn them into families that fit within the CSS spec’s ideas of how family members can differ. WPF is an optional install with Vista, and will also be made available for XP.

May I interject and give the opinions of a web developer and Mac user:

As a user I like Family names to be atomic, and Style (“Sub-family”) names to be diverse. This way I get a shorter font menu with names such as “Univers”, which if selected gives me the Regular style. If I then want to choose a variation of Univers, I can select “47 Condensed Light” from the submenu. Contrast this with how Arial displays. I have four Families listed for this one font! Arial (r,b,i,bi) Arial Black (r,i) Arial Narrow (r,b,i,bi) and Arial Rounded MT Bold (r). I can’t for the life of me understand how Monotype managed to screw up that last one so much!

As a web developer, I recognise that the CSS properties (font-family, font-style, font-weight, font-variant) are very crude. How does one reliably select “Univers 47 Condensed Light”? But I have found the solution is also very simple, use the Postscript name as the font-family! html { font-family: ‘Univers-CondensedLight’, ‘Univers’, sans-serif; } The above works on all the browsers I have to hand, but obviously I can’t vouch for WinIE.

Looking through Thomas' presentation (thanks!) I see that how Vista handles Fonts will be very similar to how the Mac has handled them for the past decade. This is great news as it seems to be converging on what I like (more styles, fewer families).

Lastly, I implore you to use Oblique for an oblique font, and Italic for an italic font, and not to mix the two—firstly it hinders the education of non-typophiles, secondly it looks bad to people who know about type, and third, as you have pointed out, you can’t subsequently add an italic style if the name has been used up. You can use the head & OS/2 style linking bits to get QuickDraw and Windows 3.1 to understand them without compromising the nice style names. CSS explicitly distinguishes between regular, italic and oblique—please support this and don’t make things more difficult for browsers/OSes or us web developers :-)
