InDesign CS2 Font Problem

January 12, 2007 • growland

We are seeing missing font errors in files that have Type 1 fonts as well as Open Type fonts in the file. We are moving to an Open Type environment and have been converting the stylesheets to now reference the Open Type version of the font. We are using Font Book 2.0 in OS 10.4.8 as our font management tool and have found that even when the fonts are active and being used correctly in the file (stylesheet applied) the typeface is subsetting from Regular to Medium. It throws a ‘Missing Font’ error and we don’t know how to get around it. The font is accurate (InDesign substitutes the font with the correct one but sees the typeface as different) behind the pink text (our designers and QC staff would know what the font looks like but our Editors don’t) but because there’s pink highlight, it is assumed the font is missing.

Has anyone else seen this? Is there a way around it?

So, you have a single family that has both “Regular” and “Medium”? If so, InDesign tends to treat these as the same thing, and may get confused. What’s the type family in question?
