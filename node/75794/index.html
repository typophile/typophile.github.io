October 30, 2010 • agisaak

I've added the character Aogonekacute (along with other ogonek+acute vowels) to a font. I'm wondering about the proper way to name this character in the final font. Currently, I'm using A_uni0328_acutecomb, but was wondering if this is acceptable or if there are compelling reasons to instead use A_acutecomb_uni0328 (or, for that matter, uni004103280301 or uni004103010328). (The ccmp feature will build this character from any appropriate combination of A, Aacute, Aogonek and acutecomb/uni0328). Any advice would be appreciated, André
was wondering if . . . there are compelling reasons to instead use . . . uni004103280301 or uni004103010328). Indeed there are. I forget the name of the document Adobe put out -- maybe someone can add that -- but if a character is *named* with the correct components as a Unicode number, it will be extracted from a PDF in proper Unicode. The program picks up on "uni", counts four digits for the first character, the next four for a second, and the next four for a third. I believe 12 digits (three characters) is all it will parse. I would use 0041 + 0328 + 0301, e.g., uni004103280301 as the name, which lists the bottom diacritical character first. But I believe it will parse with either diacritical given first.

This isn't the document I was thinking of (I'll try to rummage it up at work Monday if you are still interest), but does show Adobe's thoughts on the matter: http://www.adobe.com/devnet/opentype/archives/glyph.html Quoting part: The glyph name is expressed as the prefix "uni" followed by two or more sequences of four hexadecimal digits. Each sequence of four digit hexadecimal digits indicates the Unicode scalar values of the standard Unicode characters, in order. For example, the character LATIN CAPITAL LETTER EZH WITH CIRCUMFLEX AND GRAVE, which is not in Unicode, should be named "uni01B703020300", since LATIN CAPITAL LETTER EZH is at U+01B7, COMBINING CIRCUMFLEX ACCENT is at U+0302, and COMBINING GRAVE ACCENT is at U+0300. A maximum of 7 name components is available with this format, due to glyph name length restrictions.
