Alternate glyphs does not receive Unicode designation. This also applies to small caps*, petite caps, swashes and so on – which are also alternates, although within a well established typographic standard. This is not a limit defined by the FL table in use, but come from Unicode itself – which maps just uppercase and lowercase glyps for Latin script. So, it will not change even if you adopt other tables for generate codes. Thus if the glyphs you do not got coded by FontLab are all *.alt*, the result you got is actually the excepted one. * Sidenote: Unicode has some small caps mapped, but for special uses (like phonetics or maths), and not as alternates to A/z set.

Unicode does not map codepoints (Unicode values or indexes) to glyphs. Unicode maps codepoints to characters (bits of information). The glyphs "A", "A.smallcap", "A.swash" or "A.titling" are all representations of the same character. Unicode does not deal with representations of characters. A fontmaker can provide different representations for the same character (via layout features) and can use the same representation for different characters (e.g. capital Omega and Ohm sign) with the cmap table. See: [[http://developer.apple.com/fonts/whitepapers/iuc15cg.pdf|Character-Glyph Model]]
