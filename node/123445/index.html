I wrote a script that takes as input a font (ttf, otf, ttc and even dfont) and that lists features. I provided no script to modify a font in any way.

In order to “fix” the fonts, you need no script at all.

To copy the GSUB tables, when the EULA allows it, you open the two fonts with FontForge, the one with the missing GSUB tables and the one from which to copy. In the font with the missing GSUB tables you select “Element > Font Info > Lookups” and then “GSUB” and you click “Import”. You will then see the lookups from the other font. You select them and then click OK. That is it.

That assumes that the glyph names are the same in the two fonts, which I guess is good practice for a font designer if one font is the regular and the other is the bold or the black.

With a ttc font, you probably need to split before; Apple’s ftxsplitter does that.
