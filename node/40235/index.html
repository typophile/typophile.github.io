Fontlab 5: Copy metrics to composites does not work for some glyphs?

December 20, 2007 • Sebastian Nagel

Dear Typophiles,

I am wondering about a problem with one of my fonts I am editing in fontlab: When opening a glyph in metrics window and clicking on it, usually there is a small arrow below the glyph which allows me to copy the metrics to its composites. But now, this small arrow is only shown for some glyphs (for example c, d, e, h, …), as for others they are not (like a, g, dotlessi, k, …). I have checked if the composites i want to apply the metrics to are really composites of that glyps - they are, so I just don’t know what I am doing wrong.

Has anyone encountered the same problem and found a solution?

Thanks for your help Sebastian

Addition: This is only occuring with one special font. When trying with another, everything is fine.

Hello,

if you can’t copy the metrics from ‘a’ to e.g. ‘adieresis’, check that ‘a’ is the first component in ‘adieresis’.
