>Thanks for any help that comes swooping in. Understanding TT instructions for hinting, starts I think, with knowing that it's based on a stack and a graphics state (both of which are poorly documented, I think). This you should also understand, that before either your stack of hand made or auto hints begins to act when the OS calls for a font of some ppm size, that the TT OS software has 'hinted' the em top and bottom, located the baseline in pixel space, as well as the origin and the set width of each glyph (undocumented). Then one has to understand that no point moves properly unless its movement is relative to another point (documented nowhere I've read). Then the instructions you or the auto-hinter make, start from the origin point and work into the glyph's points, using the origin as a basis for the rounding of one (or more) of the points on the glyph, from which the instructions further down the stack round the other points. It would be so cool if the folks who were flogging this auto-hinting process would show one proof that it is useful for more than "sloppy work" but I'm not holding your breath. I think, in case you can't tell, that knowing "what they do" is pretty important if you want to know "how the tools work." Cheers!
2010-06-22 12:42:54 • dberlow
David, in FLS autohinting is a two-step process. First, it analyses outline and generates type 1 hints, then it converts type 1 hints into "visual instructions" which you can see and edit in TT hinting mode. In between is a step of generation and optimization of font-level hints (zones and stems). Last and final step is conversion of "visual instructions" into native TT hinting code. All intermediate steps are visible and very simple to check.

2010-06-24 21:33:24 • Becolt
>How do you decide where to place links and anchors? The instruction stack begins with reference to the origin point 0,0, as this is rounded by the font software before the hints begin. Then the instructions are an orderly march across the glyph, and/or an orderly march up from 0 to the top of the glyph and down from 0 to any descent. If your reference points and march are not orderly, and the stack is just running around moving points in no pattern, then stop it! Cheers!
