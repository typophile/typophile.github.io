Get'n Organized!

April 19, 2007 • AndrewSipe

Well, I’m in a crappy mood this morning, I came in and was told that I needed to make some visual changes to a website I designed the end of last year. It’s been well over 6 months since I’ve looked at this site.

So, I went to the folder where I have everything and I mean everything. If this were a physical place, I could imagine boxes piled high and in every box there’s tiny pieces (and lots of duplicates with minor imperfections.)

It’s clear that when it comes to saving files I get into a very simplistic mode (I’m just surprised I save often enough to not have to worry about NOT having a file.), but this mess clearly proves I need to go the extra step.

I was wondering how you guys tackle the organizational problems of design in our computer world. Do you sort everything into specific folders, give them version names, dates, letters and numbers? OR do you delete all the development at the end and save only the major components.

I could use any helpful insight.

I rely on the “view by date” view in the Finder when I’m trying to reconstruct what I did on past projects. (For this reason, I detest programs that change the modification date of files when you only open them.)

I also make a folder while I’m working on a project labeled “older files” or similar. I move files into that folder as they are superseded by newer versions. If I need to back track, I know where they are.

2007-04-19 15:21:50 • Mark Simonson

At the end, I only save the final files, but up until then I usually save at least one version back, just in case. I have a large hard drive on my iMac but also back up files (the main files) on an external hard drive, and when the job is complete I burn a CD or DVD for additional backup. I’ve had hard drives (both internal and external) crash on me so I’m very careful to have all the main job files in more than one place. By main job files I mean the Quark or InDesign files, not necessarily all the jpegs or tiffs. I usually date the folders but not each file, altho I’ll tack on a version number at the end to keep track. (I should add that I design books so each job usually has multiple files, one for each chapter).

My office/apartment is a mess but my computer is very tidy. Next to nothing on my “desktop” and files all where they should be and clearly organized. I usually have separate folders for text, images, layouts, pdfs.

It may take a bit of extra time to be vigilant while you’re working but it’s worth it. One friend suggested you schedule your backups, say tuesday mornings or something so that you’re never more than a week behind.

I usually also have a folder with a lot of redundant and obsolete versions of files. After I know where I’m going, I usually make a folder inside the folder and name it something like ‘old’, and put everything in there, exept the latest version of the file(s) that I am actually using. Especially handy when visiting the folder later, like you are doing now. Sometimes I add something like ‘finish’ or ‘printversion’ to the name of the file that’s being used. This is how I do it for freelance work and my own work. At the company I was working for a year ago, we really had a system for storing the data, because everything was backed up, and put into the archive. Those filenames had date, order number (especially), year, name of client, etc. That way we could search the number or name in the archive-computer, and know on which CD/HDD/Server the files were stored for later use/reprint.

With work like InDesign, I sometimes make folders inside a ‘master’ folder, with names like ‘images’, ‘resources’ (original text-files etc.) and have the InDesign-file itself in the master folder. Basically what InDesign does when it makes a Package. This is also the way we needed to do it at the company I mentioned before.

2007-04-19 19:05:04 • AndrewSipe

The simplest is to tack on a suffix like .1, .2, etc. Sometimes it becomes .2a and such. I don’t add dates because it makes the file names too long. But I’ll date the folders - when I have my s**t together. That’s helpful also for reference with the client - you can be sure you’re talking about the same version.

Because I do books, I usually set it up as follows: Folder: book title + date (or sometimes Layouts1, Layouts2) Files: a.frontmatter_pp1-15 b.chapter1_pp16-35 etc.

Then .1, .2 as the revisions happen. Often I’ll put a cap F at the end of the final version (the one that went to the printer). But then, of course, there are more changes so it becomes F1, F2 etc.

The trick too is to remember to do a “save as” before you start reworking the file so that you have preserved your earlier version intact. And back it up elsewhere in case you mess up.

2007-04-19 19:34:35 • pattyfab

I organize my stuff into a main folder with the project’s name and the main file, and subfolders called texts, images, illustrations, recherche, etc.

While working on the project I generally use the option “open recent files” to get to the latest version of my document and sometimes I take an extra look onto the date (especially when working with the book palette).

Then, first thing I do, at least at a new day, is saving the file under a new number (filename_1, filename_2, etc.). Additionally I have a second version of the file (filename_1bak, filename_2bak). This has really saved me sometimes, it doesn’t happen very often, but if, I was always very grateful about this second chance. With these two versions I can save the file now and then by “save as” (filename_1 “save as” filename_1bak and vice versa the next time). At the end of the day I’ve got two identical versions (file_2 and file_2bak) so it wouldn’t be that bad if I’d mess up one of them the next day ;-) The good thing about doing a “save as” in InDesign is that this minimizes the file size as well. Now and then I drop the older files into a folder called “older versions”. And in the evening I (try not to forget to) transfer the latest versions to an external hard drive.

A PDF I like for quick overviews while working.

2007-04-20 01:46:09 • tina

I, too, am not the most organized person with my files.

My method has evolved like this: I open a folder for whatever it is (InfoBooklet 07-08). Inside, I start the layout and save each version, marking the version with _1 _2 _3 as applicable. Then, the version chosen to be the working layout gets the number removed, and the numbered versions go into a folder of their own within the main folder. As I add stuff to the document I’ll create, all within the main folder, folders for logos, pictures, translation, and so on. I also create an email folder for the project because most updates come to me by email. So even if I’ve managed to delete or screw up something along the way, I’ll have the original update in the email.

For finalized files, I put _final. I make a PDF of it, and package the files. Sometimes I need to have a different final version for web and for print, and I name them accordingly.

But I also don’t work for multiple clients. And my publications are only updated once a year, so it’s pretty straightforward.

Of course, I also heard of someone who everyday created a folder with that date and put whatever they did that day into the folder. I know I would never maintain that, and figure it wouldn’t be a practical way to track specific project components.

I am not always willing to go the extra mile for organization so the model I use is to start the project with a name ‘projectx’ or what ever & then put a 0001 & then the file extension. When a project takes a really new jig I will often just skip to 0100 & keep going from there. And then 0200 for each new step after meeting with a client. Itterations within a thread sometimes take letters too so ‘0123g’ is a name I can build up to. So far I have never needed to get past 2000.

The reason I like this method is that it is good for ‘threaded’ projects where you diverge into a few ideas & cut off some options & then spin some new ones out from a base design. Really linear projects would probably not be hurt by it either though. I suppose if you really wanted to keep a thread’s integrity you could always use a001, b001 & so on. But I have not found that I benefited from keeping things that reified.

when color is an issue -usually in the second half of the project or near then end when final art is being made I will indicate that with a .rgb or a .bwrgb or .bwcmyk or .cmyk - so far I have not needed to archive lab color stuff. 2007-04-20 03:05:16 • ebensorkin

I work at an ad agency and I am pulling my hair out trying to work through my colleagues folder structure and naming conventions. I believe each project should follow the same protocol so anyone (freelancer, colleague, you 6 months later) can jump in and be familiar with the logic. This thread has been really insightful to me.

I’ve found the best approach for me, at the expense of disk space, is to version out in rounds. This is an example of a website:

top level / [copy][concepts][production] concepts / [PSDs][JPGs][FLAs][PDFs][Assets] PSDs / [round_01][round_02][round_03][round_XX] round_01 / 01_homepage.psd, 02_aboutus.psd, 03_clients.psd JPGs / 01_homepage.jpg, 02_aboutus.jpg, 03_clients.jpg PDFs / filename_date.pdf

Its probably over the top, but each time I get a new direction, or if there is a shift in design I duplicate the round. The old files are left intact*. I never change the file name because what does “new”,“newer”,“this_is_really_the_final finalFINALFINAL” really mean? Believe me, I did this for a long time, and my new approach has saved me hours of unnecessary work. The file name always stays the same from round_01 to round_XX.

This helps out tremendously when you are saving out for client/internal presentations. You don’t have to create rounds in your JPGs folder. Its the same file name in every round. Then, if you have to create a PDF for the presentation, the 01_,02_,03_ prefix allows you easily order your presentation. Its already in the order you determined in your PSDs.

  • this has saved me so many times. you can easily re-trace your steps and show the iterations to clients who can’t easily see the evolution of your designs. Or if they want to go back to “the design you had a couple of weeks ago.”

2007-04-23 16:06:14 • mauimatt

I simply make a folder with the clients name, marking it green (using Mac Os X) as to indicate that I am currently working for that particual client. Within the client’s folder I make a folder with the project’s short name. Say I am working in Illustrator. I begin by naming the file I am working on “xxx_1.1.ai”. Whenever I make any big changes I save that file again as “xxx_1.2.ai” and so on.

Withing the project folder I have a folder for all working material like unedited images, documents and other stuff. I usually call that folder “Material”. For images I import or place in the final project I have a folder called “Images”.

So lets say my client’s name is Cacadoo and the project’s name is Poster:

Cacadoo

  • Poster – Images – Image1.eps – Image2.eps – Material – Textforposter.txt – Image1.jpg – Image2.jpg Poster_1.1.ai Poster_1.2.ai Poster_1.4.ai Poster_1.5.ai

Then when I am ready to send the file to print or the web developer I make a folder called “Final” and save my final document there as a PDF or .PSD.

Of course there a scenarios that are a bit more complicated but this is my basic principle of project organization.
