Choosing a typeface for an engineering/architecture/construction firm

October 2, 2006 • JonnySchneider

Hello, I'm making an important typeface selection for a logo that belongs to a recruitment agency specialising in engineering, architecture and constructions. Their current logo (mis)uses Bauhaus. [attached] It's probably okay for the mark, but definitely doesnt work for the company name at small type sizes, not surprising for a display face. There are a few key considerations:
  1. It must be legible across print and web delivery, specifically at small sizes in newsprint.
  2. They dont want to move too far from the existing design - they want me to 'clean up the mark and use a typeface that is more legible'
  3. The new logo will set the scene for a mini-branding in a few months, where a new website and stationary will be produced.
For newsprint ad campaigns, so far I've mainly used DIN for display settings and Oficina Sans for body setting. I know its seen as a cliche to use 'square sans' typefaces like DIN and Meta for engineering/architecture firms, but given the requirements, I think this is a fitting direction that is not likely to upset a fairly conservative client. So far i'm looking at: Klavika Because its handsome looking, will print well on web-offset, and offers a lot of weights and variations which should make light work of stationary design later. Locator For similar reasons to Klavika, except that I think this looks too much like a Real-Estate magazine face. Anyone have any other suggestions, or thoughts on my direction so far? BTW, i didn't design the original logo! Thanks, Jonny
