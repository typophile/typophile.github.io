2007-09-05 12:49:08 • dberlow

There is no UI for it (so you will have to use some Python script every time you want to hide or show outlines), but that is bit 0x00000200 or the options.PaintStyleEx:

to hide outline: options.PaintStyleEx = options.PaintStyleEx | 0x00000200

to show outline: options.PaintStyleEx = options.PaintStyleEx & ~0x00000200

This is a full list of “constants” to control apperance in the GW (all changes should be made before GW is open, use on your own risk, no additional description, no support provided, no questions answered)

2007-09-08 20:47:06 • William Berkson

Sounds good!

One thing I’d like to point out though, and I hate to seem like I’m trying to break the record for sounding like a broken record, but…The best way to preview glyphs while editing the outline should have two pair-o-modes; one where the user has turned off the nodes, in which case they should never appear, and one where the nodes are on, and they don’t disappear. The other mode pair being that the contour should fill with and without showing itself.

So what’s the broken record? FOG had this right. One turned off the nodes and could edit them “sight unseen”, i.e. the nodes never appeared when “turned off” even when selected. Later, PvB’s very first customization of FOG was a cmmd key to show a filled glyph in the glyph window. We felt this last little thing was the completion of a perfect letter drawing interface.

With the nodes hidden, a filled outline could be edited with the appearance of magic occurring — the designer waving and clickin' around a glyph responding to seemingly invisible “strings”. Cool as it was, it was also the most transparent and comforting method I ever used to draw and shape letters: seeing whatever visual crutches one needed, or none, with a single key while still being able to edit.

Cheers!
