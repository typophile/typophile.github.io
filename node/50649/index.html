2008-10-20 02:02:01 • acnapyx
Well, they work for me. I give my dingbat glyphs special names for several reasons. I don’t want to start calling symbols “a” that do not represent the glyph “a” just because it is technically convenient. I always design type 1 versions first (so try the pfb/afm combination). Most software should handle the type 1 (PostScript) files correctly. When I generate ttf fonts from type 1 fonts, programs such as FontForge put the specially named symbols in an area outside the regular glyph area, and so, renaming may be necessary for some applications, as you pointed out. In summary: try the type 1 version, and let me know.
