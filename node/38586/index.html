You can only remove control points if both BCPs within a curve segment are retracted.

Not quite sure if this is what you meant. Note that it is always two BCPs that belong to one curve segment – the notion that the two BCPs on one node belong together is wrong.

In case both BCPs of a curve are retracted (i.e., it is in fact a straight line) you may be able to solve this quickly by using the remove overlap function.

Btw, there does not seem to be anything technically wrong with retracting one BCP within a curve. You may encounter problems when converting them to TrueType curves, though.

This is typically something that designers want to control easily. Because FM is in the first placed developed for internal use here at DTL, we constantly try to improve and simplify the behavior of the BE and IK editors. The consequence is that the wish list is always growing, of course. Recent improvements for instance include the placing of (magnetic) guide lines by simply clicking on points and the enhanced control over BCP’s.

In the first versions of FM removing Control points naturally resulted in a straight line between two adjacent (Smooth) Anchor points. It took some discussion to convince the programmers that in the design process we also could use an option for directly placing Control points on the Anchor points, without the necessity to use the coordinates editor (illustration 1).

At the end the solution was to place the Control points on the Anchor points by simply ‘deleting’ the BCP’s (illustration 2) and to use the ‘Change Label’ option for the conversion from curve to line and vice versa (illustration 3).
