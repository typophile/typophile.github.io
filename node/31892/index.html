Chris, I had to read this several times to figure out how you could think there is any relationship between ppi and file size. I think there may be a cultural gap here that’s hindering communication between people who just use images and people who muck about with file formats. A Photoshop user might think of pixels as just the things visible on the screen, and might think the underlying data are something different. This would seem like a sensible assumption, because it corresponds to the reality of preparing image files for printing. But actually file formats such as JPEG and GIF consist of “abstract” pixels that ideally correspond to the pixels on your screen. There’s a file header that contains various information about the file (including, in JPEG/JFIF format, 4 bytes that define the ppi values); everything after that file header is just a compressed or uncompressed array of pixels. When you open up a JPEG in Photoshop and then select “Image Size”, the display tells you the dimensions in pixels at the top and the dimensions in your units of choice (inches, centimeters, etc.) below. The pixels in the upper half of the Image Size dialog are the image data - no matter what dpi the Resolution box shows, the amount of image data is equal to the pixel dimensions.

When Photoshop (or some other image viewer) displays an image at 100%, that means that one abstract file pixel is equal to one screen pixel.* When a Photoshop file is sent to a printer, though, it doesn’t work in pixels - it works in units of area. So when the file gets printed, it is converted into different information based on the dpi setting in the file. So you are correct that the amount of data depends on ppi in the world of (print) production, but Darrel is also correct in saying that changing the ppi of a file has no effect on the file size unless you also change the pixel dimensions.

Web browsers will display the image at 100% unless forced to do otherwise, like in the example from Patty’s site that started this thread. So for web use, the size on screen depends solely on the pixel dimensions, and ppi has no effect. This is very different from how images are prepared for print in Photoshop, but it’s the reason why, in the alternate-size image example that Dan posted, it’s necessary to have separate files for each size.

  • Obviously, if the image is being displayed at something other than 100%, either some pixels of the file data are being dropped out or additional pixels are being extrapolated. This doesn’t affect the file size, because the number of pixels actually stored in the file remains the same.

I really don’t know why this topic is so difficult to understand, but obviously it is, as I run across people every now and then who confuse things…

I’ll try to explain:

Lets start with print: PPI means Pixels per Inch. You have a certain amount of Pixels, distributed on the length of one inch. The more Pixels you have, the smaller they get, the sharper the image looks like. The PPI-Value set in Photoshop is only a number: It defines the relationship between number of pixels (pixel dimensions) and the physial size of the image (length and width dimensions). There is no additional information in this number. If you have a picture with 300300 pixels, and a dpi value of 300, it will print 11 inch in size. If you take the same picture, and change the dpi value to 100 (picture is still 300300px large!), it will print 33 inches. There is the same amount of pixels printed, resulting in a grainier picture with larger pixels. I think this quite clear.

So let’s switch to screen: How large is a Screen with 16001200 pixels? It differs. There are small 12" notebook displays with this pixel dimensions, and large desktop displays as well. Both have the same pixel dimensions, but the differ in: PPI. A small notebook display has let’s say 200 pixels per inch, a large desktop-display maybe 100. The Pixels of the Desktop Display are larger. What happens if you display a picture with 300300 pixels here? It will display in different sizes. On the notebook, you will have a picture of 1,51,5 inches. On the Desktop, the same picture will be 33 inches. Each pixel of the display shows one pixel of the picture. As the notebook pixels are much smaller, the resulting display of the image is smaller too. There is (yet) no way to determine, how big a picture (in inches) is displayed on a screen. All you can do is give pixel dimensions.

So what is it about 72ppi (or 96ppi)? Well, for some purposes, the operating system has to convert pyhsical lenghts to pixels. This happens for example with type. You say “my window titles should have the size of 10pt”. pt is a physical length. But the operating system has to display pixels, so it needs to know what “10pt” is. Ideally, you would tell the system, how large your screen is in inches (lenght, height). Then, if you say “10pt”, it could display the type exactly 10pt. But until today this is theoretically possible, but practically not, as for programming reasons. So, the system assumes your screen has a physical resolution of 72ppi (apple) or 96dpi (windows), even if it is not exactly true. For most desktop displays, the value today is between 90 and 100ppi, so 96 is quite a good number (72 was quite good in the 80ies and even 90ies, but displays became better).

So: if you have a 96ppi screen, and you take a picture of 9696 pixels in photoshop, and enter the value 96ppi, it will show exactly as large as it would print (11 inch).

But all of this only matters if you want to print a picture. The PPI value is worthless on a screen, which dimensions are unknown to you (which happens on the internet or on TV). As Aluminium tells us with the tower of Babel, if you have the same number of pixels in a picture, it does not matter whether it is saved at 30000ppi or just 30. On screen, each pixel is shown 1:1, size according to the size of the actual display. If you took this picture to photoshop and print it, the printed size would differ dramatically (this does not apply to print from a internet browser, they act like displays when they print).

I know I’m late jumping into this, but…

PPI and DPI are very different things that often get confused.

DPI, or dots per inch, is a printing term that refers to the number of dots per linear inch in a printed halftone or screentint. (I’m ignoring lines per inch for the sake of simplicity.) If you print a photo at 150 dpi, that means that if you get out a magnifying glass and count the dots, you’ll find 150 of them per inch in each row.

The general rule is that it takes 1.5 to 2 pixels of data in a digital file to accurately produce one halftone dot. In other words, a photograph that will print at 150 dots per inch will require a digital photo with enough pixels to fill the space when printed at 300 pixels per inch (pip).

It’s wrong to think of ppi as being related to the file size or as being some kind of inherent quality having to do with a particular photo. What really matters are the number of pixels in the photo itself. A 1,000-pixel-wide photo, for example, can print or display at any ppi you’d like, but its dimensions will change; for example, on a monitor that displays 100 pixels per inch, the photo would be 10 inches wide. If the monitor displayed 1,000 pixels per inch, the photo would be 1 inch wide.

The number of pixels displayed by the monitor is sometimes referred to as its resolution. Most monitors are capable of various resolutions depending on how many pixels you’d like to see on the screen. The higher the resolution of the monitor, the smaller the objects will appear to be on the screen, because the number of pixels being forced into, say, a one inch measurement increases.

When preparing photos for the Web, pixels per inch are not really something to be concerned with. The resolution of a person’s monitor, as mentioned above, decides how many pixels it can display per inch. What really counts are the dimensions of the photo in pixels — a 300-pixel-wide photo will appear smaller on a higher-resolution monitor and larger on a lower-resolution monitor. Again, this is because the monitor is set to display a certain number of pixels per inch.

In Photoshop, the whole pixels per inch thing can really be ignored if you’re preparing material for screen resolutions. Like I said, the only thing that counts is if you have enough pixels in the image to fill the space. It doesn’t really matter on the Web if the document is set up at 10,000 ppi or 2 ppi, it’s all the same as long as you have enough pixels to fill the space. Pixels per inch (PPI) refers to how many pixels will be displayed in a linear inch on whatever medium you’re outputting the file to, whether this is print or on a monitor. Another way to think about it is that PPI refers to the resolution requirements of the output, whether print or monitor.

And like I said, the pixels per inch on the monitor are predetermined by its resolution settings, so it’s not a concern when designing exclusively for monitor display (like the Web). There, you just need to think in terms of total pixels, not pixels per inch. In print, however, PPI is a factor, of sorts, because of the relationship between the pixels per inch and the halftone dot frequency (DPI) and the resolution of the output device. Even there, it would be totally possible to ignore the PPI if you were sure you had enough pixels in the image to do the job, but working with pixels per inch eliminates some of the math and just makes it easier. And, of course, the print world is built around inches, picas, centimeters and other hard measurements, not pixels.

Anyway, when working with most desktop publishing apps from Quark to Photoshop, they assume that you’re thinking in terms of PPI, so lots of their dialogue boxes are phrased around that assumption, so it’s important. Again, though, it’s not a factor when designing for the Web – there, think only in terms of pixel dimensions, not pixels per inch. Unlike the print world, monitors just aren’t built around displaying things in terms of inches — it’s pixels that count. The whole 72-ppi or 96-ppi resolution of monitors, in practical terms, is something that can be ignored when preparing images for the Web.

This whole concept is fairly easy to understand once it clicks. Explaining it in an understandable way, however, is difficult.
