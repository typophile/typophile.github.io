No, they don’t. When working in Fontographer long ago, I’d put in a 0 (“zero”), which caused the number line to be commented out. I forget exactly what, in FontLab, turns the whole thing into a comment line, but it can be done.

BTW, we also use to assign our own, unique names to Type 1 fonts in the old days. Some RIPs would cache fonts, and if your font was slightly different but had the same name, the printer’s RIP might use the wrong font (or metrics). Our New Baskerville roman, for instance, was named B0500000, with a commented-out font ID number. I’ve been assured this doesn’t happen anymore, but we still do something similar with modified OT fonts.
