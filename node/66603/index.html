Font Book validation bug that can affect font testing

January 24, 2010 • blank

I have found a nasty problem with the font validation routine in Apple’s Font Book. Once Font Book detects some font problems it will continue to report any font as having unspecified problems. The only way to get Font Book to start validating properly is to reboot the computer every time a font is listed as having a problem. I think I had this problem when my OT KERN feature listed a non-existant glyph, but I cannot be sure because Font Book does not always report details about the the problems it finds in fonts.

I believe that this problem exists on 10.5 and 10.6, I do not have a way to test other versions of Mac OS.

This is probably a font cache issue.
