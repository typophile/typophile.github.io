2009-11-03 17:55:06 • sergeym

In a page layout application, when you want to use smallcaps, you select a smallcap function from the UI, you don’t input an OTL four-letter feature tag. The application then applies smallcaps according to what the font supports and how it supports it: it might be the OTL ‘smcp’ feature or it might be outline scaling or it might be via some other smart font format (e.g. AAT) that the application recognises and knows how to process. The CSS discussion is about the same higher-level function approach, so instead of using a font format-specific tagging system, it will use its own syntax which can then be interpreted according to the feature systems of any smart font format supported by the browser. [And in a web authoring tool or environment the function may be presented as a UI option, hiding the CSS syntax, as exists for e.g. bold and italics in such tools.]

Also note that application layout functions call more than one OTL feature, e.g. an allcaps function might call both the OTL ‘case’ feature and the ‘cpsp’ feature. Same with CSS.

@dberlow

DB>“The syntax “invented” for CSS, as I read it, is that a typeface family is called a “font” and a typeface style is called a “face”.

A typeface family is called a “font-family”. “face” to my knowledge, appears only in the @font-face declaration. It has no special meaning, just a label with a vague connection to “typeface”. In CSS, a declaration with an @ in front of it works like a wrapper containing yet more declarations that are applied only under certain or under all conditons. For example, to apply a set of rules only for when the web page is printed: @media print{ … put the print-only rules here. For example, this next declaration: body{font-family:constantia, georgia, serif;} }

Basic CSS properties like word-spacing, letter-spacing, line-height, etc… probably don’t map in the ways you might expect if you come from a print design background. For example, there is no property that corresponds exactly to leading in CSS. The space between lines is the value of the line-height property minus the bounding box created by the choice of font-family and font-size.
