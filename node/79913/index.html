Thanks, Matt.

Not sure I’m following what you’re trying to accomplish, but character count is always going to be an estimate of some average (whatever that happens to mean).

If you get a line of content with a lot of capitals or ‘m’s and ‘w’s, then your character count is going to go way down. If you have a line with lots of ‘i’s ‘l’s ‘t’s and a bunch of punctuation, then your count can go way up. A 15% variance sounds pretty likely to me, if not more.

You could always specify a monospaced font. That way, character width is fixed and character count becomes very easy to calculate. That’s one of the primary advantages of a monospaced font.

You’ll still have variance based on the length of any word in proximity to the end, especially if you don’t implement any hyphenation. But it will be considerably less.
