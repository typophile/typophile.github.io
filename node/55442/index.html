Maximum character width?

March 3, 2009 • kegler

Is there a maximum allowable character width for any given glyph position? also Is there a maximum area beyond the side-bearings that a character can extend?

Yes, there are maximums defined for these. Maximum width is given by the ‘hhea’ table and is stored as an unsigned 16-bit integer with maximum capacity of 65535 (em units). Max left and right sidebearings (which are actually named ‘minLeftSideBearing’ and ‘minRightSidebearing’) are given by signed 16-bit integers, so the allowable range is -32768 to +32767 (again, in em units).

However, you should note that these are merely the limits of the OpenType data structures. The actual/practical limits imposed by operating systems, font-creating applications, and font-consuming applications may be smaller. There are also other values within the ‘hhea’ table and within other tables that probably prevent you from, say, starting a glyph at the furthest possible point left of the LSB and extending across the max advance width to the furthest possible point right of the RSB.

“Is there a maximum allowable character width for any given glyph position? There are maximums for each of the kinds of glyph data in fonts.

Each glyph of the Type 1 font must exist in a maximum 8,000 X 8,000 space with a 1000 unit em. How the glyph relates to the em is up to the designer as long as they stay within the 8,000.

Each glyph of the TrueType font must exist in a maximum 64,000 X 64,000 space (divided evenly between - and +), with an adjustable em. How the em and glyphs relate to the 64k space is up to the designer as long as they stay within the 64k.

The maximum area beyond the side-bearings that a character can extend must fit in this description.

I was a little concerned about this when designing some of the large flourish ornaments for Gabriola, the widest of which is 11,796 units. This is well-within the limits of the TT spec, but since I hadn’t seen any glyphs that wide in other fonts I wasn’t sure what might happen ‘in the wild’. So far, so good, and I have not encountered any problems in apps etc.

I also have zero-width combining versions of these flourishes, and again I have not encountered any problems with the massive negative side-bearings.
