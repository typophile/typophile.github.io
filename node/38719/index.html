2007-11-05 13:33:39 • Goran Soderstrom

Your wish is understood, I think.  ;-)  If the weights/styles are compatible (in terms of both spacing and kerning) then it’d be better to space/kern one, and only in a second step copy values to the other master as described above, to check and correct values where necessary.

If you manage to keep them 1:1 compatible, then only one style actually needs kerning values: generate the kern feature (with FLS' “Update kern feature” or the AFDKO2 script) and copy/paste it to the other fonts. If you use AFDKO2 to generate fonts, you wouldn’t even need to do this but just refer to a common kerning file. So kerning values need not be present in all masters. But it is most unlikely that this would work.

Well, on some fonts, if seeing the kerning on both masters – the kerning could also be used as a tool to find small errors in sidebearings (or developing a system where the kerning and sidebearings work together). That is pretty much why I want this function to be available.

I am not talking about a traditional serif font here, but if you take a look at Flieger Pro, it is easy to see the positive sides of being able to see the kerning right away on both masters.

http://www.autodidakt.se/index.php?page=flieger

I could here decide that I want to use only one kerning (to save time) and then adjust the sidebearings according to that on each master to make the joining strokes “connect” to the next glyph where kerning is needed for that.

I think it makes sense in a way. Especially when the sidebearings in not the same on both masters.
