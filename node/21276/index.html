OpenType superscript & ligature problems

July 14, 2006 • quinnanya

Hello,

I’m new to font-making, and I’ve been pulling my hair out trying to get some OpenType features working for a Cyrillic font I’m putting together. Superscript variants and ligatures are supposed to work in Word 2003, right? (I did try InDesign too, also with no luck.)

My font needs superscript glyphs that look somewhat different than just a raised standard glyph. If it makes a difference, the superscript glyphs have dimensions such that they should show up above the previous character.

In addition, the superscript glyphs take on a different appearance if combined with Uni0483 (combining titlo); they should still appear above the previous character.

I’ve read everything I can on OpenType superscripts and looked at all the fonts I have for clues, but I wasn’t able to find much in terms of examples. All I know is that typing any of the relevant characters in superscript doesn’t yield the superscript variants, and none of the ligatures appear when the correct combinations are typed.

Here’s what I did for the OpenType code:

feature sups { # Superscript script cyri; sub [uni0434 uni043C uni0440 uni0441] by [uni0434.sups uni043C.sups uni0440.sups uni0441.sups]; } sups;

and also

feature liga { # Standard Ligatures script cyri; sub uni0434 uni0483 by uni0434_uni0483.sups; sub uni043C uni0483 by uni043c_uni0483.sups; sub uni0440 uni0483 by uni0440_uni0483.sups; sub uni0441 uni0483 by uni0441_uni0483.sups; } liga;

Any help at all would be wonderful.

Thanks a lot!

~Quinn

first of all the Cyrillic script tag should be cyrl, secondly you should copy your lookups to to the default script so for your superscript feature it should look something like:

feature sups { # Superscript lookup sups1 { sub [uni0434 uni043C uni0440 uni0441] by [uni0434.sups uni043C.sups uni0440.sups uni0441.sups]; } sups1; script cyrl; lookup sups1; } sups;

if i’ve gotten anything wrong, Adam should show up shortly to straighten you out. >^p

Typotheque has a list of applications that support OpenType features: http://www.typotheque.com/fonts/opentype_feature_support/ But read with caution: For some features in XPress7, the mark only means that the according glyphs can be accessed by way the glyph palette (this is: ‘ornaments’, ‘historical forms’, ‘slashed zero’, ‘stylistic alternates’). And that almost all features are marked in the Microsoft Avalon column does NOT mean that any application actually support them. As it seems, Word for Vista does not (there was a thread on this topic).
