I’m most certain that is because i have two paths on the same glyph. I’ve done two or more paths so it could be easier to bezier. First, correct the path direction: Contour > Paths > Set PS Direction

Then, remove the overlaps: Contour > Transform > Merge Contours (Default shortcut seems to be Apple-key + F10 in the Mac version. In the same menu part there are Get Intersection and Delete Intersection which you may not need that often.)

You might not want to Merge Contours if you want to be able to easily tweak characters along the way. I usually only merge right before I’m generating a otf/ttf (and not save it afterwards).

Usually setting PS Direction on paths solves the overlap. If not, you can manually choose ‘Reverse Contours’ from the right click menu on any shape.

2008-01-28 22:07:19 • Quincunx

I’m with Q on this one - I have seen some real benefits to leaving contour merging (a.k.a. overlap removal) to the end as a near-final step in generating the output font. This is especially true if you’re working in MM space, and/or going to be doing italics that will start out with modifying the roman.
