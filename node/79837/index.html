In the OpenType panel, you can “Open features” and “Save features” .fea files using the little flyout menu in the corner of the OpenType panel of FontLab Studio. Also, if you place a .fea file such as MyFont.fea in /Users/myname/Library/Application Support/FontLab/Studio 5/Features/ , then in the lower-part of the OpenType panel you can write just one line:

include(MyFont.fea);

(with the rest of the OpenType panel, and perhaps also the Classes panel empty).

This will link the OpenType panel to the external .fea file. This way, you do not need to edit the feature definitions in the OpenType panel, but instead, you can use the external file. This may have the advantage that you can use one and the same .fea file for multiple fonts.

Note that in this case, it’s best not to include the “kern” feature in this external file. Instead, you should have the kerning defined in the Metrics Window, and have the Preference option “Generate OpenType & TrueType / Kerning / Generate OpenType ‘kern’ feature if it is undefined or outdated” enabled.

Best, Adam

Yes. Just save the TextEdit file with a .fea extension, then read it in via the FL > OenType panel.

I have a separate features file for every font, and it (eventually) includes kerning. Edit it as you need, and save it. On the same flyout panel, there is also something like “Import Features,” or “Open Features.” That will let you read in the external features file you’ve created with the text editor. Then compile.

I do have a “basic” features file I start with, created with a text editor. But different fonts have different details, so I don’t put it in the library, or use a link to an external file – I read it in through that OT panel flyout.
