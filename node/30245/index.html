2006-12-29 22:00:08 • Si_Daniels

In my opinion FontLab is the only choice even though I kind of like the retro feeling in Fontographer. I started with Fontographer at first (some months ago) and would have probably stayed with it if the application only had a couple of updates:

  1. Better screen/preview resolution. I got tired not seeing the glyphs with a good resolution directly in the app. It felt very old style (but had a charm none the less). In FontLab you see the letters much more clearer.

  2. OpenType support (to be able to produce OT-fonts with it).

If these two things got updated while I had the app I would probably continued using it for a while.

BUT… in that case I didn’t have had the opportunity to see all the other great tools that FontLab offer. It’s loaded with helpful tools that makes things so much more smooth when working with the fonts. In other words, I recommend FontLab if that wasn’t clear :) Not that you automatically make better fonts with it, but just because it’s a very helpful tool no mather what kind of fonts you intend to produce. And not to forget, OpenType. Trust me, you will want to add OpenType features to your fonts. OpenType is a fantastic development in the font area – making almost everything possible.

2007-01-03 15:09:59 • metalfoot

Metalfoot,

TypeTool not only “leads into FontLab”, it also includes FontLab’s technology under the hood. The FontLab engine has been tested very extensively in professional font production by most of the top font foundries of the world. TypeTool includes the same engine, it just has a simpler user interface and some limited functionality. (The currently shipping TypeTool 2 is analogous to FontLab 4.6, the upcoming TypeTool 3 will be based on FontLab Studio 5’s engine).

FontLab has been known for very good autohinting, and the same technology is included in TypeTool, while FontCreator apparently does not. TypeTool supports both Type 1 and TrueType fonts, and allows drawing in both Bezier and B-splines mode, while FontCreator only supports TrueType and B-splines, as far as I know. On the other hand, FontCreator has integrated autotracing for which you need ScanFont when working with TypeTool.

The .vfb files saved out of TypeTool can be passed on to a production partner using FontLab Studio 5 who could then generate state-of-the-art OpenType fonts. Altneratively, you could generate a Type 1 font out of TypeTool and then use TransType Pro with Microsoft VOLT or the free AFDKO (Adobe FDK for OpenType) to turn that Type 1 into a fully-functional PostScript-flavored or TrueType-flavored OpenType font. This is something that is not so easily accomplished with FontCreator (since it does not generate fonts with PostScript outlines).

I work for Fontlab Ltd. so my opinions may be biased.

Regards, Adam Twardoch Fontlab Ltd.
