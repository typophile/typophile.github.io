TTF hints won’t work for OTF. They are completely different outline formats.

TTF uses TrueType quadratic Bézier curves, OTF (also known as CFF for Compact Font Format) uses PostScript cubic Bézier curves. They are related, but not the same, so the hints are not compatible or transferable.

That said, you can autohint an OTF font using the autohint tool that comes with the AFDKO (Adobe Font Development Kit for OpenType) or the autohint command in FontLab. (You can also run the Adobe autohinter from within FontLab using a script that comes with the AFDKO.) The results won’t be the same as a hinted TTF font because the format and the way hinting works is different.

2015-03-06 15:51:24 • ferfolio_2

No, for several reasons.

First, I don’t think “most” printers use PostScript natively. There is a thick layer of driver software between the font file and the printer.

Second: font formats may be changed by the OS, by user-end software, or by printer drivers to suit the destination device.

Third: native support for TrueType fonts has actually been added to PostScript, years ago :P

For everyday practical purposes there really is no difference between TrueType and PostScript flavour fonts – especially when both are wrapped inside an OTF. A designer may have a preference (me, I can’t wrap my head around quadratic curves) and some older tools such as MS Volt work on a TTF format only – but even that can be worked around.

2015-03-06 18:51:19 • ferfolio_2

Adobe added native support for TrueType fonts to PostScript Level 2 back in the early 90s, more than 20 years ago. Even ten-year-old sources are wrong to warn against TrueType, for the most part.

It is Adobe’s official position (which I used to spout on behalf of Adobe) that they support TrueType fonts and TrueType flavored OpenType equally with their PostScript-outline counterparts. That position is well founded in reality.

There is a bias in the graphic arts against TrueType, but 99% of the time it has very little to do with reality.

There are a very few devices and workflows, usually very old, where TrueType fonts are not supported. There are actually many more situations where PostScript or .otf fonts don’t work.
