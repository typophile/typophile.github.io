Problem with overlap in new script font

November 1, 2007 • Joebob

Hi all, I’ve been working on a new script font named dearJoe 5 and I am encountering a problem I’ve never seen before: After manually kerning the whole thing I am now testing the font (in Photoshop etc.) and I notice that in some letter combinations, when they overlap, the letters are not displayed correctly. Where the overlap is, the color is inverted: what should be black shows as white. When not overlapping all characters seem fine. Now I could not find the solution myself and I hope it’s something simple like a problem with the contours but really I couldn’t tell. So my question is: what could cause this problem, how do I solve this and is there a way to check all characters at once for this error?

Thanks a lot for helping me out on this.

All the best, Joe

It’s a very simple error. You need to make sure your contours have the RIGHT (depending on your outline format) and the SAME direction.

2007-11-01 11:17:45 • Joebob

If you use PostScript outlines take a look at chapter “3.5 Direction of Paths” in this PDF: http://partners.adobe.com/public/developer/en/font/T1_SPEC.PDF

If you use TrueType contours, it’s the other way around.

Ralf

2007-11-01 11:49:53 • Typography.Guru
If you’re working in FontLab, select all, then CONTOURS > PATHS > SET PS (or TTF) PATH DIRECTION. In Fontographer, select all, then ELEMENT > CORRECT PATH DIRECTION.
