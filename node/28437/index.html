I’m doing a stylistic set (ss01) where there will be a unicase substitution on the lowercase letters.

So it looks like this:

sub b by B; sub c by C; etc.

So it will jsut change most of the characters to Uppercase letters.

However on the letters I’m changing glyphs, I don’t know exactly if it’s OK to do like this:

sub a by a.unicase; Could I have any name as I wish on the unicase glyphs after the period. I can’t go for a.alt, since it’s already taken.

Thanks in advance, aaaaall the way from Sweden.

I am not really en expert on this subject (trying my best to be sometimes) but I have named glyphs similarly in some fonts and I find that it works ok.

2006-09-26 19:24:33 • magnus_gaarde

[[Adam Twardoch]]’s suggestion is that the suffixes follow the 4-letter feature tag, so a.salt for the stylistic alternates feature, a.swsh for the swash feature, etc. Following this scheme, the appropriate suffix for unicase alternates within a ‘unic’ feature would be a.unic. However, if you are, as you said, putting it in a stylistic set, you may opt instead for a.ss01 instead. For a full listing of OpenType Layout tags, follow this link: http://www.microsoft.com/typography/otspec/featurelist.htm

Having said all this, it doesn’t really matter what suffix you use.* However, i’m guessing that the reason Adam suggests as he does is that future versions of FontLab may incorporate this naming scheme as part of automatic OT scripting. In any case, i agree that using these suffices is the most logical method to follow (except in instances of say multiple forms for the same feature, perhaps Adam has suggestions for this sitution as well?)

*within reason and with some technical limitations.
