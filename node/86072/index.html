Aligning Fractions Within Tables

October 4, 2011 • noctisariel

I'm currently designing a product catalog that contains a lot of tables. I'm curious if anyone has any thoughts on how fractions should be vertically aligned within a table. I've always tended to gravitate towards right-justification for numerals, but I'm wondering if it might be more effective/look better to align fractions to the slash.
Depending on the typeface you're using, there may be specifically-designed fractions, which may solve the alignment issue - if you're using an OpenType, it may be worth a hunt through the glyphs. As far as I know, there's nothing in Bringhurst or Hart's about aligning fractions against the solidus, however Hart's does have a useful section on tables that discusses when to align numerals against the column heading or also with the final digit or decimal point (when the data is related or not). Using that rule of aligning decimal points, it could be reasoned that the same would apply with aligning against the solidus in large numbers (no pun intended) of fractions, preserving the vertical flow. Hope that helps.

2011-10-04 19:18:11 • Jackson
In the absence of a visible decimal point, align the ones place. In most situations this will allow you to align the solidus as well, but you can't count on that if you get a big numerator like 15/16. It is all about organizing information so that it can be quickly understood. If someone is looking through a big list for parts that are 10" long or longer, numbers that have been pushed a whole figure width to the left will be unwelcome distractions.

2011-10-14 12:39:13 • dberlow
I worked in a small agency in the 1980s and designed a few product catalogs for tools companies, which contained tables like this with fractions and inch marks. That gave me an appreciation for this kind of "technical" typography. It's not the sort of thing that gets much respect in the design community, but it's the real deal. Based on that experience as a typographer, I've been putting a Fractions feature in all my fonts since OpenType, but have only put proper inch marks in the Modern Suite fonts. Frankly, it doesn't seem worth the effort—primarily because they're so hard to locate in the glyph palette that I doubt people would use them. For serifed fonts, it's easier to get them from the Symbol font. I did briefly entertain the idea of rendering all my dumb quotes (quotedbl) angled, so they would perform both as inch marks, and as an admonition to typographers to set proper curly quotes—but realized that wouldn't be very user-friendly.
