Is "PRO" really "PRO"

February 16, 2008 • gen79

Having some font that have “PRO” in their names, I wonder if that means the same? I’ll explain: in the font Adobe Caslon Pro the characters like g with ^ and c with ^, which are being used in the independent language Esperanto could not be found, whereas in the font Minion Pro, there are present, just like in the font Garamond Premier Pro.

Any comment on this?

PS. I am new here, and I hope I can learn a lot from everybody.

In Adobe’s terminology, a “Pro” font is one that contains diacritics for Central European and Baltic languages (Polish, Czech, Slovak, Hungarian, Lithuanian, Latvian, Estonian, Croatian, Slovenian), and an “Std” font is one without those characters even if it contains Western European small caps, swashes etc.

But of course, this is the minimal prerequisite. A “Pro” font may contain many more characters for different scripts, including Cyrillic and Greek, and more Latin characters, as well as alternate glyphs, small caps etc.

2008-02-16 13:34:05 • Florian Hardwig

Perhaps slightly off topic, our use of “Pro” in the designations of our CJK fonts, which are based on static glyph sets called character colections, is a bit different.

If a character collection Supplement contains primarily glyphs that correspond to character set standards, the “Std” designation is used for such fonts. Adobe-Japan1-4 (/Registry “Adobe,” /Ordering “Japan1,” and /Supplement 4) represents the glyph repertoire level that allows Japanese fonts to be declared as “Pro” fonts. Adobe-Japan1-4 defines 15,444 glyphs (CIDs 0 through 15443). The current Supplement is 6 (Adobe-Japan1-6), which defines 23,058 glyphs (CIDs 0 through 23057).

Glyphs above and beyond character set standards are included in “Pro” fonts, and are intended to be used for professional and commercial publishing. This means additional punctuation, common ideograph variants, a rich set of Latin glyphs, to include a full set of italic forms (made accessible through the ‘ital’ GSUB feature), and so on. We have yet to define Supplements for our Chinese and Korean character collections that can be considered “Pro” level, and when time permits, that is something that I plan to do. Curiously, Adobe-GB1-5 defines 30,284 glyphs (CIDs 0 through 30283), but fonts based on this character collection are still designated as “Std” fonts, because the collection of glyphs is primarily based on character set standards, specifically GB 18030-2005.

Interestingly, some font developers apparently have other criteria for designating their CJK fonts as “Pro,” such as Apple for their Chinese fonts that are based on GB 18030-2005. Their criteria is based solely on the number of glyphs. I find this to be a poor criteria, because the threshold is highly subjective. The criteria that I briefly described in the previous paragraph represents a much more practical and useful criteria for using the “Pro” designation for CJK fonts.

Dr. Ken Lunde Senior Computer Scientist, CJKV Type Development Adobe Systems Incorporated lunde@adobe.com

In Adobe’s terms, Helvetica Std is a “Std” font even though it includes Western+Cyrillic (but no CE). Though I must admit such cases are rare. In the vast majority of cases, “Pro” does indicates “the character set beyond Windows Western, with at least Central European Latin diacritics included”. Typotheque sets the bar higher, as their Std fonts do include CE glyphs as well (but their Pro includes also Cyrillic and Greek).

URW++ has a more elaborate scheme that involves “L1”, “L1.5” and “L2” character sets. I somehow agree that there is a need for a term to describe the Western-only fonts (“Std” is OK here) but then there is also a need for two different terms, one to denote the Latin-only multilingual fonts (i.e. extended character coverage within one writing system), and another to describe fonts that offer more than that (Cyrillic, Greek and other alphabets, potentially).

For my internal purposes, I’ve adapted the optional “Basic” vs. “Plus”/“Extended” suffix. An “Std Basic” font would really be just the Western codepage, an “Std Plus” or “Std Extended” font would have small caps, oldstyle numerals, extra ligatures or perhaps swashes, but still only for the Western characters, a “Pro Basic” font would offer Western and Central European diacritics, and perhaps some alternate glyphs for those, but no Cyrillic or Greek, while “Pro Plus” or “Pro Extended” would mean an open-ended font with at least two writing systems supported (one other than Latin).

But then again, once you start making those finer distinctions, you can never stop, and also, there is no way to “standardize” those terms across vendors (not that I would think such standardization is necessary).

I first used the designation “Pro” to distinguish an OpenType upgrade to a Latin 1, Type 1 font, by the addition of contextual alternates. (Handsome Pro.) That’s all, no extra language support.

However, in future I plan to release basic fonts that will have Extended Latin support, and Pro fonts that will also have typographic OpenType features (small caps, alternate figures, etc.)

IMO, “everyone” now needs Extended Latin support (and perhaps Greek and Cyrillic–blame the E.U.), but only professionals need true small caps and superior figures.
