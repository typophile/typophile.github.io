I completed a connecting script a few months ago. My first attempts at making it more automatic worked but since there are some inherent weaknesses in OpenType feature code it would have led to frustration on the part of the end user so I abandoned that.

To simplify it, I set it up to make using the beginning alternate glyphs and the ending alternate glyphs easier for the end user. I used the code in a Stylistic Set, as follows:

For ending strokes: sub zcaron' space by zcaron.fina; For beginning strokes: sub space a' by a.init;

You will have to have a similar line for each alternate glyph you wish to substitute. You don’t need the /ignore line you currently have.

Your client must have software that supports Stylistic Sets too.

The way to go (which works with FontForge, there should be a way with Fontlab) is to give yourself two lookups, one to substitute the first letter, which is just

lookup subfirst {
  ignore sub  @LETTER s';
  sub s' by s.init;
} subfirst ;

The second lookup, for the last letter can simply take the form

lookup sublast {
  ignore sub @LETTER' @LETTER  ;
  sub s' by s.fina;
  [...]
  sub otilde' by otilde.fina;
  sub odieresis' by odieresis.fina;
} sublast;

The first line says not to substitute a letter when followed by a letter. Then you list the substitutions for the last letters. Then you simply use the feature calt as follows

feature calt {
    lookup subfirst ;
    lookup sublast  ;
} calt;

where the first lookup takes care of the first letter, and the second lookup takes care of the last letter (they are applied one after the other line by line). Of course you need to include in LETTER s.init if ss may occur at the beginning of a word, and maybe other characters. Be careful, there is just one apostrophe per line which corresponds to the glyph (or class of glyphs) that is to be substituted.

It would not work without some corrections, for instance for oacute because your rule is

sub oacute s' by oacute_s.fina;

which asks to substitute the final s by oacute_s.fina when preceded by a oacute. You probably meant

sub oacute_s' by oacute_s.fina;

The rule for o_s does not have that problem and does not work either. Here is what I would try. I would simply put the rules for o_s in a third lookup, that would take o_s and replace it with o_s.fina, otilde_s by otilde_s.fina etc. That would give three passes for each line of input text, a first lookup that fixes the initials, a second lookup that fixes the final o, b_s w_s etc and a third lookup that fixes the o_s etc.

Be careful. The lookups are the actual code that is executed. If you want the three lookups to be executed in some order, you need to put them in that order in your feature file. The order they are listed in calt is irrelevant.
