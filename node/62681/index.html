2009-10-08 00:40:52 • paul d hunt

Paul: i believe there is a bug that limits the number of characters for font names within Mac Word. Could this perhaps be the problem?

I reckon that to be absolutely sure that the PostScript name will not generate errors, it should be limited to 28 characters. For at least older Mac versions of MS Word the Full font name (nameID 4) should not contain more than 32 characters. One way to circumvent this, is to limit the Compatible Full name (nameID 18) for the Mac to this number.

2009-10-08 19:31:08 • blokland

The name length thing may also be a problem, and should be checked out, but I have also seen this behavior in Word 2008 with names that are well within those limits.

One thing I’ve noticed is that it is not necessarily predictable or even repeatable: for example, installing a family with members Light, Light Italic, Regular, Italic, Medium, Medium Italic, Bold, and Bold Italic, in one instance it might be the Light Italic that doesn’t show in the menu. A separate install could omit the Medium Italic. It is this apparent randomness that leads me to believe there’s a Word 2008 issue at play, and not necessarily a font naming or styling issue.
