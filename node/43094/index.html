GREP and ID CS3 query

March 12, 2008 • tomottoe

Hi I’m trying to use GREP for the first time for a simple find/replace. I have an endnotes section for a book where the space between the note number and the note entry needs to be changed to a tab. For example, a note reads “13.spaceIbid., p. 36.” and it needs to read “13.tabIbid., p. 36.” So for the double-digit note numbers, I’ve put this expression in the find field:

~b?(\d\d).

where the last character in the expression is a space; and this expression in the Change To field:

~b$1.\t

By including the paragraph return symbol in the front of the expressions, I was hoping to avoid finding any of the two digit occurrences within the text of the endnote. But no luck–those end up getting changed during the search/replace. Is there a more definitive symbol for a paragraph return (I tried using \r as well, with no luck), or a better expression I can use? This will work if I punch through manually and “find next” on the instances within the note text, but there are about 1500 endnotes in this book and I’d rather not sit here for the next hour!

thanks tom

Have you tried using the symbol for “Beginning of Paragraph” (^)? Something like this: Find what: (^\d+.) The last character is a space Change to: $1\t
