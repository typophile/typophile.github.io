Photoshop CSx not recognizing calt feature

March 22, 2007 • Mark Simonson

I’m finishing up a script font that uses calt (Contextual Alternates) to pick appropriate glyphs depending on where they fall in a word–mainly selecting different glyphs for the beginning and ending of lowercase runs, but not exclusively. All is working exactly as it should in InDesign (CS, CS2), Illustrator (CS, CS2), and Quark 7.

However, Photoshop (CS, CS2, and CS3-beta) is not cooperating. It does not seem to recognize that the feature is present (it’s grayed out in the OT feature menu). Not only this, it seems to think my font has a titl feature instead–which it does not.

I’m using FLS5 (Mac) and generating OT CFF fonts. What’s going on with this? Anybody run into this problem before?

I have seen this too, but it was Swashes that were missing. Strangely, quitting Photoshop, clearing font caches, and reactivating the font seemed to fix the problem. But don’t quote me on this as it was more than a few weeks ago.
