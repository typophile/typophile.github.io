Well, Miguel is bound to have a lot more experience than me with Macs and Unicode, but I do believe that dividing the character set into two fonts would do the trick, albeit it at a considerable loss of convenience (especially if you're using InDesign). That's because, in an "alternate" font, the characters that would have high Unicode numbers can be put in the slots normally reserved for common characters. So, for instance, in Scala Sans Regular Expert, the dimension sign (×) glyph can be found in the slot normally reserved for the tilde, and the fl ligature resides in the slot normally reserved for the question mark. In other words, when you switch to Scala Sans Regular Expert and type a tilde, you get ×, and when you type a question mark, you get an fl ligature. As far as the operating system is concerned, there are no characters above ASCII 255; the OS doesn't need to support Unicode for you to be able to access "special" glyphs if they're put in the slots for "normal" glyphs.

In an Unicode world that’s called hacking :) as you’d be assigning incorrect Unicode values to characters. Tom, the bottom line is that a proper OpenType font with all bits and pieces in the proper places is better than the old "hacks" -- such as PS1 fonts divided into basic and expert etc -- for a number of reasons, especially standards compliance and ease of use (with software that can handle automatic glyph substitution etc such as InDesign). If you plan to release your font(s), you should upgrade your OS rather than opt for what Miguel has rightly called a hack. From your comments, it seems that you need to study Unicode a little more before expending more time and effort on your font production. A good place to start is the Unicode web site. A useful tool is their Unibook application (free), which is actually not very easy to find on the site.
