“You can run it at the command line.”

But probably not on a native Windows.

My batch scripts (Windows):

@echo off call FDKvars copy Input*.* Output cd %FDK% for %%f in (%~dp0Output*.otf; %~dp0Output*.pfb; %~dp0Output*.pfa) do %autohintCom% -q -a “%%f”

While FDKvars.bat is:

@echo off set “FDK=C:\FDK_25_21898\Tools\win” set “autohintCom=autohint.cmd”

The folder, that contains the two batch files, must also contain the folders “Input” and “Output”.

I have created the FDKvars.bat, because now I only have to make slight corrections in that file, when Adobe updates the FDK. And I renamed the folder FDK to FDK_25_21898, because I am using three versions of the FDK at the moment.
