I realize the PC is behind the Mac in this type of functionality. I have also considered purchasing a program like FontCreator in order to re-map an existing font into different font sets so all the characters are available for use (just using different font names). Thanks in advance.
Among the applications that come to my mind, Adobe apps (InDesign, Illustrator) come to mind first. Even the CS2 version would do.

You don't necessarily need OT feature functions in an application to access alternates. It's more tedious but as long as the alternates have been given Unicode values, you can make a text file which the customer can copy and paste into Corel Draw or Word. They can also be accessed with the character map tool. (accessories/system tools/character map) Windows has supported OpenType(OTF) with no additional software required for a decade . . . a couple of years before OSX was released. It's the apps that are the issue.
