2007-06-26 16:37:45 • canderson

Our main goal was to achieve the chaotic aspect of a realistic drawing tool and the emotion of a human handwriting!!! So we skipped the vector approach (loss of detail, greyscales, no color and inappropriate existing tools). We also wanted to have an unlimited number of versions for each letter or ligatures. And since Opentype features programming was also a lot of pain, we went our own way, much easier one ;-)

Now, we can also add specific “behaviors” to fonts and letters (àla kerning pairs, special ligatures replacement, and new ideas too…), that do not exist on current font formats.

We have developed our own tools not only to work on print projects, but we are also porting the interactive editors to the web. You can try our first demos, so our system can easily be implemented in any existing website (no plugin). For more interactions between type and the user, there will be a much more flexible and playful version too…

2007-09-03 06:32:10 • Dan Gayle

This is cool, but, really, nothing new. Rendering text as images via a web server has been around for quite a while.

While this is fun, for true web use, it’s not ideal, as image based text just isn’t as accessible…which is a primary benefit of using the web.

That said, you could improve upon this a bit by making sure that you also render appropriate alt attributes for the images that are created.

Also, I’m thinking GIF files would likely be a better option for type.

BTW, the hand-rendered typefaces are lovely and I’m sure there’s a market for this.
