Hi there, I am generating a 'non style-linked' font family and for that reason I am not using the checkboxs 'Font is Italic', etc. Following advice given in this forum, I set a value in the 'Italic Angle' field (Font Info > Metric and Dimensions) because in that way styles are better listed in applications like InDesign. The problem is that when I test the generated OT font in Font Validator, it seems to find an inconsistency in Italic styles: 'The italicAngle is nonzero, but the head.macStyle italic bit is no set'. The error is marked red in the report. Is it really something to fix or I should ignore it? Thanks in advance.

In the AFDKO package there's a file named MSFontValidatorIssues.htm that lists some of the issues you might run into when testing OpenType-CFF fonts in MS Font Validator. And one of the things listed indeed sounds like the problem you're having. Here's the quote from that file: A font which is not an italic style of another face in the same family can nevertheless have large italic angle. Examples are script fonts and italic designs which are the only font in the family. In this case the Adobe Type Department prefers to enter the real italic angle in the post table italicAngle field. This way, the insertion bar is correctly slanted in most programs. However, this will cause the following messages: E1316 The macStyle italic bit is clear, but the post table italic angle is nonzero E2308 The italicAngle is nonzero, but the head.macStyle italic bit is not set
