The three names marked in red are not the Unicode names for glyphs shown, but are commonly used (e.g. slash for solidus, period for full stop).

Strictly speaking, Unicode doesn’t use names, or if you want to insist it does, then the name is “uni” or “U” plus the Unicode number.

PostScript Type 1 however, was dependent on names, and many operating systems had a hard-wired encoding vector. “solidus” “full stop” were not in that encoding vector; characters so named would not display.

If you want to say the names you’ve given were preferred in the UK before, say, 1990, that’s fine. But he’s asking a technical question.

2010-09-29 20:29:27 • charles ellertson

This indicates a mismatch of your glyph naming with the standard.nam file that maps glyphnames to Unicodes. The standard.nam file only “permits” a handful of double-mappings. Because of Adobe recommendations, FontLab Studio recommends to duplicate glyphs such as “hyphen” into two glyphs: “hyphen” and “uni00AD”.

The mismatch would also be shown if the glyphnames and Unicodes would REALLY mismatch, i.e. if the glyph “A” had the Unicode of the glyph “C”, for example.

2010-10-02 01:49:10 • dezcom

Rachel,

you may have installed an updated standard.nam (e.g. through installing Fontographer 5 demo or full version on the same machine as FontLab Studio), or you’ve picked a different standard glyphname-to-Unicode mapping file in Preferences / General / Unicode and OpenType.

BTW, if you feel your glyphnames are right but your Unicodes aren’t, you can fix this using Glyph / Glyph Names / Generate Unicode. If your Unicodes are correct, you can generate recommended glyphnames using Glyph / Glyph Names / Generate Names.

In fact, you can fix most of these problems by first doing Generate Unicode, and then Generate Names. The first step will accept obsolete glyphnames to generate correct Unicodes, the second stem will use the corrected Unicodes and will generate new, recommended glyphnames in place of the obsolete ones.

BTW, I should point out that installing Fontographer 5 demo will give you a new set of glyphname-to-Unicode mapping files, which will be also available in FontLab Studio. Along with new and updated encodings that also come with Fontographer 5, this will upgrade your FontLab Studio to year 2010 of “proper” glyph naming and encoding.
