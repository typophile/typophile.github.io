Problems with adding ligatures to Warnock Pro

July 28, 2006 • charles ellertson

We have to typeset a book where the designer specified the font Warnock Pro (light) for the text. The book has a bibliography, and of course, in a bibliography where there are multiple books written by a single author, the second, third, etc. entries start with a dash three ems long (a three-em-dash) instead of repeating the author’s name.

There is no threeemdash in Warnock Pro (light). And the emdash itself has sidebearings, so setting three dashes in a row also doesn’t work. Nor were multiple dashes kerned in the foundry font.

OK, In FontLab 5, I can make a threeemdash and a twoemdash for the font, so named, with no Unicode index (I take U2015 to be something else).

Then go into the features file and add:

sub emdash emdash emdash by threeemdash; sub emdash emdash by twoemdash;

right after the f_j ligature.

Did this, recompiled, and went to the Preview panel:

Everything appears fine, all the other ligatures still work, and three emdashes in a row give the “threeemdash” character.

BUT: the ligaturing doesn’t work in InDesign, either through using the three em dashes via the Glyph pallet, or by typing three Alt_shift_hyphens (where there is no intervening code). I can add the glyph directly via the glyph pallet of course, but that is a lot of work – first erase the existing dashes, then insert the “threemdash,” for every such entry in the bibliography.

There should be a way to take the existing three emdashes in the supplied MS Word file & wind up with a threeemdash in the InDesign file, right?

I’m pretty sure this is due to some complexities in Warnock Pro, since we’ve gotten this ligaturing feature (multiple em dashes to a single long dash) to work in fonts we’ve created from scratch.

Anybody know what’s going on?

Thanks,

Charles

Charles,

I don’t have a whole lot of time to open up FontLab right now and try and come up with a more dependable answer, so here are just a couple of quick thoughts:

(a) In my experience, problems such as these are often due to silly little mistakes that are difficult to catch exactly because they’re so minor. For instance, have you checked that the new glyph you created has exatcly the same name as the one you’re using in the ligature substitution code?

(b) If you decide that you’ve already spent too much time trying to solve the problem in FontLab, why not just set the whole book in InDesign, then run find/change and have the program replace all occurrences of three consecutive dashes with your new m-dash? I imagine you’ve had this idea before – what’s stopping you from using it?

2006-07-29 18:43:38 • Thomas Phinney
As Eric, Miguel and Thomas have pointed out, this is simple to address in InDesign. I have to use 3-em dashes all the time for setting bibliographies, so I’ve just built a character style that I load into each new book project (with negative tracking built into it). From there I quickly run a find/replace for every instance of 3 em-dashes and the sidebearings problem is solved, without having to muck with the font. (In particular I’m not big on de- and re-compiling a font with as many OT features built into it as Warnock; I’m just not experienced enough with FL to know for sure that I’m not simply introducing errors in the process).

Thanks to all. We decided to put in the new three-em-dash via search and replace in InDesign tagged text files, then bring them back into InDesign. When we later make the XML files, we’ll change he threeemdash back to three em dashes. There were various reasons for not using the other approaches.
