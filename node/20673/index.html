Multiple hand-lettered forms with Opentype?

June 14, 2006 • aaronseymour

One of the limitations with hand-lettering style digital type is that every letter form is identical and the set type looks to regular. Given that Opentype allows for thousands of characters is there a way that multiple versions of each letterform could be used in a font (imagine 5 verions of lowercase ’m') with the computer then randomly selecting one of them (each time the ’m' key is pressed)?

I imagine this would involve assigning one ’m' shape to the correct unicode character and the other four to ‘empty’ codes, and then having the computer somehow alternate between them…?

Thanks in advance, Aaron

Yes, it’s possible. And technically you off to a good start. And it might be a good idea to persue if the design is made in such a way that it benefits from the randomness. Before Opentype a type 1 font called Kosmik was made by Letterror that does. See? http://www.letterror.com/foundry/kosmik/index.html

But about the hand lettering. Good hand made letters (and therefore words - no?) are made with their forms in relationship with each other - not randomly. So making a type face with context driven substitution might make more sense.

There are lots of threads on Typophile that talk about these issues. Take advantage of them by searching Typophile. Cheers!
