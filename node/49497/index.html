FontExplorerX Nightmare: Help Me Wake Up!

September 16, 2008 • Dan B.

Dear Typophiles,

Desperate is not a strong enough word to describe my present situation. With all the projects I should be working on, the last thing I wanted was to spend my morning trying to locate my fonts!

So here is what happened. I while ago I installed FontExplorerX for Windows (I run XP). I figured it would help me finally organize my fonts. So I ran the program and it seemed to “index” all my fonts from my default font folder (in the Control Panel). It created a folder for each (!) font file and placed them in a larger folder. Now, when you look in the Control Panel font folder, all the fonts appear to be shortcuts to the new location of the font file.

Anyway, the program would not install properly. It would jam during the installation. I tried to repeat the process, but the same happened. What I’m trying to do right now is simply move all my font files back to the default folder, because I’ve been noticing that some fonts appear to be missing. Today I opened up a file I was working on yesterday and a missing font window appeared! Everything was fine yesterday!

I’m wondering what to do next. The only way I can think of is copy all the font files from the FontExplorerX folder (this means going through hundreds of folders) to a separate folder, deleting the shortcuts in my default font folder, and then pasting the copied files back to the default.

Do you know of any other solutions? Or is my solution flawed? Has anyone encountered a similar problem? I’d really appreciate your help. Thank you.

Dan

2008-09-18 00:39:06 • Thomas Phinney

If you do acquire a copy of ATM Deluxe 4.1 and proceed to use it there is one caution: don’t name any of your font sets with more than 31 characters. Should you do so, everything in your font list from that folder and below will disappear. The only way to fix the problem is to delete the font database and start over. This was a lesson I learned the hard way.

That being said, the program is indeed rock-solid and I use it to this day on my Windows machines.
