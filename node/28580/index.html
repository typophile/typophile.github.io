Paul, your ‘liga’ feature is not entirely correct. Here’s the complete version: feature liga {

script DFLT;
	language dflt;
		lookup liga1 {
			sub f f i by f_f_i;
			sub f i by fi;
		} liga1;

		lookup liga2 {
			sub f f l by f_f_l;
			sub f f by f_f;
			sub f l by fl;
		} liga2;

script latn;
	language dflt;
		lookup liga1;
		lookup liga2;

	language DEU exclude_dflt;

	language AZE exclude_dflt;
		lookup liga2;

	language CRT exclude_dflt;
		lookup liga2;

	language TRK exclude_dflt;
		lookup liga2;

} liga;

BTW, we’re currently using a different approach on Adobe’s fonts. We handle the laguage-specific substitutions in the ‘locl’ feature, and make a ‘liga’ feature with no lookups. For more info on this, read pages 97–102 of the presentation that Christopher Slye gave at ATypI in Lisbon.

Oops, I forgot to mention that we’re not treating f-ligatures differently for German any more. If you still want to do that, yes, you’ll need lookups in ‘liga’, or perhaps a different solution in ‘locl’.

Here’s how we’re doing presently: feature locl {

“script DFLT;” omitted in this feature

since there is nothing to be defined for script DFLT

script latn;
	language TRK;
		lookup IDOT {
			sub i by i.dot;            # i.dot is a duplicate of i
			sub i.sc by i.dotsc;       # i.dotsc is an i.sc with a dot on top
			sub I.sc by Idotaccent.sc; # Idotaccent.sc is a duplicate of i.dotsc
		} IDOT;

	language AZE;
		lookup IDOT;

	language CRT;
		lookup IDOT;

} locl;

feature liga {

sub f f i by f_f_i;
sub f f l by f_f_l;
sub f f by f_f;
sub f i by f_i;
sub f l by f_l;

}

Since “script DFLT;” is not used in this solution, you can test the behavior in FontLab.
