This is pretty simple. The quoteright (U+2019) and the quotedblright (U+201D) are outside of the Mac OS Roman range. In order to display those characters correctly, a text file containing them has to be saved in Unicode (UTF-8). If you then reopen such file using the Mac OS Roman encoding, the quotes get replaced by the multiple characters you’re seeing (in UTF-8, 2019 = E2 + 80 + 99 and 201D = E2 + 80 + 9D)
