Try this:

Choose Window>Workspace>Export Workspace…

FontLab will suggest the name “UserWorkspace.rc”. If you’re already using this name, you might want to use a different name.

Quit FontLab.

In the Finder, navigate to [your home folder]/Library/Application Support/FontLab/Studio 5/Workspaces/ and find the file you just created. Open the file in a plain text editor such as TextEdit.

Search for “33296_Left” and “33296_Top”. The numbers to the right of these entries are the left and top coordinates of the Preview panel (relative to the top left corner of the menu bar). Change both of these coordinates to 100. Save the file.

Open FontLab. Select the file you saved from the list of workspaces in Window>Workspace>. If it isn’t already open, open the Preview panel. It should now be positioned near the top of your screen.

It’s not really program code. FontLab’s workspace files are just lists of things in FontLab’s interface, sometimes with numbers for coordinates or dimensions. It’s “human readable” xml, so a lot of it is self-explanatory. The panels, however, were not so simple. They are listed with meaningless number codes in a section called “@DOCKING_PANELS”. I was able to figure out what a few of them were by saving the current workspace, moving a panel, saving another workspace, and then comparing the two in a text editor.

There’s normally no reason to edit these files directly–most of it can be changed from within FontLab–but it’s nice to know you can.

I guess it’d be these: 33296_Width 809 33296_Height 429

There’s also 33296_State 0 which probably controls if the window is visible (1) or hidden (0).

You can also try to reset the workspace by doing this:

  1. Launch FontLab and press the ‘ctrl’ key.
  2. Keep holding until the “Reset Workspace” window appears.
  3. Select “Reset UI to default state” and press the “Reset UI” button.

Hope it helps.
