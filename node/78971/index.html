Your question is not clear.

The size of a pixel is determined by the computer screen on which something is viewed. If you use Photoshop or another paint program, if a letter is 60 pixels high on your computer, it will be 60 pixels high on any machine on which the image is viewed, whether because they downloaded the image, or because it appears in a web page.

When including text on a web page, one can specify that the text is to be larger or smaller than normal by specifying ‘font size="-1"’ or something like that. These sizes depend on the user’s settings, and don’t allow fine control.

However, when using CSS, you can also specify point sizes for text.

And, when using Photoshop, normally you don’t specify the size of text in pixels, it is also specified in points.

Here, you can run into trouble.

Back in the days of Microsoft Windows 3.1, and Macintosh computers that used 68000-family processors:

On a Windows machine, 72 pixels were normally treated as “one inch” for translating the point size of type into the screen size of a typeface. That could actually have been any size, because a PC-compatible computer had no way of knowing how big your screen was. You could choose to use “large fonts”, and to change that relationship so that the computer would treat 96 pixels as one inch.

On a Macintosh, the size of the screen was known to the computer, and it determined how many pixels the screen had. The standard resolution of the Macintosh platform was 60 pixels per inch.

Now, the standard resolutions is much greater on the Macintosh than it had formerly been. Changes on the Windows platform have not been so large, but usually the choice is between 96 pixels per inch and 144 pixels per inch.

So, if your computer is a Macintosh, and your web page is being viewed by someone using Windows…

if your CSS script says a certain type of text is “14pt”, the height of that in pixels will be determined by the number of pixels per inch (72 points) on the user’s Windows computer…

if you used 14 point text when preparing an image, the height of that text in pixels will be determined by the number of pixels per inch on your Macintosh.

This could be what the person was mentioning.

Never use ‘font size="-1"’ in the HTML tags, define this in the CSS:

p { font-family: Tahoma, sans-serif; font-size: 14px; }

This will style all the p tags in your HTML with 14px Tahoma, and where Tahoma is not present (for example on the iPad or on Android systems) it will use the standard sans-serif in those systems instead (and most are not as narrow as Tahoma, and might not share it’s x-height — 14px might look bigger or smaller in a fallback font). The font size can be specified in px’s (pixels), em’s or percents, the former being the most static as it only relates to the ppi of the screen. Depending on rendering engine in the end users OS and web browser and the design and hinting of the font, letters may appear with slight differences in width and height.

Mocking up in Photoshop is well and good, but a serious web designer should have some understanding of how the web works.
