2008-01-17 21:43:48 • Chris Rugen

I actually learned that saving as an interchange file is good to do every once in a while especially if you are using old or previous jobs as pick-ups. In the environment I work in, we are using jobs for years at a time, and almost all jobs are picked up, and saved as. This can cause a ballooning effect in file size after a while. If you save as an interchange file, it strips it down to code essentially and is then build or put back together so to speak when you open it.

I’d say you are pretty safe saving as interchange and opening in CS2. I do it all the time and notice no difference.
