Nothing is going wrong. The aalt feature can only include one-to-one substitutions, not ligature substitutions. makeotfexe is simply letting you know that this particular substitution is being stripped from the aalt feature.

I can’t comment on the naming issue since you haven’t provided enough information. Which name field is ‘dotsFDK’ taken from? What’s in the other fields?

He is merely including the dlig feature as part of aalt. That’s pretty normal to include all your features in the aalt so that you can get glyph submenus to map properly in the glyph palette of AI and other apps. I’ve never run into an issue with that.

The warning sounds to me like its finding the dlig feature incorrect (which it is) and is catching the error in its first reference which would be the aalt feature which is typically at the top of your features list.

2009-11-19 21:12:08 • Stephen Rapp
André is right in this case.. The aalt feature can use only single and multiple substitution rules. That particular message means that the logic which is building the aalt rules can’t find any appropriate rules from the dlig feature. Unfortunately, the logic can’t tell if the feature was never defined, or was defined, but didn’t have any rules that could be used for the aalt feature.
