If you are talking about an OpenType font:

There is a bug in some applications programs, where if there are no substitutions (a G-sub table), the G-pos table (positioning, which includes kerning) isn’t applied.

In FontLab, try writing a ligature feature, make sure you add the kerning to the features, recompile the features, and see if the font doesn’t work properly.
