FontNuke & Caches

March 13, 2009 • Michael Jarboe

Is the consensus around here, when going through test edits with a newly exported font that the only way to ensure against corruption, and problems is to use some type of cache clearing program (such as FontNuke). I've run into some strange glitches such as missing glyphs showing up in my Illustrator glyph palette today… but I've also resisted clearing my cache every time and have just been deleting from FontExplorer X and re-importing… Is there a faster way than having to clear caches with either FontNuke, FE X, and restarting? Thanks, Mike
I find that as long as I get the naming right, and the Adobe apps don’t crash while I’m working, I don’t need to clear the caches at all. If you do find yourself needing to clear system caches and reboot, don’t load the test fonts in Fex, just output them directly to ~/Library/Application Support/Adobe/Fonts and restart the Adobe app to get the new fonts. If you have problems just delete the fonts, clear the application caches, and you won’t have to reboot. Obviously this strategy works better on a desktop with a fast drive than on a laptop where program loading is slower.

I've had no trouble with just outputting the new font version with another name. It doesn't matter what you call it while you're doing development testing, so I just append a revision number to the name, written out in English, and I haven't had any font conflicts that way. It also allows me to keep the earlier versions installed so I can easily make comparisons.
