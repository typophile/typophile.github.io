CAPTCHA and the word superiority effect

October 16, 2009 • Chris Dean

[[http://www.captcha.net/|CAPTCHA]] and the [[http://en.wikipedia.org/wiki/Word_superiority_effect|word superiority effect]] (WSE). I have noticed facebook and other sites are starting to use real words more often (as opposed to [[http://en.wikipedia.org/wiki/Pseudoword|pseudowords]] &c.) thus making them significantly easier to recognize. Does anyone know if this is on purpose?
Using words do make it easier for people to solve the CAPTCHA, but it also makes it much easier for a computer program to solve. Using words is a terrible idea if a site is trying to stop frequent attacks from computer programs, but probably just fine if the site isn’t expecting someone to write custom programs to break your CAPTCHA. I worked on a project where we investigated how to build a CAPTCHA that would be recognizable by people, but not by computer programs. http://research.microsoft.com/en-us/um/people/kumarc/pubs/chellapilla_chi05.pdf Interestingly, computer programs recognize single characters at a higher rate than people. To build an effective CAPTCHA you need to make it difficult for the computer program to segment the location of each of the characters. http://research.microsoft.com/en-us/um/people/kumarc/pubs/chellapilla_ceas05.pdf
