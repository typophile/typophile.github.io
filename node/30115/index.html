Low-cost Word processing program (PC) that supports OT ligaturing?

December 17, 2006 • charles ellertson

I know this isn’t really a “build” issue, but it is a software issue & a lot of expertise resides on this forum . . .

I have sort of inadvertently gotten involved with a project for the Cherokee Museum in North Carolina.

They wanted a smaller version of the Syllabary, which my wife made up. When we were up at Cherokee, one of the people who prepares documents written in Cherokee mentioned how much a problem it was for them to use the proprietary LaserCherokee software—not that it didn’t work, but was too expensive to put in classrooms so kids could learn the written form of the language.

It occurred to me that since we’re dealing with a syllabary, and one already in Unicode to boot, it wouldn’t be that hard to write a ligature feature (sub h n a by uni13BF etc.) for a font using a freeware Latin alphabet & the Cherokee font they already own. You have to cheat a little with the vowels. I wrote up a little test font & it seemed to work using InDesign.

The problem is finding a word-processing system that supports OpenType ligaturing features, and is cheap. They seem to use the Windows operating system.

Anyone have any suggestions?

Charles,

if complex-script processing is enabled in Windows XP (in Control Panel / Regional Settings), OpenType Layout features are processed using Uniscribe in applications such as Notepad or BabelPad (http://www.babelstone.co.uk/Software/BabelPad.html ). Both of these are plain-text editors only, though.

Funnily enough, there does not seem to be any widely-available word processor that would support this, though.
