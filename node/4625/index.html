Well, other characters as well…they don't show up on the screen in OS X applications when you use (for example) Postscript Type 1 fonts, yet when you change the font to a system font included with OS X, no problem. When they don't show up onscreen, there's a space there, but they do print fine. Weird. Anybody know anything?
Many Type 1 fonts use the ISO-Adobe character set, which does not include 14 math characters that are part of MacRoman. When combines with Adobe Standard Encoding, the system font driver substitutes the right glyphs from the "Symbol" font for both on screen and print usage. (This can get more complicated with additional special flags and re-encoding vectors in the FOND resource, but the simplest case is overwhelmingly the most common one.) Sounds like OS X broke this for on screen use, even though it still works in print. What exact version of OS X are you running? Regards, T
