I am a bit of a newbie here myself, so take this with a pinch of salt. Topics regarding software tools are usually found in "Build". Most topics concentrate on technical aspects of software, rather than on how to design or use type as such. There are some topics about type design in "Design", covering things like how to make a good 's', or how to create italics. You will also find useful advice in the various subsections in "Critique". This is the place to post samples of your designs for others to comment on. Reading comments from experts about other people's designs might help you gain insight into some of the important aspects of designing type. Hope this helps. Have fun!

Yes. I'm to blame for some of the confusion, I suppose. If you need critiquing it is obvious, but otherwise it is confusing. Anything about your personal work usually belongs in CRITIQUE or DESIGN. DESIGN if you are just showing. CRITIQUE if you want opinions. BUILD if it is tech of some kind. RELEASE if it is for suggestions on how to release your designs (type or otherwise). If you want type suggestions, DESIGN. If you are are trying to find a face, TYPE ID. Otherwise I'm the one of the persons whom move your threads.
