Universal(ish) unicode fonts for web use

November 12, 2008 • thefsb

I’m considering converting my web site and its database from latin1 to utf8. Just thinking about the process for now.

Say I want to allow a user to enter into a web form the name of a musician in the native language, it gets stored in the database and can be displayed later for other users. Preferably the form would not make the user explicitly specify a language or character set – it would just accept a utf8 string and pop it in the DB.

To keep it simple, I’d want to use in the HTML a font with the broadest selection of unicode characters. But it would have to be a font commonly available in web browsers.

I checked some fonts in FontBook on my Mac and it seems that Arial and Tahoma are the best candidates. Verdana not so great.

Is that about the status of things?

In addition to Arial I would specify “Lucida Grande” for Mac users.
