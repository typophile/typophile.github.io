2012-01-08 14:37:19 • Pat2

Manual TrueType hinting is done one character at a time. Note that there’s a difference between TrueType hinting and Postscript Hinting. Fontlab’s PS autohinting isn’t half bad, the other one have a tendency to bloat the file size without much better rendering results.

Truetype hinting gets more complicated the older the systems you want to support are, and for text sizes Truetype is still unfortunately the only format that “works” on screen in Windows.

2012-01-08 16:07:18 • Frode Bo Helland

Correcting the alignment issues you’re seeing does not require extensive superhinting of individual glyphs; as was mentioned it’s a much simpler issue with vertical zones.

BTW for older (read: Win-XP) systems in b&w you could use the GASP table to make all sizes render anti-aliased, resulting in blurry but passable rendering (essentially equivalent to OSX). However some of your thin parts might need to be thickened.
