December 6, 2009 • James Deux

I was a little peeved the other day. I find it ironic that the premiere program for print design has a bit of a… design flaw. Or is it just me?

Apparently, long copy conforms to the baseline. (Without my asking it to! How convenient, for one thing.) But I find it a little weird that the Baseline settings are found in the Preferences menu. And that, unlike the type with which it works, Baseline is adjusted in increments of inches/picas/etc., and not, point.

Is there a justification for this? It seems like a poorly placed control that’s set up in a way that makes it harder to control.

It certainly sounds like you have set Align To Baseline as a default option; usually it is off, for new documents.

The settings in Preferences are mixed: some of them Document-wide, others Application-wide. If you change any preference (any at all! imagine!) with no document open, it’ll become the default for new documents.

.. unlike the type with which it works, Baseline is adjusted in increments of inches/picas/etc., and not, point.

Its value is show in your very own preferred vertical measurement unit. One could argue that leading is measured in points in general (as well as in InDesign!), but one could also argue that the baseline grid is meant for the Respect the Grid guys, who don’t care for leading but want to define this grid in their own design units.
