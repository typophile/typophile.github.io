For positioning diacritics, don't rely on any one font. As you may already suspect, there is variance due to design and preference (and error). Start with the Unicode 5 book itself or download the individual charts as (free) PDFs from www.unicode.org if shelf space or cost is an issue. That particular glyph: Acircumflexgrave (1EA6) is in the "Latin Extended Additional" Range 1E00-1EFF. Then (as it appears you've already done) a survey of other fonts for any glyphs in question to see how they have been composed before. The more recent (available) examples such as Vista fonts and OS X fonts are likely to reflect recent additions/corrections. Hope this helps. D

You'll find significant differences of opinion on this, but the grave-on-left seems to have won out. One note on your examples- both the Palatino Linotype and Gentium ones are cut off at the top, caused by the mess that is line spacing on some platforms. The correct images look like this (including both versions of Gentium): [img:sites/default/files/old-images/PalL_3911.png] [img:sites/default/files/old-images/Gen_3434.png] Keep in mind that there are languages (typically tonal ones) for which the Vietnamese positioning of the circumflex/grave is not appropriate. They need something like this: [img:sites/default/files/old-images/GenNV_6234.png] Ideally, this is controlled by an OpenType or Graphite feature that will do the correct rendering based upon the language tags of the current text. Most applications can't handle this yet, though. Until then you can have separate font versions, or make the alternate a manual feature (as we have done in some fonts). Victor
