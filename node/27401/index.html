Disclaimer: this is just my own half-baked theory. I think most console games - even the newest ones - use their own custom purely-bitmap fonts. From a programming standpoint it would be madness to include vector data when the output is going to be a fixed size and will for all intents and purposes be displayed only on screen.

Retorik’s idea will work. For bonus clarity, when you resize the tiny type, size it proportionally by multiples of 100%; that way each tiny pixel becomes a perfect square in the larger size.

If you need to set a whole lot of text like this, you could try a font out there like SCR or some such.

2006-08-03 01:38:51 • pica pusher

Disclaimer: this is just my own half-baked theory.

But it’s mostly correct. The XBox 360 for example supports unhinted TrueType fonts, but as these need to be redrawn n-times a second few processes use them due to the performance hit. The Xbox dashboard being an exception.

So what games creators do is run a font through a utility and create a derivative bitmap font containing characters at a set size - these are then texture mapped onto surfaces, such as the game UI or other elements.
