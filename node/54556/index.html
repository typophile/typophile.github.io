A couple of warnings re. using AFMs to export/import kerning. AFMs have a presumed UPM of 1000, which means that if your font has a different UPM you will get rounding errors in kern pair values. FontLab, historically at any rate, had problems with importing AFM kern data over a certain size: pairs would be dropped. The real answer to this is Python scripts, which can very easily write lists of kern pairs for any UPM and import the same cleanly. Karsten Luecke wrote mine for me.
