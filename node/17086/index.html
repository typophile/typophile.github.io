James, have you given any of these fonts a Mac Name (second panel in FL 4.6)? This is the name used in non-OT aware apps in like MS Word/Mac. The name must not be longer than 29 characters or it will not appear in MS Word/Mac font menus. Normally, all members of OT families appear in MS Word/Mac (assuming their Mac Names are not too long). MS Word/Win only shows the roman (plain) member of each family. It sounds like some of your OT font Mac Names are too long for MS Word/Mac, so they are not all showing up. (I am not sure what happens if you leave the Mac Name blank. I think the system will get it by combining the Family Name and Style Name (OT names panel) but I am not sure. I don't ever leave it blank.)

That makes sense. This field in the OpenType Names panel relates to name table ID 18: Compatible Full (Macintosh only); On the Macintosh, the menu name is constructed using the FOND resource. This usually matches the Full Name. If you want the name of the font to appear differently than the Full Name, you can insert the Compatible Full Name in ID 18. It is OpenType related insofar as it was added in the OT specification, and was not previously part of the TT spec.
