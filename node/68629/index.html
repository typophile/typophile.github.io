S-cedilla and S-comma in Arial Unicode

March 22, 2010 • dtw

I’ve a sneaking suspicion that this has been raised before elsewhere, but hey… We’ve got articles on the web that mainly use Verdana, but which switch to a style whose font stack starts with Arial Unicode for strings that contain (relatively) obscure characters. Up pops a name like “Ş Erdoğmuş”, and the name displays wrongly. Looking at the code (in Courier!) it looks fine… printing it out (the print style sheet uses Georgia) it looks fine. On screen (with the style sheet that’s using Arial Unicode), we get s-comma instead of s-cedilla. Whassup with Arial Unicode here? 015E and 015F should be clear and separate from 0218 and 0219 shouldn’t they? Is this a designer mistake?
…as an aside, I’ve just checked the relevant characters in the default font - Verdana - and they’re all there so the immediate thing for me to tell our web team is not to switch to that alternative style for this name! … but the question about AU remains :-)
